# 华创证券有限责任公司机构业务管理办法

> 来源：用户上传的制度文件
> 整理时间：2026-03-07

---

## 一、总则

**第一条** 为规范公司主经纪商业务，促进业务稳定、有序的运行，根据《私募投资基金监督管理条例》、《私募投资基金监督管理暂行办法》、《私募投资基金服务业务管理办法（试行）》等相关规定制定本办法。

**第二条** 本办法适用于公司主经纪商业务服务主体的准入及合作项目的日常运行。

---

## 二、主经纪商业务释义

**第三条** 本办法所指主经纪商业务是指：为私募、信托等专业机构及高净值客户提供投资交易管理系统服务。

**投资交易管理系统服务**：为私募、信托等专业机构投资者，以及高净值客户提供投资交易管理系统的使用权。

---

## 三、主经纪商业务的组织架构

**第四条** 公司主经纪商业务的实施由机构经纪业务委员会统一部署，由机构经纪业务办公室统筹业务组织与管理，信息技术管理部、运维中心、杭州研发中心、合规与法律事务部、风险管理部、各分支机构及业务团队等协同分工，紧密配合，各司其职。

**第五条 各部门职责**

| 部门 | 职责 |
|------|------|
| 机构经纪业务委员会 | 遵循《华创证券有限责任公司执行委员会各专门委员会工作职责》的规定履行职责 |
| 机构经纪业务办公室 | 负责机构经纪业务的组织与管理；负责专业机构及高净值客户的准入审核；统筹管理公司 PB 业务系统 |
| 信息技术管理部 | 为专业机构及高净值客户提供投资交易管理系统系统运维服务 |
| 运维中心 | 为专业机构及高净值客户提供投资交易管理系统业务运维服务；负责日终清算交收 |
| 杭州研发中心 | 维护基金公社平台；为投资交易系统申请初审流程及电子档案管理提供技术保障 |
| 合规与法律事务部 | 负责主经纪商业务制度的合规审查；业务协议、合同的合规审核 |
| 风险管理部 | 负责主经纪商业务制度和客户准入的风险审查；交易行为监控 |
| 分支机构及业务团队 | 承揽主经纪商业务；负责对专业机构及高净值客户的尽调 |

---

## 四、主经纪商业务投资者适当性的管理和分工

**第六条** 主经纪商业务投资者适当性管理由公司统一部署，由机构经纪业务办公室、分支机构及业务团队共同完成。

1. **机构经纪业务办公室**：履行投资交易管理系统服务业务客户适当性统一管理职责，划分风险等级，制定准入标准
2. **分支机构及业务团队**：对客户进行投资者分类、风险承受能力评估

---

## 五、专业机构投资者及高净值客户的准入资格

**第七条 专业机构准入条件**
- 营业执照、税务登记证等证照齐全
- 私募基金须具备资产管理业务批复/私募基金管理人登记证明文件
- 产品结构、募资、投资者等各产品要素符合监管规定
- 提供各资质证明资料，且资料内容真实、准确、完整

**第八条 高净值客户准入条件**
- 同意并自愿承担使用投资交易管理系统可能导致的所有风险事项
- 证券投资资产 ≥ 50万元
- 申请专业投资交易管理系统的高净值客户原则上应为专业投资者
- 提供各资质证明资料，且资料内容真实、准确、完整

---

## 六、专业机构投资者及高净值客户的禁入条件

**第九条 私募基金、信托等专业机构禁入情形**

| 序号 | 禁入情形 |
|------|----------|
| 1 | 非境内合法注册的法人机构 |
| 2 | 存在重大违法违规记录 |
| 3 | 控股股东、实际控制人、董监高因违法被处罚或市场禁入 |
| 4 | 存在严重不良诚信记录 |
| 5 | 工商登记信息存在特殊异常 |
| 6 | 提供的资质证明及产品合同等相关材料失真 |
| 7 | 产品募集或投资管理过程中存在严重违法违规 |
| 8 | 未按合同约定使用资金，刻意隐瞒资金用途 |
| 9 | 发生严重影响偿债能力的重大事项 |
| 10 | 账户出借给他人使用 |
| 11 | 涉嫌场外配资 |
| 12 | 存在下设子账户、分账户、虚拟账户的情形 |
| 13 | 预期会存在内幕交易 |
| 14 | 预期会存在利益输送 |
| 15 | 预期会出现大量异常交易 |
| 16-17 | 机构经纪业务委员会认定的其他禁入条件 |

**第十条 高净值客户禁入情形**
- 存在重大违法违规记录
- 因违法被处罚、取消从业资格或市场禁入
- 提供的资质证明等相关材料不真实
- 存在严重不良诚信记录
- 账户出借给他人使用
- 涉嫌场外配资、非法集资、资金来源不合法
- 存在下设子账户、分账户、虚拟账户的情形
- 预期会存在内幕交易、利益输送、大量异常交易

## 七、反洗钱调查及监控

**第十一条** 分支机构及业务团队开展尽职调查时应包括反洗钱相关事项。

**第十二条 反洗钱背景调查内容**
1. 核查专业机构的成立时间、注册资本，评估历史产品运行情况
2. 评估专业机构注册资金与参投产品的自有资金比例是否合理
3. 评估专业机构出具的资金来源说明是否合理
4. 评估专业机构的主营业务是否涉及较高的洗钱风险
5. 专业机构是否具备开展相关业务的资质证明，所发行的产品是否通过备案
6. 其他反洗钱相关事项

**第十三条** 公司根据可疑交易相关规定对专业机构及高净值客户的交易行为进行监控。

---

## 八、主经纪商业务流程

**流程概览**
1. 分支机构及业务团队对拟合作对象进行尽职调查
2. 拟合作项目的预沟通
3. 合同的拟定、审核及签订
4. 账户的开立
5. 日常交易监控

**第十五条 尽职调查**
分支机构及业务团队遵照本办法及《华创证券有限责任公司主经纪商业务操作规范》《华创证券有限责任公司投资交易系统业务尽调指引》对申请服务的专业机构及高净值客户的资质、相关合作资料等进行尽调，并承担尽调责任。

**第十六条 预沟通**
PB专员就拟合作项目与机构经纪业务办公室进行预沟通，预沟通成功后可进入下一流程。

**第十七条 合同签订流程**
1. 机构经纪业务办公室协助PB专员拟定产品合同及协议
2. PB专员发起主经纪商业务申请流程
3. 司务附件遵循《华创证券有限责任公司主经纪商业务操作规范》的要求提交
4. 分支机构根据相关指引进行司务申报
5. 司务审批流程通过后，可签订相关合同、协议及用印

**第十八条 账户开立**
申请使用投资交易管理系统的专业机构及客户，由分支机构及业务团队PB专员为其向机构经纪业务办公室申请开立投资交易管理系统账户，并报备IP及MAC地址。

**第十九条 日常交易监控**
- 风险管理部将主经纪商业务客户通过投资交易系统开展的交易统一纳入公司客户交易行为管理进行监控
- 客户交易行为存在异常的，分支机构须及时联系客户，警示其合规交易
- 原则上，分支机构须在预警当天完成客户交易行为分析并在2个工作日内通过录音电话回访客户，5个工作日内提交流程

## 九、程序化交易的管理与报告

**第二十条** 依据《关于股票程序化交易报告工作有关事项的通知》要求，对程序化交易实施报告管理，未经报告的投资者不得进行程序化交易。

**第二十一条 程序化交易报告信息**
1. 账户基本信息（投资者名称、联系人、联系方式、证券账户代码等）
2. 账户资金信息（资金规模及来源、杠杆资金规模及来源、杠杆率等）
3. 交易信息（交易策略类型、交易指令执行方式、最高申报速率等）
4. 交易软件信息（软件名称及版本号、开发主体等）
5. 交易所规定的其他信息

**第二十二条 程序化交易报告流程**
1. 客户首次开展程序化交易前，应当向分支机构及业务团队报告，并填写《程序化交易投资者信息报告表》
2. 分支机构协同信息技术管理部、机构经纪业务办公室对报告表信息进行核查
3. 风险管理部对信息报告表进行审核
4. 审核后的信息报告表提交至运维中心，由运维中心向交易所进行报告

**第二十三条 程序化交易监控管理**
风险管理部负责程序化交易监测监控。程序化交易投资者报告信息发生重大变更的，应当在变更发生后按交易所要求时间内进行变更报告。

---

## 十、廉洁从业

**第二十四条** 参与主经纪商业务的各部门、分支机构及业务团队在执业过程中，做到维护客户和其他相关方的合法利益，遵守社会公德、商业道德、职业道德和行为规范。

**第二十五条 禁止行为**
1. 在信息技术服务外包、物品和服务的采购、项目招投标及人员招聘等事项中，违反公平公正原则
2. 通过互联网、自媒体等任何方式，诱导不满足适当性要求及合格投资者要求的客户参与相关业务
3. 协助客户通过提供虚假信息、伪造材料等方式，向不满足适当性要求及合格投资者要求的客户准入提供服务
4. 违规向第三方泄露利用工作便利获取的客户资料、账户信息、交易情况等
5. 侵占挪用客户资产或擅自变更委托投资范围
6. 通过返还佣金或者其他利益，违规给予部分客户特殊优待等方式，输送或者谋取不正当利益

---

## 十一、附则

**第二十六条** 本办法由公司机构经纪业务委员会解释和修订。

**第二十七条** 本办法自印发之日开始实施。

## 附录：PB系统引入流程各部门评估职责

PB系统引入流程的评估工作由信息技术管理部、杭州研发中心、风险管理部、合规与法律事务部、机构经纪业务办公室协同完成。

### 各部门评估职责

| 部门 | 评估要点 | 评估内容 |
|------|----------|----------|
| 杭州研发中心 | 系统上线相关资料 | 操作手册、测试报告、部署方案、运维手册、应急处理方案是否齐备 |
| 风险管理部 | 系统风控功能 | 是否具有风控模块；风控模块是否支持各项风控参数自定义设置；是否支持客户设置风控指标后经由公司审核 |
| 风险管理部 | 系统交易安全机制 | 单笔最大订单控制、订单流速控制、订单类型控制、撤单率控制、验资验券等后台安全机制 |
| 风险管理部 | 系统风控监测功能 | 实时查询资金、持仓、交易记录；实时监控风控模块的预警拦截情况；事后静态查询风控指标超限情况 |
| 机构经纪业务办公室 | 人员配备 | 是否确定系统相关业务管理、运维人员、审批人员、主备岗人员，并明确划分职责 |
| 信息技术管理部 | 审计功能 | 系统是否具备审计功能，对操作过程进行留痕记录，且记录保存时间不少于一年 |
| 信息技术管理部 | 系统架构 | 系统是否具有合理的架构，足够的性能、容量、可靠性、扩展性和安全性 |
| 合规与法律事务部 | 系统设计 | 业务系统的流程设计、功能设置、参数配置和技术实现是否遵循业务合规的原则 |
| 信息技术管理部 | 交易终端信息 | 系统是否记录、存储客户交易终端信息，并采取有效的技术措施保障相关信息真实、准确、完整 |
| 机构经纪业务办公室 | 系统测试 | 系统是否通过严格的业务测试、安全检测、压力测试，并提供相关测试报告 |
| 风险管理部 | 风控系统 | 是否同步上线与业务活动复杂程度和风险状况相适应的风险管理系统或相关功能 |
| 信息技术管理部 | 安全防护措施 | 系统是否具备完善的信息安全防护措施，能够保障经营数据和客户信息的安全、完整 |
| 信息技术管理部 | 应急保障措施 | 若系统发生技术故障等突发事件时，是否有应急措施保障业务正常进行 |
| 信息技术管理部 | 系统备份能力 | 是否具备符合要求的信息系统备份及运维管理能力 |
| 信息技术管理部 | 技术资料 | 是否提供了系统技术资料，至少包括系统架构图、网络拓扑图、设备配置参数、安装使用手册 |
