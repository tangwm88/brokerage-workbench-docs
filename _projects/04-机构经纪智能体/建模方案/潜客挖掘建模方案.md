# 潜客挖掘建模方案

> 基于公司机构客户库、朝阳永续私募数据库、证券基金业协会备案公示数据
> 版本：v1.0
> 日期：2026-03-07

---

## 一、概述

### 1.1 方案目标

基于三大数据源构建机构客户潜客挖掘系统：
- **公司机构客户库**：历史成交客户、活跃交易客户、沉睡客户
- **朝阳永续私募数据库**：私募基金管理人备案信息、管理规模、投资策略
- **证券基金业协会备案公示数据**：公募/私募基金管理人资质、产品备案信息

### 1.2 核心能力

1. **批量挖掘**：定时任务从三大数据源批量获取潜客信息
2. **实时挖掘**：新员工入职、客户询价等场景实时触发
3. **智能评分**：六大维度综合评分，自动排序推荐
4. **规则配置**：灵活配置挖掘规则，支持不同业务场景

## 三、挖掘规则设计

### 3.1 规则总览

```yaml
挖掘规则配置:
  版本: "v1.0"
  生效日期: "2026-03-07"
  
  规则分类:
    - 规模匹配规则（权重25%）
    - 策略匹配规则（权重25%）
    - 区域匹配规则（权重15%）
    - 合规筛选规则（权重20%）
    - 活跃度评估规则（权重10%）
    - 竞争态势规则（权重5%）
  
  评分机制:
    满分: 100分
    及格线: 60分
    优先级划分:
      高优先级: ≥80分
      中优先级: 60-79分
      低优先级: <60分
```

### 3.2 规模匹配规则（权重25%）

| 规则ID | 规则名称 | 条件 | 评分权重 |
|--------|----------|------|----------|
| SCALE_001 | 大型机构识别 | 管理规模 ≥ 50亿 | 25分 |
| SCALE_002 | 中型机构识别 | 管理规模 10-50亿 | 18分 |
| SCALE_003 | 成长型机构识别 | 管理规模 1-10亿且成立<3年 | 15分 |
| SCALE_004 | 小型机构识别 | 管理规模 < 1亿 | 8分 |

**数据源映射：**
- 朝阳永续：取最新季度管理规模数据
- 基金业协会：取备案时申报规模，结合产品数量推算
- 内部库：取历史成交最大规模或当前持仓规模

### 3.3 策略匹配规则（权重25%）

| 规则ID | 规则名称 | 条件 | 匹配度 |
|--------|----------|------|--------|
| STRATEGY_001 | 量化策略匹配 | 投资策略含"量化""程序化" | 高匹配（25分） |
| STRATEGY_002 | 权益策略匹配 | 投资策略含"股票""权益" | 高匹配（25分） |
| STRATEGY_003 | 债券策略匹配 | 投资策略含"债券""固收" | 中匹配（18分） |
| STRATEGY_004 | 混合策略匹配 | 投资策略含"混合""多策略" | 中匹配（18分） |
| STRATEGY_005 | 衍生品策略匹配 | 投资策略含"期货""期权""衍生品" | 高匹配（25分） |

**数据源映射：**
- 朝阳永续：fund_strategy字段
- 基金业协会：投资策略描述
- 内部库：客户历史交易品种偏好

### 3.4 区域匹配规则（权重15%）

| 规则ID | 规则名称 | 条件 | 评分 |
|--------|----------|------|------|
| REGION_001 | 核心区域 | 注册地在北京/上海/深圳 | 15分 |
| REGION_002 | 重点区域 | 注册地在杭州/广州/成都/南京 | 12分 |
| REGION_003 | 一般区域 | 其他省会城市 | 8分 |
| REGION_004 | 就近服务 | 与当前客户经理同区域 | +5分（额外加分） |

### 3.5 合规筛选规则（权重20%）

| 规则ID | 规则名称 | 条件 | 处理方式 |
|--------|----------|------|----------|
| COMPLIANCE_001 | 正常机构 | 合规状态=正常，无处罚记录 | 正常评分（20分） |
| COMPLIANCE_002 | 预警机构 | 近1年内有监管关注函 | 扣分10分，标记预警 |
| COMPLIANCE_003 | 高风险机构 | 近1年内有监管处罚 | 直接过滤，不推荐 |
| COMPLIANCE_004 | 新备案机构 | 备案时间<6个月 | 加分5分（新客拓展） |
| COMPLIANCE_005 | 即将到期 | 备案资质剩余<3个月 | 扣分5分，标记关注 |

**数据源映射：**
- 基金业协会：诚信信息公示、备案状态
- 朝阳永续：机构诚信标签
- 内部库：公司合规风控黑名单

### 3.6 活跃度评估规则（权重10%）

| 规则ID | 规则名称 | 条件 | 评分 |
|--------|----------|------|------|

| ACTIVITY_002 | 中活跃 | 近6个月新发产品1-2只 | 6分 |
| ACTIVITY_003 | 低活跃 | 近6个月无新发产品 | 2分 |
| ACTIVITY_004 | 沉睡唤醒 | 历史活跃但近1年无新发 | 5分（特殊标记） |

**数据源映射：**
- 朝阳永续：产品备案时间序列
- 基金业协会：产品备案公示

### 3.7 竞争态势规则（权重5%）

| 规则ID | 规则名称 | 条件 | 评分 |
|--------|----------|------|------|
| COMPETE_001 | 空白客户 | 无券商合作记录 | 5分 |

## 四、挖掘流程设计

### 4.1 批量挖掘流程

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  数据源同步   │───▶│  数据清洗    │───▶│  规则匹配    │
│  (定时任务)   │    │  (标准化)    │    │  (评分计算)  │
└──────────────┘    └──────────────┘    └──────────────┘
                                               │
                                               ▼
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  结果通知    │◀───│  人工审核    │◀───│  结果排序    │
│  (推送员工)   │    │  (确认/忽略)  │    │  (优先级)    │
└──────────────┘    └──────────────┘    └──────────────┘
```

### 4.2 实时挖掘流程

**触发条件：**
- 新员工入职（分配区域客户）
- 客户主动询价（官网/电话）
- 市场活动报名（展会/路演）
- 竞品客户流失预警

**处理逻辑：**
1. 实时查询三大数据源
2. 即时计算匹配度
3. 推送给责任员工
4. 24小时内必须响应

## 六、API接口设计

### 6.1 批量挖掘接口

```http
POST /api/v1/prospecting/batch
Content-Type: application/json

{
  "data_sources": [1, 2, 3],
  "rule_config": "default",
  "limit": 100,
  "callback_url": "https://example.com/callback"
}

Response:
{
  "code": 200,
  "data": {
    "task_id": "TASK202603070001",
    "status": "processing",
    "estimated_time": "5min"
  }
}
```
