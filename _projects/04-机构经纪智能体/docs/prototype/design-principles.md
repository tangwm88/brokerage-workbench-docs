# 设计原则与规范

## 一、核心设计原则

### 1. 简洁明了
- 功能边界清晰，使用门槛低
- 3-4步完成核心业务
- 避免信息过载

### 2. 预判驱动
- 主动识别用户需求
- 提前准备下一步动作
- 减少用户决策负担

### 3. 移动端优先
- 375px为基准设计
- 单手操作友好
- 触控区域不小于44px

### 4. 一致性
- 遵循Vant4设计规范
- 统一的色彩、字体、间距
- 一致的交互反馈

## 三、交互规范

### 按钮状态
- 默认：主色背景
- 按下：透明度0.9
- 禁用：灰度背景
- 加载：显示loading图标

### 页面转场
- 前进：从右向左滑入
- 后退：从左向右滑出
- 时长：300ms
- 缓动：ease-out

### 反馈规范
- Toast：2秒自动消失
- Dialog：需用户确认
- Loading：操作完成消失
- 震动：错误时震动反馈

---

## 四、无障碍规范

### 颜色对比度
- 文字与背景对比度 ≥ 4.5:1
- 大号文字对比度 ≥ 3:1

### 触控区域
- 最小触控区域：44×44px
- 相邻触控区域间距：≥8px

### 屏幕阅读器
- 所有图片添加alt属性
- 表单元素添加label
- 按钮说明操作结果

## 五、安全区域适配

### iOS安全区域
```css
.safe-area-top {
  padding-top: env(safe-area-inset-top);
}

.safe-area-bottom {
  padding-bottom: env(safe-area-inset-bottom);
}
```

### 底部固定元素
- 距离底部：env(safe-area-inset-bottom) + 16px
- 背景色：#fff（防透明穿透）
