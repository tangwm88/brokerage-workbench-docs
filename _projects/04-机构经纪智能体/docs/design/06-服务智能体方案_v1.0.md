## 一、方案概述

### 1.1 基本信息
| 项目 | 内容 |
|------|------|
| **方案名称** | 机构经纪服务智能体方案 |
| **定位** | 公司八大应用智能体之一（第4个） |
| **覆盖业务** | 资产管理、投资银行、债券销售交易、机构经纪、财富管理、信用交易、研究所、固收投资 |
| **核心目标** | 让员工3-4步完成机构经纪业务，无需跨部门协调 |
| **战略价值** | 体现"割裂的多部门"→"统一能力平台"的组织转型 |

### 1.2 设计原则
- **简洁明了**：功能边界清晰，使用门槛低
- **预判驱动**：主动识别需求，提前准备动作
- **可连接性**：支持智能体之间的协作编排
- **底座优先**：深度调用公司自动驾驶大模型底座能力

## 五、考核机制方案（AI驱动双重考核）

### 5.1 考核体系

| 考核对象 | 考核类型 | 考核内容 | AI采集数据 | 考核方式 |
|----------|----------|----------|------------|----------|
| **业务支持中心（中后台）** | 公司总量考核 | 整体业务目标达成率、服务质量、底座能力贡献度 | 客户转化总数、服务满意度、数据回流质量 | AI自动统计，实时看板 |
| **前台业务团队（客户经理）** | 单项业务达成考核 | 个人客户引入数、账户开立数、交易服务量、客户服务评分 | 个人业绩看板、客户跟进及时率、业务闭环率 | AI自动统计，日/周/月报 |

### 5.2 AI驱动考核机制说明

1. **实时数据采集**
   - 底座自动记录每个动作的执行情况和结果
   - 全流程留痕，可追溯

2. **智能评分**
   - 基于业务规则自动计算考核得分
   - 多维度权重可配置

3. **预警提醒**
   - 考核指标偏离时自动提醒相关人员
   - 支持自定义预警阈值

4. **双向反馈**
   - 前台业绩数据回流到底座
   - 优化客户画像和风控规则

## 八、多视角改进点整合

### 8.1 前端员工视角改进

**角色重新定义：**
- 客户经理 = AI训练师 + 高端服务专家
- AI处理：标准化、重复性工作
- 人处理：关系维护、复杂谈判、危机处理
- 考核转变：从"开户数"→"AI训练贡献度+客户满意度+高价值服务数"

**人工接管机制：**
```
紧急接管触发条件：
- 客户明确要求"找人工"
- 交易金额超过阈值（单笔>1000万）
- 系统故障或响应超时（>10秒）
- AI连续3次无法满足客户需求

接管方式：一键切换，AI退出，人工接手，保留上下文
```

**客户关怀提醒：**
- AI完成基础服务后，自动提醒客户经理跟进
- 提供客户画像摘要（近期交易、持仓变化、潜在需求）
- 生成关怀话术建议

**零培训上手设计：**
- 界面极简：3个按钮（发起服务、查看进度、紧急接管）
- 小抄卡片：快捷键指南
- 沙箱演练：5分钟学会
