```
    ├── 5.1 业务概览
    ├── 5.2 业绩指标
    ├── 5.3 趋势分析
    ├── 5.4 排行榜
    └── 5.5 业绩分析
```

### 1.2 闭环设计原则

每个功能必须包含：
1. **入口触发** - 用户如何发起操作
2. **信息收集** - 需要哪些输入信息
3. **AI处理** - 智能体如何处理
4. **底座调用** - 需要调用哪些底座能力
5. **结果输出** - 输出什么结果给用户
6. **后续动作** - 用户可以继续做什么
7. **状态跟踪** - 如何跟踪处理进度

## 二、提示事项模块（闭环设计）

### 2.1 重大事项请示

#### 2.1.1 功能定位
向公司领导层提交需要决策的重大事项，包括战略客户政策、重大资源投入、跨部门协调等。

#### 2.1.2 闭环流程

**Step 1: 入口触发**
- 用户输入："我要申请战略客户费率优惠" / "重大事项请示"
- AI识别意图，进入请示流程

**Step 2: 信息收集（AI引导填写）**
```
AI: 请提供以下信息：
1. 请示类型（单选）：
   □ 战略客户政策  □ 重大资源投入  □ 跨部门协调  □ 其他

2. 客户/事项名称：__________

3. 请示背景（AI可自动生成初稿）：
   [客户基本情况...]
   [业务机会分析...]

4. 具体诉求：
   □ 费率优惠 ____%  □ 专家支持  □ 系统资源  □ 其他____

5. 预期收益：__________

6. 紧急程度：□ 普通  □ 紧急  □ 特急
```

**Step 3: AI智能处理**
- 自动查重：检查是否已有类似请示
- 生成草稿：基于模板生成正式请示文档
- 风险评估：评估资源投入风险
- 推荐审批人：根据类型推荐审批领导

**Step 4: 底座调用**
| 底座能力 | 用途 |
|----------|------|
| 统一语料 | 查询客户历史信息 |
| 关系网络 | 分析客户关联关系 |
| 风控规则 | 评估政策风险 |
| 工作流引擎 | 提交审批流程 |

**Step 5: 结果输出**
- 生成正式请示单（带编号）
- 提交至审批系统
- 返回跟踪链接

**Step 6: 后续动作**
- 查看审批进度
- 补充材料
- 撤回申请
- 催办提醒

**Step 7: 状态跟踪**
```
状态流转：草稿 → 已提交 → 审批中 → 已通过/已驳回
           ↓        ↓         ↓         ↓
        可编辑   可撤回    可催办    可归档
```

#### 2.1.4 数据模型

```sql
-- 重大事项请示表
CREATE TABLE major_requests (
    id BIGSERIAL PRIMARY KEY,
    request_no VARCHAR(32) UNIQUE NOT NULL,     -- 请示编号：REQ-20260308-001
    request_type VARCHAR(50) NOT NULL,           -- 类型：战略客户政策/重大资源投入/跨部门协调
    
    -- 申请人信息
    requester_id VARCHAR(50) NOT NULL,           -- 申请人ID
    requester_name VARCHAR(100),                 -- 申请人姓名
    requester_dept VARCHAR(100),                 -- 所属部门
    
    -- 客户/事项信息
    customer_id BIGINT,                          -- 关联客户ID
    customer_name VARCHAR(200),                  -- 客户名称
    
    -- 请示内容（AI生成+人工编辑）
    title VARCHAR(200) NOT NULL,                 -- 标题
    background TEXT,                             -- 背景
    request_content TEXT,                        -- 具体诉求
    expected_benefit TEXT,                       -- 预期收益
    
    -- AI辅助信息
    ai_draft TEXT,                               -- AI生成草稿
    risk_assessment TEXT,                        -- 风险评估
    similar_cases JSONB,                         -- 相似案例
    
    -- 审批链
    current_approver_id VARCHAR(50),             -- 当前审批人
    approval_chain JSONB,                        -- 预设审批链
    approval_history JSONB,                      -- 审批历史
    
    -- 状态
    status VARCHAR(20) DEFAULT 'draft',          -- draft/submitted/approving/approved/rejected
    urgency VARCHAR(10) DEFAULT 'normal',        -- normal/urgent/critical
    
    -- 时间
    created_at TIMESTAMP DEFAULT NOW(),
    submitted_at TIMESTAMP,                      -- 提交时间
    completed_at TIMESTAMP,                      -- 完成时间
    
    -- 附件
    attachments JSONB                            -- 附件列表
);
```

### 2.2 问题反馈

#### 2.2.1 功能定位
向相关部门反馈业务开展中遇到的问题，包括系统问题、流程问题、协作问题等。

#### 2.2.2 闭环流程

**Step 1: 入口触发**
- 用户输入："系统无法登录" / "开户流程太慢" / "问题反馈"

**Step 2: 信息收集**
```
AI: 请描述您遇到的问题：

1. 问题类型（单选）：
   □ 系统故障  □ 流程问题  □ 协作障碍  □ 其他

2. 问题描述：__________

3. 发生时间：__________

4. 影响范围：□ 单个客户  □ 多个客户  □ 全部客户

5. 紧急程度：□ 低  □ 中  □ 高（影响业务开展）

6. 截图/附件：[上传]
```

**Step 3: AI智能处理**
- 问题分类：自动归类到对应处理部门
- 相似问题匹配：查找历史解决方案
- 自动派单：根据类型分配给对应团队
- 优先级评估：结合紧急程度和影响范围

**Step 4: 底座调用**
| 底座能力 | 用途 |
|----------|------|
| 统一语料 | 查询历史问题库 |
| 工作流引擎 | 派发工单 |
| 知识库 | 匹配解决方案 |

**Step 5: 结果输出**
- 生成工单号
- 告知预计处理时间
- 提供临时解决方案（如有）

**Step 6: 后续动作**
- 补充信息
- 催办
- 评价处理结果
- 关闭工单

**Step 7: 状态跟踪**
```
状态流转：已提交 → 已受理 → 处理中 → 待确认 → 已解决/已关闭
           ↓         ↓         ↓        ↓
        可补充    可催办    可沟通   可评价
```

## 七、实施建议

### 7.1 优先级排序

| 优先级 | 模块 | 功能 | 原因 |
|--------|------|------|------|
| P0 | 提示事项 | 重大事项请示 | 核心决策支持 |
| P0 | 业务组织 | 确认前端客户 | 业务入口 |
| P0 | 业务组织 | 开户审核确认 | 核心业务 |
| P1 | 风险提示 | 集中度风险 | 风控刚需 |
| P1 | 业绩看板 | 业务概览 | 管理刚需 |
| P2 | 其他 | 其他功能 | 后续迭代 |

### 7.2 技术实现要点

1. **统一状态机**：所有功能使用统一的状态流转机制
2. **消息通知**：关键节点推送消息通知
3. **权限控制**：不同角色看到不同功能
4. **数据留痕**：所有操作记录审计日志
5. **AI辅助**：关键决策点提供AI建议
