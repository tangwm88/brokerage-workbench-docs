### 1.1 AI潜客挖掘提示词模板

#### 场景1：企业官网访客分析
```
角色设定：
你是一位B2B销售分析师，擅长从企业行为数据中发现高价值潜客。

分析任务：
请根据以下访客行为数据，评估该访客的购买意向等级：
- 访问页面：{visited_pages}
- 停留时长：{total_duration}秒
- 下载资料：{downloaded_materials}
- 来源渠道：{utm_source}

输出要求：
1. 意向等级判定（S/A/B/C/D）
2. 判定依据（2-3条关键行为特征）
3. 建议跟进时机（立即/24小时内/3天内）
4. 推荐触达方式（电话/邮件/微信）
```

### 1.2 客户评估沟通示例

#### 评分维度说明沟通

**维度1：需求匹配度（权重25%）**
```
评估沟通：
"了解您的业务痛点后，我们发现您目前面临的[具体痛点]与我们的[解决方案]有较高契合度。\n
让我为您做一个需求匹配度分析：\n
✓ 您的业务场景：{business_scenario}\n✓ 核心痛点描述：{pain_points}\n✓ 期望解决时间：{expected_timeline}\n✓ 预算区间：{budget_range}\n
【评分标准】\n- 10分：痛点精准匹配，需求明确，决策紧迫\n- 7-9分：主要需求匹配，部分细节待确认\n- 4-6分：潜在需求存在，需进一步引导\n- 1-3分：需求模糊，匹配度低\n
当前评分：{score}/10"
```

**维度2：决策影响力（权重25%）**
```
评估沟通：
"为了更好地为您服务，我需要了解一下贵司的决策流程。请问：\n
1. 在采购[产品/服务]方面，最终决策是由您主导还是需多方审批？\n2. 通常需要哪些部门或人员参与决策？\n3. 预算审批周期大概是多长？\n
【评分标准】\n- 10分：最终决策者，可单独拍板\n- 7-9分：核心影响者，有重要建议权\n- 4-6分：参与决策，非主导角色\n- 1-3分：信息收集者，无决策权\n
当前评分：{score}/10"
```

**维度3：预算能力（权重20%）**
```
评估沟通：
"关于项目预算，我们希望确保方案的性价比最优。可否了解一下：\n
1. 贵司本年度的[相关类别]预算规模？\n2. 针对此次需求，预算范围大致在什么区间？\n3. 预算是否已获得内部批准？\n
【评分标准】\n- 10分：预算充足且已获批，可立即执行\n- 7-9分：预算基本满足，审批流程中\n- 4-6分：预算有限，需协商方案\n- 1-3分：无明确预算，仅信息收集\n
当前评分：{score}/10"
```

**维度4：时间紧迫度（权重15%）**
```
评估沟通：
"了解您的时间规划有助于我们提供最合适的服务节奏。请问：\n
1. 您希望何时正式启动该项目？\n2. 是否有特定的截止期限或里程碑？\n3. 延迟启动会有什么影响？\n
【评分标准】\n- 10分：立即启动，有硬性deadline\n- 7-9分：1个月内需启动\n- 4-6分：3个月内有计划\n- 1-3分：时间灵活，无明确规划\n
当前评分：{score}/10"
```

**维度5：竞争格局（权重15%）**
```
评估沟通：
"了解竞争情况有助于我们展示差异化价值。冒昧问一下：\n
1. 您是否还在对比其他供应商？\n2. 目前有哪些方案在您的考虑范围内？\n3. 您最看重供应商的哪些特质？\n
【评分标准】\n- 10分：我方是唯一首选，无竞品\n- 7-9分：我方优势明显，在短名单内\n- 4-6分：多方比较中，优势不明显\n- 1-3分：竞品主导，我方备胎\n
当前评分：{score}/10"
```

#### 3.1.2 等级划分标准

| 等级 | 分数区间 | 定义 | 处理策略 | 资源投入 |
|-----|---------|-----|---------|---------|
| **S级** | 90-100分 | 高价值+高意向+高紧迫 | 立即跟进，高管介入 | 优先级最高 |
| **A级** | 75-89分 | 中高价值+明确意向 | 24小时内跟进 | 优先资源 |
| **B级** | 60-74分 | 潜在价值+需求待确认 | 72小时内跟进 | 标准流程 |
| **C级** | 40-59分 | 低紧迫或信息不全 | 定期培育 | 自动化+人工 |
| **D级** | 0-39分 | 无意向或完全不符 | 进入公海池 | 暂不投入 |

#### 3.1.3 评分置信度规则

```yaml
置信度计算:
  
  高置信度 (0.8-1.0):
    条件: 
      - 人工已确认评分
      - 关键字段完整度 > 80%
      - 沟通记录字数 > 200字
    处理: 直接采用AI评分，无需复核
    
  中置信度 (0.5-0.79):
    条件:
      - 部分字段缺失
      - 或沟通记录较简略
    处理: AI评分+销售确认
    
  低置信度 (0-0.49):
    条件:
      - 关键信息严重不足
      - 或数据存在矛盾
    处理: 强制人工复核，AI仅作参考
```

#### 3.1.4 评分动态调整规则

```
评分更新触发条件:

1. 正向调整 (+5分以上):
   - 客户明确表达购买意向
   - 预算超预期或已获批
   - 决策周期大幅缩短
   - 竞品退出或我方明显优势
   
2. 负向调整 (-5分以上):
   - 预算大幅削减或取消
   - 决策时间无限期推迟
   - 关键决策人变更
   - 竞品已签约或明显领先
   
3. 自动降级:
   - 连续3次跟进无响应 → 降1级
   - 超过SLA时效未跟进 → 降1级
   - 明确拒绝或不需要 → 降为D级
```

### 3.2 线索分配规则

#### 3.2.1 负载均衡规则

```yaml
负载均衡算法:
  
  算法类型: 加权轮询 + 能力匹配
  
  负载计算公式:
    综合负载 = 当前跟进线索数 × 0.4 + 
              本周新增线索数 × 0.3 + 
              当前商机金额 × 0.00001 × 0.3
    
  负载等级:
    低负载: 综合负载 < 阈值下限 → 优先分配新线索
    中负载: 阈值下限 ≤ 综合负载 ≤ 阈值上限 → 正常分配
    高负载: 综合负载 > 阈值上限 → 暂缓分配
    
  动态调整:
    - 每日凌晨重新计算各销售负载
    - 高负载人员触发自动分流
    - 支持手动负载调整（经理权限）
```

#### 3.2.2 匹配度规则

| 匹配维度 | 权重 | 匹配规则 | 优先级 |
|---------|-----|---------|-------|
| **行业匹配** | 30% | 销售历史成交行业 vs 客户行业 | 高 |
| **区域匹配** | 25% | 销售负责区域 vs 客户注册地 | 高 |
| **规模匹配** | 20% | 销售擅长客户规模 vs 客户规模 | 中 |
| **产品匹配** | 15% | 销售产品专长 vs 客户需求 | 中 |
| **来源匹配** | 10% | 销售渠道专长 vs 线索来源 | 低 |

```yaml
匹配度算法:
  
  综合匹配度 = Σ(各维度匹配分 × 维度权重)
  
  匹配度等级:
    A级匹配: 综合匹配度 ≥ 80分 → 优先分配
    B级匹配: 60分 ≤ 综合匹配度 < 80分 → 正常分配
    C级匹配: 综合匹配度 < 60分 → 备选分配
    
  特殊规则:
    - 老客户转介绍：优先分配给原对接人
    - 指定客户：按客户归属强制分配
    - 冲突处理：匹配度相同时，负载低者优先
```

    流程: 进入公海池 → 销售自主认领 → 经理审批 → 确认分配

#### 3.4.2 转化指标计算

```yaml
核心转化指标:
  
  阶段转化率:
    公式: 下一阶段数量 / 当前阶段数量 × 100%
    示例: 需求确认→方案报价转化率 = 进入方案阶段数 / 完成需求调研数
    
  整体转化率:
    公式: 赢单数 / 线索总数 × 100%
    统计周期: 按月/季度/年度
    
  阶段流失率:
    公式: 流失数 / 进入该阶段总数 × 100%
    流失定义: 30天内无推进且未进入下一阶段
    
  平均转化周期:
    公式: Σ(赢单转化天数) / 赢单总数
    细分: 按线索等级、行业、产品线分别统计
    
  销售漏斗健康度:
    公式: 各阶段商机金额 × 阶段赢率权重
    预警: 某阶段金额突降或突增触发分析
```

| 商务谈判 | 20% | 80% | 按预估金额×80% | 最终冲刺 |
