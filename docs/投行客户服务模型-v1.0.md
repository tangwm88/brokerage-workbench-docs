# 投行客户服务模型 v1.0

> **版本**：v1.0
> **日期**：2026-05-31
> **定位**：独立业务标准模型，服务"客户关系持续经营"核心诉求
> **依据**：智能体业务模型设计方法论-统一参考.md + 投行定增业务分析模型 v5.0 角色体系

---

## 一、模型定位

**模型名称**：投行客户服务模型  
**模型类型**：业务标准模型（不是动作标准模型）  
**目标角色**：
- **主要调用者**：客户经理（判断客户关系状态、发现新需求、维系深度）
- **参考使用者**：组织者（判断客户价值分层、资源倾斜）、产品支持（判断交叉销售机会）

**核心诉求**：这个客户现在跟我们关系怎么样？还能做什么业务？怎么维系？有没有流失风险？

**适用场景**：
1. 项目完成后持续服务（定增/IPO/可转债等项目交割后）
2. 定期客户复盘（季度/年度客户关系体检）
3. 新需求发现（从存量客户中挖掘增量业务）
4. 流失风险预警（关键客户 relationship 变化监测）

## 三、本质维度（5个）

> **硬性标准**：5个维度，不能再多。每个维度讲人话，有客观数据支撑。

| # | 维度名称 | 讲人话 | 客观数据来源 | 谁最关心 |
|---|---------|--------|------------|---------|
| 1 | **服务后遗症** | 上次的项目做得怎么样？客户满不满意？有没有遗留问题没解决？ | 项目交割记录 / 客户反馈 / 问题清单 / 投诉记录 | 客户经理、组织者 |
| 2 | **需求层次** | 客户现在还需要什么？是显性的（明确要做的事）还是隐性的（没说但可能需要的）？近期还是远期？ | 客户公开信息 / 行业动态 / 公司公告 / 竞品动态 | 客户经理、产品支持 |
| 3 | **关系深度** | 我们跟客户关键人的关系铁不铁？是只认识一个人，还是打通了整个决策链？有没有私交？ | 互动频次 / 拜访记录 / 决策链覆盖 / 非业务互动 | 客户经理、组织者 |
| 4 | **信任程度** | 客户信不信任我们？是只把我们当服务商，还是当战略伙伴？关键时刻会不会第一个想到我们？ | 复购率 / 推荐行为 / 关键决策参与度 / 信息开放度 | 客户经理 |
| 5 | **流失风险** | 这个客户有没有可能跑掉？竞争对手在不在挖？客户内部有没有变化？ | 竞品动态 / 客户人事变动 / 服务中断信号 / 市场传言 | 组织者、客户经理 |

## 四、维度详解

### 维度1：服务后遗症

**本质目标**：上次项目做完后，客户到底满不满意？有没有后遗症没处理？这是继续服务的基础。

**分析框架**：
```
【项目交割质量】
  ├─ 项目是否按期完成：按时/延期（延期天数）
  ├─ 项目成果是否达标：完全达标/基本达标/未达标
  ├─ 客户验收态度：满意/基本满意/有意见/拒收
  └─ 验收问题清单：已解决XX项 / 待解决XX项 / 新增XX项

【遗留问题】
  ├─ 是否有未解决事项：有/无（如有，清单+责任人+截止日期）
  ├─ 客户投诉记录：有/无（如有，次数、内容、处理状态）
  ├─ 后续服务承诺：已兑现/部分兑现/未兑现
  └─ 客户主动反馈频次：经常/偶尔/从不（反馈内容摘要）

【服务感知】
  ├─ 客户对我们服务速度的评价：快/一般/慢
  ├─ 客户对我们专业能力的评价：高/中/低
  ├─ 客户对我们响应态度的评价：好/一般/差
  └─ 与同业的对比评价：优于/持平/弱于同业

【判断标准】
  ├─ [通过] 良好：交割顺利，客户满意，无重大遗留问题
  ├─ [注意] 需关注：有遗留问题待解决，或客户有微词，需主动修复
  └─ [不通过] 差：客户不满，遗留问题严重，需立即介入修复关系
```

**关键输出**：
```
服务后遗症评分：X/5
├─ 交割质量：X/5（依据：按期/达标/验收态度）
├─ 遗留问题：X/5（依据：待解决项数/投诉次数）
├─ 服务感知：X/5（依据：客户评价/同业对比）
└─ 综合判断：[通过]/[注意]/[不通过]

遗留问题清单：
1. [高优先级] XX问题（截止XX日，责任人：XX，状态：进行中）
2. [中优先级] XX问题（截止XX日，责任人：XX，状态：待启动）
3. [低优先级] XX问题（截止XX日，责任人：XX，状态：计划中）

修复建议：
- 立即行动：XX（针对[不通过]项）
- 本周完成：XX（针对[注意]项）
```

### 维度2：需求层次

**本质目标**：客户现在和将来需要什么？不仅是他说的，还有他没说但可能要做的。需求分层次，从浅到深。

**分析框架**：
```
【显性需求】（客户明确表达或公开宣布的）
  ├─ 近期明确需求：
  │   ├─ 需求1：XX（如：再融资、并购重组、债券发行）
  │   ├─ 预计时间窗口：XX月
  │   └─ 客户准备度：已启动/计划中/意向阶段
  ├─ 远期规划需求：
  │   ├─ 战略方向：XX（如：产业链整合、国际化、技术升级）
  │   ├─ 预计触发条件：XX（如：产能利用率达到XX%、技术突破）
  │   └─ 我方匹配度：高/中/低
  └─ 显性需求价值预估：XX万元（按同类项目平均收费）

【隐性需求】（客户没说，但基于行业规律和客户状态可以推断的）
  ├─ 行业驱动型：
  │   ├─ 行业趋势：XX（如：行业整合加速、监管政策变化）
  │   ├─ 客户必选项：XX（如：环保合规升级、数字化转型）
  │   └─ 我方切入点：XX
  ├─ 客户状态型：
  │   ├─ 财务信号：XX（如：现金流紧张、负债率上升）
  │   ├─ 经营信号：XX（如：产能扩张、新业务孵化）
  │   └─ 我方切入点：XX
  ├─ 竞争驱动型：
  │   ├─ 竞争对手动态：XX（如：竞对做了并购、发了债）
  │   ├─ 客户可能的反应：XX（跟进/观望/差异化应对）
  │   └─ 我方切入点：XX
  └─ 隐性需求价值预估：XX万元

【需求紧迫度排序】
  1. [紧急+高价值] XX需求（预计XX月启动，价值XX万）
  2. [计划内+中价值] XX需求（预计XX季度，价值XX万）
  3. [远期+高价值] XX需求（预计XX年，价值XX万）
  4. [机会型] XX需求（需主动培育，价值XX万）

【判断标准】
  ├─ [通过] 需求明确：显性或隐性需求清晰，我方有匹配能力
  ├─ [注意] 需培育：有需求信号但不明确，需主动挖掘和培育
  └─ [不通过] 需求不清：当前无明显需求信号，建议保持触达
```

**关键输出**：
```
需求层次评分：X/5
├─ 显性需求清晰度：X/5（依据：近期需求数量/准备度/价值）
├─ 隐性需求挖掘度：X/5（依据：行业/客户/竞争信号识别数量）
├─ 需求与我方匹配度：X/5（依据：我方能力覆盖比例）
└─ 综合判断：[通过]/[注意]/[不通过]

需求清单（按紧迫度排序）：
1. [P0-紧急] XX需求 | 预计XX月 | 价值XX万 | 我方准备度：就绪/需准备/需培育
2. [P1-计划内] XX需求 | 预计XX季度 | 价值XX万 | 我方准备度：XX
3. [P2-远期] XX需求 | 预计XX年 | 价值XX万 | 我方准备度：XX
4. [P3-机会型] XX需求 | 待触发 | 价值XX万 | 我方准备度：XX

下一步行动：
- 立即跟进：XX（P0需求，本周内拜访确认）
- 主动培育：XX（P1/P2需求，准备案例和方案）
- 保持监测：XX（P3需求，设提醒定期跟踪）
```

### 维度3：关系深度

**本质目标**：我们跟客户的关系到底到哪个层面了？是只认识一个经办人，还是打到了董事长？关系网络健不健康？

**分析框架**：
```
【决策链覆盖】
  ├─ 最高决策层：
  │   ├─ 董事长/CEO：认识/听说过/不认识
  │   ├─ 互动频次：每月/每季/每年/从未
  │   └─ 互动深度：业务交流/私人交往/战略合作讨论
  ├─ 执行决策层：
  │   ├─ 董秘/财务总监/投资总监：认识/听说过/不认识
  │   ├─ 互动频次：每周/每月/每季
  │   └─ 互动深度：业务对接/方案讨论/私人交往
  ├─ 经办操作层：
  │   ├─ 证券事务代表/具体经办人：认识/听说过/不认识
  │   ├─ 互动频次：每周/每月
  │   └─ 互动深度：日常对接/信息传递/私人交往
  └─ 覆盖健康度：
      ├─ 单层关系（只认识经办人）：脆弱
      ├─ 双层关系（认识经办+执行层）：一般
      ├─ 三层打通（经办+执行+决策层）：健康
      └─ 多层多线（每个层级不止一条线）：非常健康

【关系触点频次】
  ├─ 业务互动：近3个月XX次（行业平均：XX次）
  ├─ 非业务互动：近3个月XX次（如：活动邀请、节日问候、私人聚会）
  ├─ 信息获取：近3个月从客户处主动获取信息XX条
  └─ 我方主动触达：近3个月主动触达XX次（电话/拜访/邮件）

【关系质量】
  ├─ 客户主动找我们的频次：经常/偶尔/从不
  ├─ 客户对我们信息开放度：高（主动分享）/中（有问有答）/低（被动回应）
  ├─ 客户是否引荐其他资源：有（XX次）/无
  └─ 同业的对比：我们 vs 主要竞对（谁的关系更深）

【判断标准】
  ├─ [通过] 关系深厚：三层打通，互动频繁，信息开放，客户主动
  ├─ [注意] 需加固：某一层薄弱，或互动频次下降，需主动加强
  └─ [不通过] 关系薄弱：单层关系或互动极少，需重新建立信任
```

**关键输出**：
```
关系深度评分：X/5
├─ 决策链覆盖：X/5（依据：覆盖层数/每层级人数）
├─ 关系触点频次：X/5（依据：互动次数 vs 行业平均）
├─ 关系质量：X/5（依据：客户主动度/信息开放度）
└─ 综合判断：[通过]/[注意]/[不通过]

关系地图：
├─ 董事长/CEO：XX（姓名）| 关系：[强]/[中]/[弱] | 上次互动：XX
├─ 董秘/财务总监：XX（姓名）| 关系：[强]/[中]/[弱] | 上次互动：XX
├─ 经办人：XX（姓名）| 关系：[强]/[中]/[弱] | 上次互动：XX
└─ 覆盖健康度：[三层打通]/[双层]/[单层] | 脆弱点：XX

加固建议：
- 紧急：XX（针对[不通过]项，如：需尽快接触到决策层）
- 重要：XX（针对[注意]项，如：增加非业务互动频次）
- 持续：XX（维持现有关系，如：定期信息分享）
```

### 维度4：信任程度

**本质目标**：客户到底信不信任我们？信任是关系的内核。是工具关系（用你就找你），还是伙伴关系（有事第一个想到你）？

**分析框架**：
```
【复购行为】
  ├─ 近3年同客户项目数：XX个
  ├─ 项目类型覆盖：定增/IPO/可转债/并购/债券（覆盖越多信任越深）
  ├─ 项目间隔周期：连续/间歇/孤立（连续合作代表持续信任）
  └─ 是否主动续约或扩大合作：是/否（客户主动找我们是信任的信号）

【推荐行为】
  ├─ 是否向其他企业推荐我们：有（XX次）/无
  ├─ 是否引荐同行或产业链伙伴：有（XX次）/无
  ├─ 是否邀请我们参与战略讨论：经常/偶尔/从不
  └─ 是否在公开场合提及或背书：有（XX次）/无

【关键决策参与度】
  ├─ 客户重大决策是否征询我们意见：经常/偶尔/从不
  ├─ 我们是否提前知晓客户战略动向：总是/有时/从不
  ├─ 客户是否分享未公开信息：高敏感/一般/公开信息
  └─ 我们在客户心中定位：战略顾问/服务商/供应商/备选

【信任危机历史】
  ├─ 近3年是否有信任危机事件：有/无（如有，次数、原因、修复结果）
  ├─ 客户是否曾经转向竞品：有/无（如有，原因、是否挽回）
  └─ 最近一次信任强化事件：XX（如：危机中我方表现、超预期交付）

【判断标准】
  ├─ [通过] 高度信任：复购+推荐+战略参与，定位战略顾问
  ├─ [注意] 工具信任：有复购但无推荐，定位服务商，需升级关系
  └─ [不通过] 低信任：孤立项目，无复购，定位供应商或备选
```

**关键输出**：
```
信任程度评分：X/5
├─ 复购行为：X/5（依据：项目数/覆盖类型/连续性）
├─ 推荐行为：X/5（依据：推荐次数/战略讨论参与度）
├─ 决策参与度：X/5（依据：提前知晓度/信息开放度）
└─ 综合判断：[通过]/[注意]/[不通过]

信任定位：
├─ 当前定位：[战略顾问]/[服务商]/[供应商]/[备选]
├─ 目标定位：[战略顾问]
└─ 差距：XX（如：缺推荐行为、缺战略参与）

升级路径：
- 从服务商到战略顾问：XX（如：主动提供行业洞察、参与客户战略研讨）
- 从供应商到服务商：XX（如：提升服务响应速度、增加主动沟通）
- 信任修复：XX（如：如有危机历史，需专项修复计划）
```

### 维度5：流失风险

**本质目标**：这个客户有没有可能跑掉？竞争对手在干什么？客户内部有没有变化？市场环境有没有不利因素？

**分析框架**：
```
【竞品威胁】
  ├─ 主要竞争对手近期是否接触该客户：是（XX机构，XX时间）/否
  ├─ 竞争对手近期是否有动作：
  │   ├─ 竞品服务价格：低于我方/持平/高于我方
  │   ├─ 竞品服务方案：有新意/常规/弱于我方
  │   └─ 竞品关系深度：比我方深/持平/比我方浅
  ├─ 客户是否曾提及或比较竞品：经常/偶尔/从未
  └─ 竞品威胁等级：高（客户明显倾向）/中（有接触但未表态）/低（无接触）

【客户内部变化】
  ├─ 关键决策人变动：
  │   ├─ 董事长/CEO是否变动：是/否（如有，新决策者背景/风格/与我方关系）
  │   ├─ 董秘/财务总监是否变动：是/否
  │   └─ 经办人是否变动：是/否（新人与我方关系）
  ├─ 公司战略方向变化：
  │   ├─ 是否有重大战略调整：是/否（如有，对我方业务的影响）
  │   ├─ 是否有新的合作伙伴：是/否（如有，是否替代我方）
  │   └─ 是否有并购或重组：是/否（如有，新主体与我方关系）
  └─ 内部变化影响等级：高（关键人变动+战略变化）/中（部分变化）/低（稳定）

【服务中断信号】
  ├─ 互动频次是否显著下降：是（下降XX%）/否
  ├─ 客户响应是否变冷淡：是（延迟XX天/敷衍回应）/否
  ├─ 是否有未解决的不满：是（XX事项）/否
  ├─ 是否有付款或结算拖延：是（拖延XX天）/否
  └─ 中断信号数量：XX个（0=安全，1-2=警惕，3+=危险）

【外部环境】
  ├─ 行业监管变化：有利/中性/不利（如：监管收紧可能影响业务需求）
  ├─ 市场融资环境：活跃/一般/冷淡（如：市场冷淡可能推迟融资计划）
  ├─ 客户行业景气度：上升/平稳/下行（下行可能压缩服务预算）
  └─ 外部环境影响等级：高/中/低

【判断标准】
  ├─ [通过] 安全：无竞品威胁，内部稳定，互动正常，环境有利
  ├─ [注意] 警惕：1-2个风险信号，需主动干预
  └─ [不通过] 危险：3个及以上风险信号，客户可能流失，需立即挽救
```

**关键输出**：
```
流失风险评分：X/5（注意：分数越低风险越高，此处反向计分）
├─ 竞品威胁：X/5（依据：竞品接触/方案/关系）
├─ 内部变化：X/5（依据：关键人变动/战略变化）
├─ 服务中断：X/5（依据：信号数量/严重程度）
├─ 外部环境：X/5（依据：监管/市场/行业）
└─ 综合判断：[通过]安全/[注意]警惕/[不通过]危险

风险雷达图：
竞品威胁：[低]/[中]/[高]
内部变化：[低]/[中]/[高]
服务中断：[低]/[中]/[高]
外部环境：[低]/[中]/[高]

预警行动：
- [危险] 立即挽救：XX（如：高层紧急拜访、专项服务方案、价格让步）
- [警惕] 主动干预：XX（如：增加互动频次、了解竞品动态、解决遗留问题）
- [安全] 维持加固：XX（如：定期关系维护、挖掘新需求、提升信任层级）
```

## 六、调用方式

### 6.1 客户经理调用

```
客户经理：
  输入：客户名称 + 上次项目信息
  调用：投行客户服务模型（5维度完整分析）
  输出：客户健康度报告 + 关键行动清单 + 需求清单
  
  使用场景：
  ├─ 项目完成后1个月内：重点评估服务后遗症，修复问题
  ├─ 季度复盘：全面评估5维度，更新客户状态
  ├─ 新需求发现：重点评估需求层次，准备跟进方案
  └─ 流失预警：重点评估流失风险，触发挽救动作
```

### 6.2 组织者调用（参考）

```
组织者：
  输入：客户名称 + 客户经理提交的客户服务模型输出
  参考：投行客户服务模型（维度3关系深度 + 维度4信任程度 + 维度5流失风险）
  输出：客户分层判断 + 资源调配建议
  
  使用场景：
  ├─ 客户分层评审：根据关系深度和信任程度判断客户层级
  ├─ 资源分配决策：高价值+高流失风险客户优先投入资源
  └─ 交叉销售统筹：根据需求层次判断跨业务线协作机会
```

### 6.3 产品支持调用（参考）

```
产品支持：
  输入：客户名称 + 客户经理提交的客户服务模型输出
  参考：投行客户服务模型（维度2需求层次）
  输出：产品匹配建议 + 方案准备
  
  使用场景：
  ├─ 需求响应：根据显性/隐性需求准备产品方案
  ├─ 交叉销售：判断客户是否需要其他业务线产品
  └─ 市场洞察：汇总多个客户的需求层次，判断市场趋势
```

## 七、同步响应机制

```
事件：客户经理完成客户服务模型分析
    ↓
【同步触发】
    ├─ 客户经理：看到完整5维度分析 → 制定跟进计划
    ├─ 组织者：看到关系深度+信任程度+流失风险 → 客户分层+资源调配
    ├─ 产品支持：看到需求层次 → 产品匹配+方案准备
    └─ （可选）其他业务线：看到需求层次 → 交叉销售机会判断
    ↓
【同步汇聚】
    ├─ 客户经理：关键行动清单（已获组织者确认资源）
    ├─ 组织者：资源调配方案（已匹配客户经理需求）
    └─ 产品支持：方案准备清单（已对齐客户经理时间表）
    ↓
【实时反馈】统一视图中所有角色看到各自任务
```

## 八、与定增业务分析模型的关系

```
┌─────────────────────────────────────────┐
│         【客户分析底座】                   │
│    （身份证 / 账户 / 关系网络 / 社会分层）  │
├─────────────────────────────────────────┤
│              ↓ 同步触发                  │
├─────────────────────────────────────────┤
│  【投行定增业务分析标准模型】               │
│  （项目能不能做？5维度：规模/定价/时机/     │
│   资金用途/合规风险）                      │
│              ↓ 项目完成后                  │
├─────────────────────────────────────────┤
│  【投行客户服务模型】                       │
│  （客户维系得怎么样？5维度：后遗症/需求/    │
│   关系/信任/流失风险）                      │
│              ↓ 持续循环                    │
│         发现新需求 → 触发定增/其他模型      │
└─────────────────────────────────────────┘
```

**关系说明**：
- 定增业务分析模型：回答"项目能不能做"（项目视角）
- 客户服务模型：回答"客户维系得怎么样"（关系视角）
- 两者共享客户分析底座，但服务于不同阶段的决策
- 客户服务模型发现的新需求，可再次触发定增业务分析模型或其他业务模型
