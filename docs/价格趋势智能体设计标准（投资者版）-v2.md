# 价格趋势智能体设计标准（投资者版）

> **版本**：V1.0
> **业态**：证券投资/财富管理
> **编制依据**：陶总业务模型方法论 + 智能体业务模型建设五步法
> **核心目标**：将价格趋势判断能力标准化、数字化、可调用化，让投资顾问能基于规则为投资者提供有依据的趋势参考
> **输出标准**：智能体能调用、员工能执行、合规能穿透、投资者能评审

---

## 1. 智能体定位

### 1.1 目标定义

**核心目标**
围绕投资者的价格趋势判断需求，构建可落地、可调用、可验证、可执行的趋势分析能力体系。不预测股价、不代客下单、不承诺收益。

**价值定位**
将专业量化分析中的"趋势判断逻辑"标准化、数字化，让投资顾问调用智能体后，能为投资者提供"有规则、可检查、可追溯"的趋势参考。

**输出标准**
智能体能调用、员工能执行、合规能穿透、投资者能评审。

### 1.2 角色定义（修正版）

| 角色类别 | 角色名称 | 定位 | 调用方式 |
|----------|----------|------|----------|
| 服务接收方 | 投资者（客户） | 趋势分析服务的接收者 | 被服务模型触达 |
| 服务交付方 | 投资顾问/客户经理（员工） | 执行动作的具体执行者 | 调用智能体执行标准判断 |
| 运营组织者 | 投资业务负责人/合规风控（老板） | 穿透监督的最终受益者 | 调用经营模型穿透数据 |
| **工具** | **价格趋势智能体** | **赋能员工的数字化分析工具** | **被员工调用，不独立对外** |

> **注意**：智能体不是独立角色，是赋能员工的工具。对外输出的所有建议必须经过员工审核，符合三句话法。

### 1.3 场景定义

**趋势咨询场景**
投资者询问某只股票或ETF"现在能不能买/卖"，投资顾问调用智能体获取多维度趋势判断参考，审核后对外输出。

**持仓诊断场景**
投资者已有持仓，投资顾问调用智能体判断当前趋势状态，提示是否需要调整。

**市场天气场景**
投资者询问"现在市场整体怎么样"，投资顾问调用智能体获取市场环境判断（热天/冷天/正常天）。

**风险预警场景**
投资者持仓标的触发趋势反转信号，智能体自动提醒投资顾问关注，由投资顾问决定是否通知客户。

## 2. 核心模型清单

### 2.1 产品模型（有什么）

#### 趋势分析产品
趋势分析产品 = {
  基础属性: [标的名称, 标的代码, 标的类型(股票/ETF/指数), 交易市场],
  分析维度: [市场环境, 整体大势, 标的趋势, 风险防护],
  输出形态: [文字报告, 信号评级, 图表辅助, 免责声明],
  状态属性: [数据时效, 信号有效期, 历史命中率, 系统版本]
}

#### 数据产品
数据产品 = {
  基础数据: [日K线(开高低收), 60分钟K线, 15分钟K线, 成交量],
  计算数据: [10日平均价格, 近期高低点, 价格偏离度, 历史波动率],
  外部数据: [沪深全A指数, 期权隐含波动率, 市场新闻政策(可选)],
  质量属性: [数据完整性, 数据时效性, 异常标记]
}

#### 报告产品
报告产品 = {
  标准报告: [市场天气摘要, 标的趋势摘要, 信号强度评级, 风险防护建议, 免责声明],
  扩展报告: [详细维度拆解, 历史信号回测, 同类标的对比, 操作建议],
  特殊报告: [极端行情预警, 假期降仓提醒, 黑天鹅事件提示],
  输出规则: [必须包含免责声明, 禁止出现预测股价, 禁止出现保本承诺]
}

### 2.2 客户模型（是谁）

#### 投资者画像模型
投资者画像 = {
  基础属性: [姓名, 联系方式, 风险承受能力等级, 投资经验年限],
  投资属性: [偏好标的类型, 持仓周期偏好, 资金规模区间, 收益预期],
  行为属性: [咨询频次, 决策周期, 反馈倾向, 投诉概率],
  适当性属性: [已开通权限, 可交易标的范围, 风险警示记录, 合规提醒历史]
}

#### 投资者分类模型
| 类型 | 特征 | 服务策略 |
|------|------|----------|
| 新手投资者 | 经验不足1年，高频咨询，依赖建议 | 多解释原理，少给具体建议，强调风险教育 |
| 成熟投资者 | 经验3年以上，自主判断为主 | 提供数据支撑，辅助验证其判断 |
| 退休投资者 | 风险承受能力低，追求稳健 | 强调风险防护，建议低仓位/空仓 |
| 激进投资者 | 风险承受能力高，追求高收益 | 标准流程执行，不特殊放宽 |
| 企业客户 | 机构账户，合规要求高 | 所有输出必须人工审核，留痕完整 |

#### 投资者价值模型
投资者价值 = {
  当前价值: [已产生佣金, 持仓规模, 活跃交易频次],
  潜在价值: [预计资产增长, 转介绍概率, 产品购买意向],
  服务成本: [咨询频次, 人工介入次数, 合规风险等级]
}

### 2.3 服务模型（标准）

#### 服务标准模型
服务标准 = {
  基础服务标准: {
    响应时效: [咨询消息15分钟内响应, 趋势报告3分钟内生成, 紧急预警5分钟内触达],
    输出规范: [必须包含免责声明, 必须标注数据时效, 禁止预测股价, 禁止承诺收益],
    审核标准: [含买卖建议→必须人工审核, 含仓位建议→必须人工审核, 纯数据展示→系统抽检]
  },
  个性化服务标准: {
    需求识别: [显性需求(直接问趋势), 隐性需求(持仓焦虑), 风险警示需求],
    服务边界: [可提供趋势分析, 不提供交易执行, 不提供个股推荐, 不提供收益承诺],
    升级路径: [标准回答→详细分析→人工介入→合规升级]
  }
}

#### 三句话法标准（核心合规标准）
服务标准 = {
  可以: {
    范围: [市场环境查询, 标的趋势方向, 历史数据展示, 投教内容推送],
    条件: [纯数据展示, 不涉买卖建议, 不对外替代人工判断],
    动作: [AI自动生成, 员工按需查阅, 系统抽检]
  },
  必须确认: {
    范围: [含买卖建议的输出, 含仓位建议的输出, 含止损建议的输出, 含产品推荐的输出],
    条件: [面向投资者, 涉及投资决策, 需留痕追责],
    动作: [AI生成→员工逐条审核→确认后对外]
  },
  不: {
    范围: [预测股价/目标价, 代客下单交易, 承诺保本保收益, 无资质给具体建议, 适当性不符仍给建议],
    条件: [违反监管红线, 超出资质边界, 触碰合规底线],
    动作: [AI直接拒绝, 标记原因, 升级人工处理]
  }
}

### 2.4 动作模型（谁做）

#### 核心动作：投资者趋势咨询响应

**事件**：投资者询问某标的趋势判断
**触发**：客户主动提问 或 系统监控到持仓标的趋势变化
**执行角色**：投资顾问/客户经理（员工调用智能体）

**标准流程：**

| 步骤 | 动作内容 | 执行角色 | 时限 | 输出标准 | 异常处理 |
|------|----------|----------|------|----------|----------|
| 1 | 接收投资者咨询 | 系统/员工 | 实时 | 记录咨询标的、投资者ID、咨询时间 | 投资者未开户→拒绝，引导开户 |
| 2 | 验证投资者适当性 | 智能体 | 30秒 | 匹配标的风险等级与投资者适当性 | 适当性不符→拒绝，提示风险等级 |
| 3 | 调取市场环境数据 | 智能体 | 30秒 | 波动状态、情绪状态、趋势方向 | 数据缺失→标记"数据不足" |
| 4 | 调取标的趋势数据 | 智能体 | 30秒 | 走势形态、高低位置、趋势强弱、时间一致性 | 停牌/退市→标记"不可交易" |
| 5 | 综合信号评估 | 智能体 | 30秒 | 信号强度评级（强/中/弱/无） | 多维度打架→标记"无信号" |
| 6 | 生成风险防护建议 | 智能体 | 30秒 | 仓位建议、止损参考、特殊提醒 | 极端行情→触发熔断建议 |
| 7 | 生成完整报告 | 智能体 | 30秒 | 标准报告格式，含所有维度+免责声明 | 格式异常→重新生成 |
| 8 | 人工审核 | 投资顾问 | 2分钟 | 确认逻辑自洽、数据有效、合规无误 | 发现错误→退回修改 |
| 9 | 对外输出 | 投资顾问 | 1分钟 | 通过合规渠道发送给投资者 | 投资者追问→记录并回应 |
| 10 | 记录归档 | 系统 | 实时 | 咨询记录、输出报告、审核记录 | 归档失败→告警 |

**总时限**：3-5分钟内完成（从投资者提问到收到回复）

**输出标准**：
趋势分析报告 = {
  市场环境: [大热天/正常天/大冷天],
  大势方向: [上升/下降/横盘],
  标的趋势: [上升/下降/横盘],
  高低位置: [接近顶部/半山腰/接近底部],
  趋势强弱: [强/中/弱/过头],
  时间一致性: [强/中等/弱/无信号],
  信号强度: [强信号/中等信号/弱信号/无信号],
  建议仓位: [满仓/半仓/轻仓/空仓],
  止损参考: [具体价格或区间],
  风险提示: [需要关注的风险点],
  免责声明: [本建议仅供参考，不构成投资建议，投资者需独立判断],
  数据时效: [基于X年X月X日X时数据]
}

## 3. 数据关联（数据清单）

### 3.1 数据需求矩阵

| 模型 | 数据需求 | 数据来源 | 获取方式 | 更新频率 | 数据质量要求 |
|------|----------|----------|----------|----------|--------------|
| 产品模型-趋势分析 | 标的行情数据 | 行情系统 | API接口 | 实时 | 延迟<3秒 |
| 产品模型-趋势分析 | 指数行情数据 | 行情系统 | API接口 | 实时 | 延迟<3秒 |
| 产品模型-趋势分析 | 期权波动率数据 | 期权系统 | API接口 | 实时 | 完整度>99% |
| 客户模型-投资者画像 | 客户适当性等级 | CRM系统 | 数据库查询 | 日级 | 准确率100% |
| 客户模型-投资者画像 | 客户持仓记录 | 账户系统 | 数据库查询 | 实时 | 权限审批后 |
| 服务模型-三句话法 | 合规规则库 | 合规系统 | 规则引擎 | 实时 | 与监管同步 |
| 动作模型-咨询响应 | 咨询记录 | 客服系统 | 数据库写入 | 实时 | 不可丢失 |
| 动作模型-审核记录 | 审核日志 | 合规系统 | 数据库写入 | 实时 | 不可篡改 |

### 3.2 数据缺口说明

| 缺口 | 影响 | 解决方案 | 优先级 |
|------|------|----------|--------|
| 非结构化数据（新闻政策） | LLM情绪分析缺失 | 接入新闻API、政策公告爬取 | P1 |
| 客户实时持仓 | 无法做持仓诊断 | 与账户系统打通，权限审批后接入 | P1 |
| 历史回测数据库 | 无法验证信号有效性 | 建立3年以上历史数据库 | P2 |
| 智能体自我进化数据 | 无法自动调优参数 | 记录每次信号结果，建立反馈闭环 | P2 |

### 4.1 标杆对象

| 标杆 | 做法 | 可借鉴点 | 我们的差距 |
|------|------|----------|------------|
| **华泰AI涨乐** | 券业首款AI原生金融终端，AI盯盘+AI选股+3-4步极简体验 | 多维度趋势判断+极简交互+合规嵌入 | 我们没有APP端，输出依赖员工传递 |

## 5. 检查清单

### 5.1 模型设计检查（陶总五大反对）

| 检查项 | 是否通过 | 说明 |
|--------|----------|------|
| 不讲意义、讲模型本身 | ✅ | 四个模型定义清晰，不空洞 |
| 不流程化、线性化 | ⚠️ | 动作模型有10步流程，但每一步有标准、有输出、有异常，不是简单流程堆砌 |
| 不角色混乱 | ✅ | 角色定义修正：智能体是工具，不是角色；客户/员工/老板边界清晰 |
| 不主观拍脑袋 | ✅ | 每个维度有具体指标、有判断标准、可追溯 |
| 不技术自说自话 | ✅ | 从投资者和员工视角出发，不堆砌技术名词 |

### 5.2 输出质量检查清单

每份趋势分析报告输出前，必须检查：

- [ ] 市场环境三项是否齐全（波动状态、情绪状态、趋势方向）
- [ ] 标的趋势四项是否齐全（走势形态、高低位置、趋势强弱、时间一致性）
- [ ] 风险防护两项是否齐全（仓位建议、止损参考）
- [ ] 信号强度是否明确（强/中/弱/无）
- [ ] 免责声明是否包含（标准话术）
- [ ] 数据时效是否标注（基于X年X月X日X时数据）
- [ ] 是否触发"必须确认"事项（如有买卖建议→是否已人工审核）
- [ ] 是否触发"不"事项（如预测股价→是否已拒绝）
- [ ] 投资者适当性是否已验证（标的风险等级是否匹配）
- [ ] 异常场景是否有处理（数据缺失/极端波动/停牌/适当性不符）
- [ ] 审核记录是否完整（审核人、审核时间、审核结论）
- [ ] 归档记录是否完整（咨询记录、输出报告、审核记录）

### 5.3 合规检查清单（三句话法）

- [ ] 纯数据展示类输出 → 系统抽检，员工按需查阅
- [ ] 含买卖建议的输出 → 员工逐条审核，确认后对外
- [ ] 含仓位建议的输出 → 员工逐条审核，确认后对外
- [ ] 含止损建议的输出 → 员工逐条审核，确认后对外
- [ ] 预测股价/目标价 → AI直接拒绝，输出标准话术
- [ ] 代客下单请求 → AI直接拒绝，输出标准话术
- [ ] 保本保收益承诺 → AI直接拒绝，输出标准话术
- [ ] 适当性不符仍要求建议 → AI直接拒绝，输出标准话术

## 6. 迭代与监督

### 6.1 监控指标

| 指标 | 目标 | 预警阈值 | 责任人 |
|------|------|----------|--------|
| 信号命中率 | 趋势判断与实际走势一致率 | 低于60%触发调优 | 投资业务负责人 |
| 合规命中率 | 输出符合三句话法比例 | 低于99%触发整改 | 合规风控 |
| 人工介入率 | 需人工审核的比例 | 高于50%说明AI能力不足 | 投资业务负责人 |
| 投资者满意度 | 调研评分 | 低于4分/5分触发优化 | 投资业务负责人 |
| 响应时效 | 从提问到输出的时间 | 超过5分钟触发优化 | 技术负责人 |
| 投诉率 | 因趋势建议引发的投诉 | 高于1%触发整改 | 合规风控 |

### 6.2 迭代触发条件

- 信号命中率连续2周低于60% → 触发模型参数重调
- 同一异常场景出现超过10次/周 → 触发异常处理流程优化
- 人工介入率高于50% → 触发AI能力升级评估
- 季度标杆跟踪更新 → 对比行业最新做法，调整模型设计
- 监管政策变化 → 触发三句话法规则更新
