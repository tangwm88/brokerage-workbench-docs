### 1.2 舆情分析AI提示词

#### 1.2.1 情感分析提示词

```
你是一位专业的舆情情感分析师。请对以下内容进行情感分析：

【待分析内容】
{{content}}

【分析维度】
1. 整体情感倾向：正面/中性/负面
2. 情感强度评分：1-10分（1=极度负面，10=极度正面，5=中性）
3. 情绪标签：愤怒/失望/满意/惊喜/担忧/其他
4. 涉及主题：产品/服务/价格/售后/其他
5. 关键短句：提取最具代表性的1-3句话

【输出格式】
{
  "sentiment": "正面/中性/负面",
  "score": 7,
  "emotion_tags": ["满意"],
  "topics": ["产品"],
  "key_quotes": ["..."],
  "summary": "一句话总结"
}
```

#### 1.2.2 风险等级判定提示词

```
你是一位舆情风险预警专家。请根据以下信息判定风险等级：

【舆情信息】
来源：{{source}}
内容：{{content}}
传播量：{{spread_count}}
互动量：{{interaction_count}}
时间：{{time}}

【判定标准】
- 红色预警：涉及安全/合规/重大负面，可能引发公关危机
- 橙色预警：影响范围较大，负面情绪强烈，需要关注
- 黄色预警：存在一定负面倾向，建议监测跟进
- 蓝色提示：轻微负面或无风险，常规关注

【输出要求】
1. 风险等级：红/橙/黄/蓝
2. 风险类型：产品质量/服务态度/价格争议/安全事故/虚假信息/其他
3. 影响评估：高/中/低
4. 处理建议：一句话建议
5. 预警理由：基于内容的分析依据

【输出格式】
JSON格式返回
```

### 1.3 回访话术模板

#### 1.3.1 30天回访话术（新客户 onboarding）

**电话回访开场**
> "您好，请问是【客户姓名】先生/女士吗？我是【公司名】客户成功部的【姓名】。您于【开通日期】开通了我们【产品名称】服务，今天是您使用的第30天，想占用您2-3分钟时间做个简单的使用回访，您现在方便吗？"

**核心问题清单**

| 序号 | 问题 | 目的 | 话术 |
|-----|------|------|------|
| 1 | 使用满意度 | 了解整体感受 | "这一个月使用下来，您对我们产品整体满意吗？如果10分满分，您会打几分？" |
| 2 | 功能使用 | 确认核心功能使用 | "目前您主要使用了哪些功能？有没有哪个功能是您觉得特别好用的？" |
| 3 | 遇到问题 | 发现并解决障碍 | "在使用过程中有没有遇到什么问题或困难？我们的帮助文档和教程是否清晰？" |
| 4 | 价值感知 | 评估产品价值 | "和之前的方式相比，您觉得我们的产品为您带来了哪些帮助或改变？" |
| 5 | 需求挖掘 | 探索深度需求 | "接下来您希望能在哪些方面得到更多支持？有没有我们还没满足的需求？" |
| 6 | 推荐意愿 | NPS评估 | "如果让您向同行朋友推荐我们的产品，您愿意吗？0-10分会打几分？" |
| 7 | 意见建议 | 收集改进建议 | "最后，您对我们有什么建议或期望吗？" |

**结束话术**
> "非常感谢您抽出宝贵时间！您的建议我们会认真记录。后续有任何问题，您可以随时联系您的专属客服【姓名/联系方式】。祝您使用愉快，再见！"

**特殊情况应对**

| 场景 | 应对话术 |
|------|---------|
| 客户说很忙 | "非常抱歉打扰您！那您看明天上午10点或下午3点，哪个时间更方便？我届时再联系您。" |
| 客户表示不满 | "非常抱歉让您有不好的体验！能否具体说说遇到了什么问题？我会立即记录并协调相关人员为您解决。" |
| 客户要退订 | "理解您的决定。能否告诉我主要原因？我们会努力改进。同时，如果您是担心[具体问题]，我们可以[解决方案]，您看是否愿意再考虑一下？" |
| 客户提出新需求 | "感谢您的建议！这个功能我们正在规划中，预计【时间】上线。如果急需，我可以为您申请开通内测资格，您看需要吗？" |

#### 1.3.2 60天回访话术（中期健康度检查）

**开场白**
> "您好，【客户姓名】先生/女士，我是【公司名】的【姓名】。您使用我们服务已经两个月了，今天想了解一下您的深度使用情况和更多需求。"

**核心问题清单**

| 序号 | 问题 | 目的 |
|-----|------|------|
| 1 | 深度使用情况 | "除了基础功能，您是否尝试过【高级功能1】、【高级功能2】？使用体验如何？" |
| 2 | 业务效果 | "目前我们的服务对您业务的帮助有多大？有没有具体的数据可以分享？" |
| 3 | 竞品对比 | "与我们类似的产品中，您是否也了解过其他家？相比之下我们的优劣势是？" |
| 4 | 续约意向 | "您的服务将于【到期日】到期，目前是否有续约计划？对续约方案有什么期待？" |
| 5 | 增值服务 | "我们近期推出了【新功能/服务】，您有兴趣了解一下吗？" |
| 6 | 案例授权 | "如果您的使用效果比较好，我们是否可以邀请您成为案例客户？会有相应的权益回馈。" |

**重点话术：续约引导**
> "您使用的效果这么好，建议您考虑升级至【更高版本】，目前老用户续约可享受【具体优惠】。我可以为您申请一个专属方案，您看需要吗？"

#### 1.3.3 90天回访话术（续约/增购转化）

**开场白**
> "您好，【客户姓名】先生/女士！您是我们非常重要的客户，已经相伴走过三个月了。今天致电主要是想和您聊聊续约事宜，同时也听听您对我们服务的长期建议。"

**核心目标**
- 确保续约
- 推动增购/升级
- 建立长期合作关系

**核心问题清单**

| 序号 | 问题 | 话术 |
|-----|------|------|
| 1 | ROI确认 | "这三个月使用下来，您觉得投入产出比如何？是否达到了最初的预期？" |
| 2 | 续约确认 | "您的服务将于【到期日】到期，我们为您准备了续约方案，您倾向于按年还是按月？" |
| 3 | 升级推荐 | "根据您的使用量，建议您升级到【高级版本】，可以享受【具体权益】，性价比更高。" |
| 4 | 长期规划 | "未来半年到一年，您的业务发展有什么规划？我们可以提前做好服务准备。" |
| 5 | 转介绍 | "如果您身边有朋友也有类似需求，欢迎推荐，推荐成功双方都有好礼相赠。" |

**异议处理**

| 异议 | 应对话术 |
|------|---------|
| "价格太贵了" | "理解您的考虑。其实按使用效果来算，每天成本不到X元。而且我可以为您申请【具体折扣/赠送时长】，这样算下来性价比很高。" |
| "想再看看" | "没问题，您还有考虑的时间。不过为了不影响您的正常使用，建议最晚在【日期】前确认。这几天我可以先把详细方案发给您参考。" |
| "想换其他家" | "理解您想多比较。能否告诉我您最看重哪些方面？我可以针对性地介绍我们的优势。另外，如果您试用其他产品期间，我们也可以保持服务，给您更多选择空间。" |
| "预算不够" | "明白，现在确实很多公司都在控制成本。我们也有针对预算有限客户的轻量方案，或者可以分期付款，您觉得哪种更合适？" |

### 1.4 投诉处理话术

#### 1.4.1 投诉受理话术

**开场安抚**
> "您好，我是【姓名/工号】，专门负责投诉处理。非常感谢您选择向我们反馈问题，您的意见对我们非常重要。请您先消消气，慢慢说，我会认真记录并尽力帮您解决。"

**信息收集话术**
> "为了更好地帮您解决问题，我需要了解以下信息：
> 1. 请问事情是什么时候发生的？
> 2. 涉及的具体产品/服务是什么？
> 3. 目前对您造成了什么影响？
> 4. 您希望怎样解决这个问题？"

**确认复述**
> "我来确认一下您的诉求：【复述客户问题】。我理解的对吗？"

**工单告知**
> "您的投诉我已经详细记录，工单号是【XXXX】，您可以凭此号码查询处理进度。我们将在【时间】内给您第一次回复。"

---

#### 1.4.2 处理中沟通话术

**进展反馈**
> "您好，【客户姓名】先生/女士！关于您投诉的【问题简述】（工单号：XXXX），目前处理进展如下：
> - 当前状态：【状态】
> - 处理措施：【措施】
> - 预计完成时间：【时间】
> 如有新进展我会第一时间通知您。"

**需要补充材料**
> "为了更好地处理您的问题，还需要您提供【材料清单】，您可以通过【方式】发送给我，收到后我们会加速处理。"

**方案沟通**
> "关于您的投诉，我们拟定的处理方案是：【方案内容】。这个方案您看可以接受吗？如有其他想法请告诉我。"

---

#### 1.4.3 投诉结案话术

**结果告知**
> "您好，【客户姓名】先生/女士！您反馈的【问题】已经处理完毕：
> - 处理结果：【结果】
> - 补偿措施：【如有】
> - 预防措施：【避免再次发生】"

**满意度确认**
> "请问您对我们的处理结果满意吗？如果10分满分，您会打几分？"

**结束语**
> "再次为给您带来的不便道歉。如果后续还有任何问题，您可以直接联系我【联系方式】。感谢您的理解与支持！"

#### 1.4.4 不同类型投诉应对话术

**产品质量投诉**
> "非常抱歉产品质量没有达到您的期望！我们对此深表歉意。目前我可以为您【换货/退款/补偿】，同时会将此问题反馈给产品部门改进。您希望如何处理？"

**服务态度投诉**
> "非常抱歉我们的服务让您感到不愉快！这确实是我们培训的不足。我已将此事上报主管，会对相关人员进行处理。同时我会亲自跟进您的需求，确保后续体验。"

** billing争议**
> "我完全理解您对费用的疑虑。让我为您详细核对一下账单...【核对过程】。如果是我们的失误，会立即退还差额；如果是理解偏差，我会为您解释清楚。"

```
【工单创建】

    ↓

├─ 重复投诉 → 原处理人/主管
├─ 高风险客户 → 主管/经理
└─ 普通客户 → 标准队列
```

#### 3.2.2 工单流转规则表

| 场景 | 流转规则 | 处理时限 | 升级条件 |
|-----|---------|---------|---------|
| 一线解决 | 一线客服直接处理并关闭 | ≤24h | - |
| 内部转交 | 转交至相关部门处理 | 接收方+24h | 超时自动升级 |
| 跨部门协作 | 发起协作，主责人跟进 | 协商确定 | 超时自动升级 |
| 升级主管 | 一线→客服主管 | 主管+12h | 客户要求/超时/复杂 |
| 升级经理 | 主管→客服经理 | 经理+8h | 重大投诉/高风险 |
| 升级总监 | 经理→客服总监 | 总监+4h | 危机事件/监管介入 |
| 外部协同 | 需产品/技术/法务介入 | 依具体事项 | 内部无法解决 |
| 客户要求升级 | 客户明确要求升级 | 立即 | - |

#### 3.2.3 工单状态流转

```
[新建] → [待处理] → [处理中] → [待确认] → [已解决] → [已关闭]
              ↓           ↓           ↓
           [挂起]     [升级]     [转交]
              ↓           ↓           ↓
           [恢复]     [处理中]   [处理中]

特殊状态：
- [挂起]：等待客户回复/外部依赖
- [升级]：已触发升级流程
- [转交]：转至其他部门/人员
- [待确认]：等待客户确认解决
- [已解决]：问题已解决待关闭
- [已关闭]：工单完结
```

#### 3.2.4 流转触发条件

| 触发条件 | 动作 | 通知对象 |
|---------|------|---------|
| 超时未响应 | 自动提醒 | 当前处理人+主管 |
| 超时未解决 | 自动升级 | 主管 |
| 客户催单 | 优先级提升+提醒 | 处理人+主管 |
| 客户不满 | 自动升级 | 主管+经理 |
| 重复提交 | 关联原工单 | 原处理人 |
| 关键词命中 | 标签+路由 | 相关团队 |
| 高风险客户 | 优先+专人 | 高级客服 |

#### 3.3.2 自动判定规则

**红色预警触发条件（满足任一）**
- [ ] 涉及人身安全/安全事故
- [ ] 涉及违法违规（涉嫌诈骗、传销等）
- [ ] 媒体/大V（粉丝>100万）发布负面
- [ ] 监管部门介入或点名
- [ ] 24小时内传播量>10万且负面率>70%
- [ ] 同一问题集中爆发（1小时内>100条）
- [ ] 涉及政治敏感/民族宗教

**橙色预警触发条件（满足任一）**
- [ ] 中V发布（粉丝10万-100万）负面
- [ ] 24小时内传播量>1万且负面率>50%
- [ ] 涉及产品严重缺陷/批量故障
- [ ] 被主流媒体报道（非负面专题）
- [ ] 涉及收费争议且金额较大
- [ ] 同一问题持续发酵（24小时增长>500%）

**黄色预警触发条件（满足任一）**
- [ ] 普通用户负面，有一定传播
- [ ] 24小时内传播量>1000且负面率>30%
- [ ] 涉及服务态度/体验问题
- [ ] 产品功能吐槽/建议类
- [ ] 价格敏感讨论
- [ ] 竞品对比类（我方负面）

**蓝色提示**
- 不满足以上条件的其他舆情
- 正面或中性内容
- 传播量较小的讨论

#### 3.3.3 风险类型标签

| 类型 | 说明 | 常见表现 |
|-----|------|---------|
| 产品质量 | 功能/性能/稳定性问题 | bug频发、闪退、卡顿 |
| 服务态度 | 客服/售后体验差 | 态度恶劣、推诿扯皮 |
| 价格争议 | 定价/收费问题 | 太贵、乱收费、隐形消费 |
| 安全事故 | 安全/隐私问题 | 数据泄露、账号被盗 |
| 虚假宣传 | 承诺未兑现 | 广告不实、功能缺失 |
| 竞品攻击 | 竞品相关负面 | 被对比、恶意抹黑 |
| 虚假信息 | 谣言/误传 | 不实信息传播 |
| 其他 | 其他类型 | - |

#### 3.3.4 舆情处置流程

```
【舆情监测】
    ↓
【AI识别分析】→ 自动分类+定级
    ↓
├─ 🔴 红色 → 立即预警 → 危机小组 → 15分钟内响应
├─ 🟠 橙色 → 即时通知 → 舆情专员 → 1小时内响应
├─ 🟡 黄色 → 日常监测 → 舆情专员 → 4小时内跟进
└─ 🔵 蓝色 → 记录归档 → 定期汇总 → 24小时内关注
    ↓
【处置评估】
    ↓
├─ 需回复 → 撰写回应 → 审核发布
├─ 需调查 → 内部核实 → 后续回应
├─ 需处理 → 转相关部门 → 跟踪闭环
└─ 无需处理 → 持续监测 → 记录归档
    ↓
【效果跟踪】→ 传播趋势 →  sentiment变化 → 报告输出
```

### 3.4 回访触发规则

#### 3.4.1 常规回访触发

| 回访类型 | 触发条件 | 触发时间 | 执行人 | 优先级 |
|---------|---------|---------|-------|-------|
| 新客30天回访 | 新用户开通满30天 | T+30天 | 客户成功 | 高 |
| 中期60天回访 | 新用户开通满60天 | T+60天 | 客户成功 | 中 |
| 到期90天回访 | 服务到期前30天 | 到期日-30天 | 客户成功/销售 | 高 |
| 年度回访 | 合作满一年客户 | 签约周年日 | 客户成功经理 | 中 |
| 续约回访 | 合同到期前 | 到期日-60天 | 销售/CSM | 高 |

#### 3.4.2 特殊回访触发

| 回访类型 | 触发条件 | 触发时间 | 执行人 | 目的 |
|---------|---------|---------|-------|------|
| 投诉后回访 | 投诉结案后 | 结案后7天内 | 客服主管 | 确认满意度 |
| 服务故障回访 | 重大故障影响客户 | 故障解决后3天 | 技术支持 | 关怀+评估影响 |
| 升级/降级回访 | 客户升级或降级套餐 | 变更后7天 | 客户成功 | 确认适应情况 |
| 长期沉默回访 | 30天未登录/使用 | 第30天 | 客户成功 | 唤醒+流失预警 |
| 高价值客户回访 | 月消费TOP10% | 每月1次 | CSM | 维护关系 |
| 续费预警回访 | 续费可能性低 | 识别后3天内 | 销售/CSM | 挽回 |
| 产品更新回访 | 新功能上线后 | 功能上线后7天 | 产品/CSM | 收集反馈 |

#### 3.4.3 回访优先级规则

```
优先级计算：
基础分 + 客户价值分 + 风险分 + 时机分 = 总分

基础分：
- 新客回访：100分
- 到期回访：100分
- 常规回访：50分
- 关怀回访：30分

客户价值分：
- VIP客户：+50分
- 企业客户：+30分
- 高价值客户：+20分
- 普通客户：+0分

风险分：
- 投诉后：+40分
- 流失预警：+30分
- 服务故障：+20分
- 正常：+0分

时机分：
- 紧急（24h内）：+30分
- 重要（7天内）：+15分
- 一般（30天内）：+0分
```

#### 3.4.4 回访执行规则

| 规则项 | 具体规定 |
|-------|---------|
| 回访时间 | 工作日上午9:30-11:30，下午14:00-17:00 |
| 回访次数 | 同一客户同一类型回访，最多尝试3次 |
| 回访间隔 | 两次尝试间隔≥24小时 |
| 拒访处理 | 客户明确拒绝后，6个月内不再主动回访 |
| 改约处理 | 客户要求改约，需在客户指定时间回访 |
| 记录要求 | 每次回访必须填写回访记录 |
| 满意度 | 必须询问满意度评分和NPS |
| 升级处理 | 回访中发现问题，需创建跟进工单 |
| 数据归档 | 回访记录需在24小时内录入系统 |

#### 3.4.5 回访结果处理

| 回访结果 | 处理规则 |
|---------|---------|
| 满意+续约意向高 | 记录归档，到期前提醒续约 |
| 满意+有增购意向 | 转销售跟进，创建商机 |
| 一般+有顾虑 | 分析原因，制定提升计划 |
| 不满意 | 立即创建跟进工单，主管介入 |
| 明确不续约 | 启动挽回流程，分析流失原因 |
| 未接通 | 24小时后再次尝试，最多3次 |
| 拒访 | 标记拒访，6个月内不再回访 |
