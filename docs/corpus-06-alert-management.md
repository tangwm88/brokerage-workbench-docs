# 提示事项模块 - 参考语料与使用规则

> **模块定位**：企业内部事项请示、问题反馈与资源协调的统一入口
> **适用场景**：战略客户政策、资源投入、合规风险、系统问题、业务流程优化、算力/人力/预算申请

---

## 三、使用规则

### 3.1 审批流程规则

#### 重大事项请示流程

```
【战略客户政策类】
提交 → 部门负责人 → 战略部会签 → 分管VP → CEO → 完成

【资源投入类】
提交 → 部门负责人 → 财务部会签 → 分管VP → (金额>100万)→CEO → 完成

【合规风险类】
提交 → 部门负责人 → 法务部会签 → 合规部会签 → 分管VP → CEO → 完成
```

#### 问题反馈处理流程

```
【系统问题】
提交 → 自动分类 → 技术团队受理 → 问题诊断 → 修复/排期 → 验证 → 关闭

【业务流程】
提交 → 运营团队受理 → 流程评估 → 优化/驳回 → 实施 → 验证 → 关闭

【建议收集】
提交 → 产品团队受理 → 价值评估 → 转需求/存档 → 反馈结果 → 关闭
```

#### 资源协调申请流程

```
【算力资源】
提交 → 直属Leader → 技术中台评估 → 分管VP（资源>50卡）→ 资源分配 → 交付

【人力资源】
提交 → 直属Leader → HRBP评估 → 分管VP（HC外新增）→ 招聘/调配 → 到岗

【预算申请】
提交 → 部门负责人 → 财务初审 → 分管VP（>10万）→ 财务总监（>50万）→ CEO（>100万）→ 拨付
```

### 3.2 审批权限规则

#### 金额分级审批权限表

| 审批事项 | ≤1万 | 1-10万 | 10-50万 | 50-100万 | >100万 |
|----------|------|--------|---------|----------|--------|
| 预算申请 | 部门负责人 | 部门负责人 | 分管VP | 财务总监 | CEO |
| 算力资源 | 技术中台负责人 | 技术中台负责人 | 分管VP | CTO | CEO |
| 人力外包 | HRBP | 分管VP | 财务总监 | CEO | 董事会 |

#### 角色审批权限定义

| 角色 | 审批权限范围 | 特殊说明 |
|------|--------------|----------|
| 部门负责人 | 本部门≤10万预算、常规算力、常规人力 | 可转交不可驳回至提交人 |
| 分管VP | 分管领域≤100万预算、大规模资源 | 可跨部门协调 |
| 财务总监 | 全公司预算审批、财务报表签批 | 资金调拨权限 |
| CEO | 全公司所有事项 | 战略决策权限 |
| 技术中台 | 算力资源评估与分配 | 技术可行性审核 |
| 法务/合规 | 合规风险相关 | 拥有一票否决权 |

### 3.3 审批时限规则

#### 标准响应时限

| 紧急程度 | 首次响应 | 审批处理 | 整体完成 |
|----------|----------|----------|----------|
| 普通 | 4工作小时 | 2工作日 | 5工作日 |
| 紧急 | 2工作小时 | 1工作日 | 3工作日 |
| 特急 | 1工作小时 | 4工作小时 | 1工作日 |

#### 各节点时限

| 节点类型 | 普通时限 | 紧急时限 | 超时处理 |
|----------|----------|----------|----------|
| 部门负责人审批 | 8小时 | 4小时 | 自动升级至分管VP |
| 财务/法务会签 | 1工作日 | 4小时 | 发送催办通知 |
| VP审批 | 1工作日 | 8小时 | 自动升级至CEO |
| CEO审批 | 2工作日 | 1工作日 | 专人跟进提醒 |
| 资源评估 | 2工作日 | 1工作日 | 自动抄送资源部门总监 |

#### SLA预警阈值

```yaml
预警级别:
  黄色预警: 剩余时间 ≤ 50% 时限
  橙色预警: 剩余时间 ≤ 25% 时限
  红色预警: 已超时

通知方式:
  黄色: 系统消息 + 邮件
  橙色: 系统消息 + 邮件 + 企业微信
  红色: 系统消息 + 邮件 + 企业微信 + 短信（重要事项）
```

### 3.4 升级规则

#### 自动升级规则

| 触发条件 | 升级动作 | 通知对象 |
|----------|----------|----------|
| 节点超时未处理 | 自动升级至上一级领导 | 原审批人、升级对象、提交人 |
| 连续2次被驳回 | 升级至双方共同上级协调 | 双方Leader、HRBP |
| 涉及跨部门争议 | 升级至相关VP协调会 | 相关部门VP |
| 合规风险一票否决 | 升级至CEO + 法务委员会 | CEO、法务总监、合规总监 |
| 预算超支预警 | 升级至财务总监 + 分管VP | 财务、业务负责人 |

#### 手动升级规则

```
【提交人主动升级】
- 条件：等待时间超过50% SLA 且 无响应
- 操作：点击"催办升级"按钮
- 结果：抄送审批人上级，标记加急

【审批人主动升级】
- 条件：超出本角色审批权限
- 操作：点击"升级审批"按钮
- 结果：流转至有权限的上级审批

【系统异常升级】
- 条件：流程节点配置缺失/审批人离职
- 操作：自动流转至流程管理员
- 结果：人工介入处理，修复流程配置
```

#### 升级后处理机制

| 升级类型 | 处理时限 | 处理要求 |
|----------|----------|----------|
| 超时升级 | 4工作小时 | 升级对象必须响应，确认或转交 |
| 争议升级 | 1工作日 | 召集协调会议，给出裁决意见 |
| 权限升级 | 即时 | 系统自动流转至正确审批人 |
| 异常升级 | 2工作小时 | 流程管理员介入修复 |

| request_subtype | 战略客户政策、资源投入、合规风险、系统问题、业务流程、建议收集、算力资源、人力资源、预算申请 |
| priority | 普通、紧急、特急 |
| approval_action | 通过、驳回、有条件通过、转交、加签 |
| current_status | 待提交、审批中、已通过、已驳回、处理中、已关闭 |
| sla_status | 正常、预警、超时 |
