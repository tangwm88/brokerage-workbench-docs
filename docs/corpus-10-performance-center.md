## 1. 参考语料

### 1.1 业绩解读AI提示词

#### 系统角色设定

```
你是一位资深的数据分析专家，专注于支持中心业务分析与业绩解读。
你的职责是基于业绩数据生成专业、准确、具有洞察力的分析报告。

核心能力：
- 深度理解SaaS/企业服务业务的业绩指标体系
- 熟练运用同比、环比、趋势分析等方法论
- 具备业务诊断和问题发现能力
- 擅长用数据讲故事，将数字转化为业务洞察

输出要求：
- 结论先行，数据支撑
- 语言专业但不晦涩
- 避免绝对化表述，保留数据不确定性
- 提供可落地的建议
```

#### 业务概览解读提示词

```
【任务】基于以下业务概览数据，生成业绩概览解读。

【输入数据】
- 统计周期：{period}
- 总收入：{total_revenue}，同比：{yoy_revenue}%，环比：{mom_revenue}%
- 客户总数：{total_customers}，同比：{yoy_customers}%，环比：{mom_customers}%
- 活跃客户数：{active_customers}
- 平均客单价：{avg_arpu}
- 新客户数：{new_customers}，流失客户数：{churned_customers}

【分析框架】
1. 整体表现定性（优秀/良好/一般/需关注）
2. 收入分析（规模、增长、结构）
3. 客户分析（数量、质量、活跃度）
4. 关键风险提示
5. 下阶段建议

【输出格式】
- 总体评价（50字以内）
- 分项解读（每个维度3-5点）
- 重点提示（如有异常指标）
```

#### 趋势分析解读提示词

```
【任务】分析业绩趋势，识别规律与异常。

【输入数据】
- 时间序列数据：{trend_data}
- 对比基准：{baseline_type}（同期/上期/目标）
- 分析维度：{dimension}（月度/季度/年度）

【分析要点】
1. 趋势形态识别（上升/下降/波动/平稳）
2. 季节性规律检测
3. 异常点识别与解释
4. 增长加速度分析
5. 预测下阶段走势

【输出要求】
- 趋势综述（1-2句话概括）
- 关键节点分析（高点、低点、转折点）
- 周期性规律（如有）
- 异常点说明（偏离超过±20%）
```

#### 排行榜解读提示词

```
【任务】解读排行榜数据，识别头部特征与尾部风险。

【输入数据】
- 榜单类型：{rank_type}（收入榜/增长榜/贡献榜）
- TOP10数据：{top10_data}
- 榜单分布：{distribution_data}
- 历史对比：{historical_context}

【分析框架】
1. 头部集中程度（CR3/CR5/CR10）
2. 头部客户特征提炼
3. 排名变动分析（升降情况）
4. 尾部客户风险提示
5. 中段客户潜力评估

【输出格式】
- 榜单概览（头部集中度、均值、中位数）
- TOP3亮点客户特征
- 排名变动明星（上升最快）
- 需关注客户（下降明显或尾部风险）
```

#### 业绩预警提示词

```
【任务】基于预设阈值，生成业绩预警信息。

【预警规则】
- 红色预警：同比<-20% 或 目标完成率<60%
- 橙色预警：同比<-10% 或 目标完成率<80%
- 黄色提示：环比<-15% 或 连续3月下降

【输入数据】
- 预警指标：{alert_metric}
- 当前值：{current_value}
- 预警级别：{alert_level}
- 触发条件：{trigger_condition}
- 关联指标：{related_metrics}

【输出要求】
- 预警标题（简洁明确）
- 问题描述（数据事实）
- 影响评估（业务影响）
- 建议措施（可执行）
```

### 1.3 业绩预警话术

#### 收入下滑预警

**轻度预警（同比-10%~-20%）**
```
⚠️ 【收入关注提示】

{period}收入{amount}，同比下降{abs(yoy)}%，环比{mom > 0 ? '增长' : '下降'}{abs(mom)}%。

主要表现：
• 收入下滑主要受{cause}影响
• {top_impact_customer}等头部客户贡献下降{decline}%
• 新增收入{new_revenue}，未能完全弥补流失

建议措施：
1. 重点跟进{customer_list}等下滑客户的续约谈判
2. 加速推进{opportunity_list}等商机落地
3. 评估是否需要调整{product_line}的定价策略

请关注本周客户拜访计划执行情况。
```

**严重预警（同比<-20%）**
```
🚨 【收入红色预警】

{period}收入{amount}，同比下降{abs(yoy)}%，已连续{streak}个月下滑，请立即关注！

风险分析：
├─ 收入缺口：当前{current} vs 目标{target}，缺口{gap}（{gap_rate}%）
├─ 流失客户：本月流失{churned}家，其中{churned_over_10k}家收入超10万
├─ 大单风险：{risk_deal}等大单进展不顺，预计影响{impact}万
└─ 行业趋势：{industry}行业整体下滑{industry_decline}%

紧急行动建议：
1. 【本周】召开收入补救专项会，制定追赶计划
2. 【3日内】拜访{key_customer}等TOP5风险客户
3. 【本周】盘点所有Q商，优先推进{amount_over_50k}万以上商机
4. 【本月】启动客户挽留专项，针对{at_risk}家高危客户

请负责人今日18:00前反馈行动计划。
```

### 1.4 排名展示说明

#### 收入排行榜说明

```markdown
## 收入排行榜

### 榜单规则
- **统计口径**：自然月累计确认收入（按权责发生制）
- **更新频率**：T+1日更新昨日数据
- **排名维度**：收入金额从高到低
- **展示范围**：默认TOP20，支持查看全部

### 字段说明
| 字段 | 说明 |
|------|------|
| 排名 | 本月收入排名，并列时按客户ID排序 |
| 客户名称 | 客户企业全称，支持点击查看详情 |
| 本月收入 | 当月确认收入（元），含订阅+服务 |
| 环比 | 较上月收入变化百分比 |
| 同比 | 较去年同期收入变化百分比 |
| 排名变动 | 较上月排名变化（↑/↓/-） |
| 累计收入 | 本年度累计确认收入 |
| 贡献占比 | 该客户收入占整体收入比例 |

### 排名标识
- 🏆 TOP1-3：金牌客户
- 🥈 TOP4-5：银牌客户
- 🥉 TOP6-10：铜牌客户
- ⭐ 新上榜：本月新进TOP20
- 📈 排名上升：较上月上升≥3位
- 📉 排名下降：较上月下降≥3位

### 使用场景
1. **月度复盘**：识别头部客户表现
2. **资源分配**：优先服务高价值客户
3. **销售跟进**：关注排名下滑客户
4. **目标设定**：为头部客户制定增长目标
```

#### 增长排行榜说明

```markdown
## 增长排行榜

### 榜单规则
- **统计口径**：环比增长率 = (本月收入 - 上月收入) / 上月收入 × 100%
- **筛选条件**：上月收入≥1万（排除基数过小导致的异常波动）
- **排名维度**：环比增长率从高到低
- **展示范围**：默认TOP15

### 字段说明
| 字段 | 说明 |
|------|------|
| 排名 | 增长率排名 |
| 客户名称 | 客户企业全称 |
| 环比增长率 | 收入环比变化百分比 |
| 增长额 | 本月较上月收入增加金额 |
| 本月收入 | 当月收入金额 |
| 上月收入 | 上月收入金额（基数） |
| 增长原因 | 系统标注的增长驱动因素（如：扩容/增购/新产品） |

### 增长类型标识
- 🚀 超高速增长：环比>100%
- 📈 高速增长：环比50-100%
- ⬆️ 稳健增长：环比20-50%
- 🔁 恢复性增长：环比>20%且上月环比<-20%
- 🆕 新签约首月：首次产生收入

### 使用场景
1. **成功经验复制**：分析增长客户共性，提炼最佳实践
2. **增购机会识别**：增长客户可能有进一步扩张需求
3. **案例打造**：筛选增长标杆客户用于对外宣传
4. **资源倾斜**：将成功经验复制给其他类似客户
```

#### 贡献度排行榜说明

```markdown
## 贡献度排行榜

### 榜单规则
- **统计口径**：客户累计收入占整体累计收入的比例
- **统计周期**：自然年度累计
- **更新频率**：月度更新
- **排名维度**：贡献占比从高到低

### 字段说明
| 字段 | 说明 |
|------|------|
| 排名 | 年度贡献排名 |
| 客户名称 | 客户企业全称 |
| 年度贡献 | 本年度累计收入金额 |
| 贡献占比 | 占整体年度收入比例 |
| 去年贡献 | 去年同期累计收入 |
| 同比变化 | 较去年同期贡献变化 |
| 客户标签 | 战略/重点/普通 |

### 贡献分层标准
| 层级 | 标准 | 占比阈值 |
```

## 3. 使用规则

### 3.1 指标计算规则（14项核心指标）

#### 收入类指标

| 序号 | 指标名称 | 指标定义 | 计算公式 | 单位 | 更新频率 |
|------|----------|----------|----------|------|----------|
| 1 | 总收入 | 统计周期内确认的全部收入 | ∑收入金额 | 元 | 日/实时 |
| 2 | 订阅收入 | 统计周期内的订阅服务收入 | ∑收入金额 WHERE revenue_type='订阅' | 元 | 日/实时 |
| 3 | 服务收入 | 统计周期内的增值/实施服务收入 | ∑收入金额 WHERE revenue_type='服务' | 元 | 日/实时 |
| 4 | 平均客单价(ARPU) | 每客户平均收入 | 总收入 / 客户数 | 元 | 月 |
| 5 | 收入增长率 | 收入增长百分比 | (本期收入 - 上期收入) / 上期收入 × 100% | % | 月 |

#### 客户类指标

| 序号 | 指标名称 | 指标定义 | 计算公式 | 单位 | 更新频率 |
|------|----------|----------|----------|------|----------|
| 6 | 客户总数 | 统计期末活跃客户数 | COUNT(DISTINCT customer_id) WHERE status='活跃' | 家 | 日 |
| 7 | 新增客户数 | 统计周期内首次产生收入的客户 | COUNT(DISTINCT customer_id) WHERE first_revenue_in_period | 家 | 月 |
| 8 | 流失客户数 | 统计周期内确认流失的客户 | COUNT(DISTINCT customer_id) WHERE churn_date IN period | 家 | 月 |
| 9 | 客户留存率 | 存量客户中继续合作的占比 | (期初客户 - 流失客户) / 期初客户 × 100% | % | 月 |
| 10 | 客户增长率 | 客户数量增长百分比 | (本期客户数 - 上期客户数) / 上期客户数 × 100% | % | 月 |

#### 目标类指标

| 序号 | 指标名称 | 指标定义 | 计算公式 | 单位 | 更新频率 |
|------|----------|----------|----------|------|----------|
| 11 | 目标完成率 | 实际完成占目标的比例 | 实际值 / 目标值 × 100% | % | 月 |
| 12 | 累计完成率 | 年度累计完成占年度目标的比例 | 累计实际 / 年度目标 × 100% | % | 月 |
| 13 | 时间进度 | 当年已过时间占比 | 已过天数 / 当年总天数 × 100% | % | 日 |
| 14 | 进度偏差 | 完成率与时间进度的差距 | 完成率 - 时间进度 | pp | 月 |

#### 计算口径详细说明

```yaml
# 收入确认口径
收入确认:
  原则: 权责发生制
  订阅收入: 按服务期按月分摊确认
  一次性收入: 服务完成当月全额确认
  退款处理: 冲减当期收入
  币种转换: 按收入确认当日汇率折算为CNY

# 客户状态定义
客户状态:
  活跃客户: 近90天内有收入记录且未明确流失
  新客户: 首次产生收入在统计周期内
  流失客户: 合同到期超过90天未续约或明确终止合作
  暂停客户: 暂时中止服务但未正式终止合作

# 环比/同比规则
对比计算:
  环比: 当前周期 vs 紧邻上一周期（日环比昨天、月环比上月）
  同比: 当前周期 vs 去年同期（考虑闰年2月29日取2月28日）
  对比前提: 上期/同期数值>0，否则标记为"N/A"

# 目标管理规则
目标设定:
  年度目标: 每年12月制定下年度目标
  月度目标: 每月初可基于实际情况微调（调整幅度≤±10%）
  目标分解: 按区域/产品线/CSM维度分解，总和=公司总目标
  目标冻结: 每月15日后锁定当月目标
```

### 3.2 排名计算规则

#### 收入排名规则

```yaml
排名维度:
  默认维度: 当月收入金额
  可选维度: 
    - 季度收入
    - 年度累计收入
    - 同比增长率
    - 客户健康度综合得分

排序规则:
  主排序: 收入金额 DESC
  次排序: 同比增长率 DESC（收入相同时）
  三排序: 客户ID ASC（前两者相同时）

并列处理:
  规则: 相同收入排名相同，下一名次跳过
  示例: 第1名:100万, 第2名:95万, 第2名:95万, 第4名:90万

排名变动:
  计算方式: 本月排名 - 上月排名
  显示规则:
    - 变动=0: 显示"-"
    - 变动<0（上升）: 显示"↑{abs(变动)}" 绿色
    - 变动>0（下降）: 显示"↓{变动}" 红色
    - 新上榜: 显示"NEW" 蓝色
    - 掉出榜单: 上月在榜本月不在，显示"OUT"

入榜门槛:
  收入榜: 当月收入>0即可入榜
  增长榜: 上月收入≥1万（排除基数过小）
  贡献榜: 年度累计收入>0
```

#### 增长率排名规则

```yaml
增长率计算:
  公式: (本期收入 - 上期收入) / |上期收入| × 100%
  
特殊处理:
  上期=0,本期>0: 标记为"新签约"，不参与增长率排名
  上期>0,本期=0: 标记为"已流失"，不参与增长率排名
  上期=0,本期=0: 排除在榜单外
  
增长分级:
  超高速增长: >100%
  高速增长: 50%-100%
  稳健增长: 20%-50%
  小幅增长: 0-20%
  持平: -5%-5%
  下滑: <-5%
```

### 3.4 数据更新频率规则

#### 基础数据更新

| 数据类型 | 更新频率 | 更新时点 | 延迟要求 |
|----------|----------|----------|----------|
| 收入数据 | 每日 | 凌晨2:00 | T+1 |
| 客户数据 | 实时+每日 | 变更实时，全量凌晨 | 实时/T+1 |
| 交易数据 | 实时+每日 | 变更实时，全量凌晨 | 实时/T+1 |
| 目标数据 | 月度 | 每月1日 | T+1 |

#### 报表数据更新

| 报表类型 | 更新频率 | 更新时点 | 数据时效 |
|----------|----------|----------|----------|
| 实时看板 | 准实时 | 每5分钟 | 5分钟内 |
| 日报 | 每日 | 上午9:00 | T+1 |
| 周报 | 每周一 | 上午9:00 | 上周一至周日 |
| 月报 | 每月3日 | 上午9:00 | 上月完整数据 |
| 季报 | 每季5日 | 上午9:00 | 上季完整数据 |
| 年报 | 次年1月10日 | 上午9:00 | 上年度完整数据 |

#### 排行榜更新

```yaml
收入榜:
  更新频率: 准实时（每1小时）
  数据范围: 当月1日至当前日
  生效时间: 00:00 - 23:59
  历史归档: 每月1日0点归档上月最终排名

增长榜:
  更新频率: 每日
  更新时点: 上午9:00
  数据范围: 完整自然月对比
  说明: 月内展示为"预排榜"，次月3日确认最终排名

贡献榜:
  更新频率: 每日
  数据范围: 年初1月1日至当前日
  年度归档: 次年1月3日生成年度最终榜
```

#### 数据质量检查

```yaml
每日检查项:
  - 收入数据完整性: 检查昨日数据是否全部入库
  - 异常值检测: 识别偏离历史均值3个标准差的记录
  - 关联一致性: 客户ID、产品ID等外键有效性检查
  - 时间连续性: 检查是否有断档日期

每周检查项:
  - 数据准确性抽样: 随机抽取10条记录人工核对
  - 指标一致性: 同一指标在不同报表中数值一致性
  - 延迟分析: 统计T+1延迟数据的比例

每月检查项:
  - 月度对账: 与财务系统月度收入核对
  - 客户状态校准: 流失客户标记准确性复核
  - 目标值校验: 检查目标分解总和是否等于总目标
```
