# 证券公司AI工作台三端设计方案（重构版）

> 基于现有员工端局部优化，明确三类角色边界，三端各自独立入口，内部均支持多业务线切换。

---

## 一、整体架构：从「导航嵌套」到「独立角色入口」

### 核心变化

| 维度 | 旧设计 | 新设计 |
|------|--------|--------|
| 入口方式 | 一个页面内三端Tab切换 | 三个独立工作台，各自独立入口 |
| 角色边界 | 顶部导航可以随便跳 | 角色边界清晰，跨端通过「提请/下发」触发 |
| 业务线展示 | 7个助理icon平铺 | 按业务线组织，可切换/可聚合 |
| 交互核心 | 列表浏览 | 对话/工单/决策流 |

### 与现有员工端的关键改进点

| 改进项 | 现有设计 | 新设计 |
|--------|----------|--------|
| 业务线区分 | 7个助理icon平铺，不区分业务线 | **业务线切换栏**，点击切换上下文 |
| 智能体展示 | icon+名称，静态 | **卡片式+状态灯+最近动作气泡**，动态 |
| 工作提示 | 列表式文字 | **AI气泡卡片**，带操作建议按钮 |
| 全局输入 | "向AI提问" | **"与AI协作"+@智能体语法**，更像协作 |
| 跨端入口 | 顶部有"业务支持端/协同端"Tab | **去掉**，改为底部快速触发按钮 |
| 待办展示 | 纯列表 | **跨业务线聚合+业务线标签**，一眼分清 |

### 交互逻辑

1. **业务线切换**：点击切换栏某业务线 → 智能体编队刷新为该业务线的智能体 → 工作提示过滤为该业务线 → 待办保持跨线聚合
2. **@智能体**：在底部输入栏输入"@" → 弹出当前业务线下的智能体列表 → 选择后对话上下文锁定到该智能体
3. **提请协同**：点击底部"提请协同" → 弹出协同申请表单 → 选择协同类型（跨部门/资源申请/重大决策）→ 自动路由到协同端
4. **发起质控**：点击底部"发起质控复核" → 自动跳转到支持端（iframe/新窗口）→ 带上下文传递

## 三、业务支持工作台（业务能力中台）

> **核心定位：不管"人"，只管业务和动作。**
> 
> 业务支持中心不是"审批质检台"，而是前台业务团队的"能力放大器"。支持人员的核心职能包括：业务分析、产品创设、客户服务、交叉验证、合规质控——本质是帮前台把业务做得更好，而不是卡流程。

### 入口定位
- **独立URL**: `/workbench/support`
- **默认打开**: 各业务支持中心人员首页（投行支持中心、资管支持中心等）
- **不显示**: 业务团队端/协同端的入口导航

### 关键设计点（与旧认知对比）

| 维度 | 旧认知（错误） | 正确定位 |
|------|--------------|----------|
| **核心职能** | 合规质检、审批卡点 | **业务分析、产品创设、客户服务、交叉验证、合规质控**——帮前台把业务做得更好 |
| **工作对象** | "工单"（被动审批流） | **"业务请求"**（前台主动请求支持中心赋能） |
| **智能体角色** | 合规审查员、复核员 | **业务分析员、产品创设员、客户服务规划员、交叉验证员、合规审查员** |
| **人机关系** | AI预审→人工裁决（审批逻辑） | **AI预处理→人工深化+创造价值**（赋能逻辑） |

### 关键设计点

1. **局势卡片而非列表**：领导的第一诉求是"一屏看全局"，不是逐个阅读。
2. **AI预分析嵌入议题**：每个请示事项都带AI预分析，领导可以快速了解背景和建议方向。
3. **决策追踪闭环**：不是简单的"完成"，而是"指令下发→执行→反馈→闭环"的全链路追踪。
4. **升级机制可视化**：小圆桌→大圆桌的升级路径要清晰，领导可以主动升级也可以等自动升级。
5. **业务线视角切换**：领导可能分管多个业务线，需要快速切换视角。
