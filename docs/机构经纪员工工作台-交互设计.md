## 六、交互设计规范

### 6.1 统一导航结构

```
┌─────────────────────────────────────────────────────────────┐
│  当前位置：引入客户 > 线索收集                                │
├─────────────────────────────────────────────────────────────┤
│  [ < 返回 ]  页面标题                              [操作按钮] │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                        内容区域                             │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│  [主要操作]  [次要操作]  [辅助操作]                          │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 状态标识规范

| 状态 | 标识 | 说明 |
|------|------|------|
| 正常/通过 | 🟢 | 绿色，表示正常或已通过 |
| 警告/关注 | 🟡 | 黄色，表示需要关注 |
| 严重/失败 | 🔴 | 红色，表示严重或失败 |
| 进行中 | ⏳ | 沙漏，表示处理中 |
| 完成 | ✅ | 对勾，表示已完成 |
| 未开始 | ☐ | 方框，表示未开始 |

### 6.3 操作按钮层级

| 层级 | 样式 | 使用场景 |
|------|------|----------|
| 主要操作 | 实心主色按钮 | 最核心的操作，如"确认"、"提交" |
| 次要操作 | 描边按钮 | 辅助操作，如"取消"、"返回" |
| 文字链接 | 纯文字带下划线 | 查看详情、编辑等轻量操作 |

### 6.4 列表交互规范

1. **点击整行**：进入详情页（右侧显示 ">" 箭头）
2. **左滑操作**：快速操作（如删除、标记）
3. **长按**：多选模式
4. **下拉**：刷新列表
5. **上拉**：加载更多

### 6.5 表单交互规范

1. **输入框**：聚焦时显示边框高亮
2. **下拉选择**：点击展开选项列表
3. **日期选择**：弹出日历组件
4. **文件上传**：显示上传进度条
5. **必填项**：标签前加红色星号
6. **错误提示**：输入框下方红色文字
