# 投行业务分析标准模型（v1.0 本质维度版）

> **版本**：v1.0  
> **日期**：2026-05-31  
> **核心定位**：投行业务通用分析模型，覆盖股权融资+债权融资本质判断  
> **设计方法**：先列出所有相关维度，再提炼4个最本质维度  
> **依据**：陶总5.28纪要（简化到3-5个本质维度）+ 用户反馈（抓住业务本质）

---

## 一、所有相关维度（Brainstorm）

> 先不筛选，把判断一个投行项目能不能做的所有可能维度列出来。

### 1.1 企业基本面维度
- 盈利能力：净利润、ROE、毛利率、盈利趋势
- 资产质量：资产负债率、有息债务占比、流动比率、速动比率
- 现金流：经营性现金流净额、自由现金流、现金储备
- 财务健康：审计意见、财务重述、关联交易占比

### 1.2 行业地位维度
- 市场地位：市占率、行业排名、竞争格局
- 行业周期：所处周期（成长/成熟/衰退）、政策支持力度
- 成长前景：收入增长、产能扩张、技术壁垒
- 行业风险：政策限制、替代威胁、产能过剩

### 1.3 融资需求维度
- 资金缺口：实际需要多少钱、缺口怎么算出来的
- 需求紧迫性：立即需要/计划内/可推迟
- 资金用途：募投项目真实性、与主业关联度、效益预期
- 替代方案：是否可通过银行贷款/其他方式解决

### 1.4 融资方案维度
- 规模合理性：相对净资产、相对市值、市场承接力
- 定价/利率：与市场可比、与投资者接受度匹配
- 期限结构：与资金使用周期匹配、偿债高峰分布
- 条款设计：还本付息方式、增信措施、投资者保护条款

### 1.5 市场窗口维度
- 利率环境：基准利率走势、信用利差变化
- 投资者需求：近期认购倍数、投资者结构、我方承销能力
- 监管态度：审批通过率、政策导向、行业限制
- 市场流动性：资金面松紧、发行拥堵情况

### 1.6 偿债能力维度（债权融资特有）
- 现金流覆盖：经营性现金流/到期本息
- 资产负债约束：债券余额/净资产、资产负债率
- 利息覆盖：EBITDA/利息费用
- 偿债来源：经营性现金流、募投项目收益、其他资产变现

### 1.7 合规风险维度
- 历史合规：近3年处罚、违约、诉讼、失信记录
- 当前状态：是否被调查、未决诉讼、负面舆情
- 信息披露：财务真实性、披露完整性、关联交易
- 特有风险：股权质押风险、同业竞争、环保安全

### 1.8 增信措施维度（债权融资特有）
- 信用评级：主体评级、展望、评级机构
- 担保措施：担保方资质、担保方式、覆盖倍数
- 抵押/质押：抵押物类型、评估价值、变现能力
- 其他增信：偿债基金、差额补偿、第三方回购

## 二、提炼：四个本质维度

> **提炼逻辑**：从10个相关维度中，找出最本质、最不可合并、最成体系的4个。  
> **提炼标准**：  
> 1. 每个维度回答一个不可替代的问题  
> 2. 四个维度之间无重叠、无遗漏（MECE原则）  
> 3. 每个维度能用具体指标衡量，信息可获取  
> 4. 四个维度覆盖"企业-交易-市场-风险"完整链条

### 2.1 四个本质维度

| # | 本质维度 | 回答什么问题 | 覆盖的原维度 | 核心判断 |
|---|---------|-------------|------------|---------|
| **1** | **发行人信用质量** | 这家企业本身靠不靠谱？财务健不健康？历史上有没有信用污点？ | 企业基本面 + 行业地位 + 历史信用 | 质地好不好 |
| **2** | **交易结构合理性** | 融资方案设计得合不合理？规模、期限、定价、条款是否匹配企业能力和市场条件？ | 融资需求 + 融资方案 + 增信措施 | 方案对不对 |
| **3** | **市场可执行性** | 现在能不能发出去？投资者买不买？监管批不批？ | 市场窗口 | 能不能发 |
| **4** | **风险可控性** | 风险大不大？出了问题能不能兜底？合规有没有硬伤？ | 偿债能力 + 合规风险 + 风险兜底 | 风险兜不兜得住 |

### 2.2 四个维度的逻辑关系

```
┌─────────────────────────────────────────┐
│           【投行业务分析四维度】           │
├─────────────────────────────────────────┤
│                                         │
│  维度1：发行人信用质量                   │
│  └─ 企业本身靠不靠谱？这是底座。          │
│     如果企业质地差，后面三个维度不用看了。   │
│                                         │
│  维度2：交易结构合理性                   │
│  └─ 方案设计合不合理？这是核心。          │
│     企业好但方案设计错了（比如发太多、      │
│     定价太高、期限不匹配），项目也做不成。   │
│                                         │
│  维度3：市场可执行性                     │
│  └─ 现在能不能发出去？这是外部条件。       │
│     企业好、方案好，但市场窗口不对          │
│     （利率太高、投资者冷淡），也发不出去。   │
│                                         │
│  维度4：风险可控性                       │
│  └─ 风险大不大、能不能兜底？这是底线。     │
│     企业好、方案好、市场好，但风险没人兜底   │
│     （没担保、没增信、合规有硬伤），不敢做。 │
│                                         │
├─────────────────────────────────────────┤
│  【判断顺序】                             │
│  先看过不过关（维度1）→ 再看对不对（维度2）  │
│  → 再看能不能发（维度3）→ 最后看兜不兜得住（维度4）│
│                                         │
│  【任意一个维度[不通过]，项目谨慎推进或暂缓】  │
└─────────────────────────────────────────┘
```

## 三、维度详解（具体指标版）

### 维度1：发行人信用质量

> **本质问题**：这家企业本身靠不靠谱？  
> **核心逻辑**：不看行业黑话，只看三件事：钱够不够还债、资产质量好不好、历史上有没有失信记录。

#### 1.1 财务健康度（具体指标，可获取）

| 指标 | 计算公式 | 数据来源 | 获取路径 | 阈值 |
|------|---------|---------|---------|------|
| **资产负债率** | 总负债/总资产 | 资产负债表 | Wind/同花顺API | ≤70%[正常] / 70-80%[偏高] / >80%[危险] |
| **经营性现金流净额** | 现金流量表直接取数 | 现金流量表 | Wind/同花顺API | 近3年是否为正 |
| **净利润趋势** | 近3年净利润对比 | 利润表 | Wind/同花顺API | 持续增长/波动/连续下滑 |
| **利息覆盖倍数** | EBITDA/利息费用 | 利润表+现金流量表 | 系统计算 | ≥3倍[正常] / 1.5-3倍[偏弱] / <1.5倍[危险] |
| **速动比率** | (流动资产-存货)/流动负债 | 资产负债表 | Wind/同花顺API | ≥1[正常] / <1[偏弱] |
| **审计意见** | 审计报告结论 | 年报 | 巨潮资讯/Wind | 标准无保留/带强调事项/保留意见 |

#### 1.2 行业地位（具体指标，可获取）

| 指标 | 数据来源 | 获取路径 | 判断标准 |
|------|---------|---------|---------|
| **行业排名** | 行业协会/市场研究报告 | 公开信息/客户经理录入 | 行业前10/前30/一般 |
| **市占率** | 公司收入/行业总收入 | 公司公告+行业数据 | ≥10%[领先] / 3-10%[中等] / <3%[边缘] |
| **政策支持力度** | 近期行业政策文件 | 国务院/发改委/工信部 | 鼓励/中性/限制 |
| **行业周期性** | 行业历史数据 | 行业研究 | 强周期/弱周期/非周期 |

#### 1.3 历史信用记录（具体指标，可获取）

| 指标 | 数据来源 | 获取路径 | 判断标准 |
|------|---------|---------|---------|
| **近3年监管处罚** | 证监会/交易所/交易商协会 | 系统抓取公示 | 0次[正常] / 1-2次[注意] / ≥3次[危险] |
| **近3年债务违约** | 中债登/上清所/征信报告 | 系统查询 | 无[正常] / 有[危险] |
| **近3年评级调整** | 评级历史 | 系统记录 | 上调/稳定/下调 |
| **失信记录** | 中国执行信息公开网 | 系统抓取 | 无[正常] / 有[危险] |
| **重大诉讼** | 裁判文书网/公司公告 | 系统抓取 | 无[正常] / 有未决大额诉讼[注意] |

#### 维度1输出格式
```
【发行人信用质量】

财务健康度：
├─ 资产负债率：XX%（来源：Wind资产负债表，阈值：≤70%）→ [正常]/[偏高]/[危险]
├─ 经营性现金流：近3年XX/XX/XX亿元（来源：Wind现金流量表）→ 均为正/有1年为负/...
├─ 净利润趋势：XX/XX/XX亿元（来源：Wind利润表）→ 持续增长/波动/连续下滑
├─ 利息覆盖倍数：XX倍（来源：系统计算，阈值：≥3倍）→ [正常]/[偏弱]/[危险]
├─ 速动比率：XX（来源：Wind资产负债表，阈值：≥1）→ [正常]/[偏弱]
└─ 审计意见：标准无保留/带强调事项/保留意见（来源：年报）

行业地位：
├─ 行业排名：前XX名（来源：行业协会/研报）→ [领先]/[中等]/[边缘]
├─ 市占率：XX%（来源：公司公告+行业数据）→ [领先]/[中等]/[边缘]
├─ 政策支持：鼓励/中性/限制（来源：近期政策文件）
└─ 行业周期：强周期/弱周期/非周期（来源：行业研究）

历史信用：
├─ 近3年监管处罚：X次（来源：证监会公示）→ [正常]/[注意]/[危险]
├─ 近3年债务违约：无/有（来源：中债登/征信）→ [正常]/[危险]
├─ 近3年评级调整：上调/稳定/下调（来源：评级历史）
└─ 失信记录：无/有（来源：执行信息公开网）→ [正常]/[危险]

维度结论：[好]/[一般]/[差]
└─ 依据：XX项正常，XX项注意，XX项危险。核心指标（资产负债率/现金流/利息覆盖）状态。
```

### 维度2：交易结构合理性

> **本质问题**：融资方案设计得合不合理？  
> **核心逻辑**：不看"规模适中"这种空话，只看四件事：发多了没、期限对不对、价格有没有吸引力、条款能不能保护投资者。

#### 2.1 规模合理性（具体指标，可获取）

| 指标 | 计算公式 | 数据来源 | 获取路径 | 阈值 |
|------|---------|---------|---------|------|
| **融资规模/净资产** | 拟融资规模/净资产 | 拟发规模+资产负债表 | 录入+Wind API | 股权≤30%[正常] / 债权≤40%[正常] / 超阈值[危险] |
| **融资规模/市值** | 拟融资规模/总市值（股权） | 拟发规模+市值 | 录入+Wind API | ≤30%[正常] / >30%[偏大] |
| **资金缺口匹配度** | 融资规模/实际资金缺口 | 拟发规模+客户说明 | 录入 | =100%[匹配] / >100%[偏多] / <100%[不足] |
| **补充流动资金占比** | 补流金额/总融资 | 客户资金分配 | 录入 | ≤30%[正常] / >30%[偏高] |

#### 2.2 期限匹配度（具体指标，可获取）

| 指标 | 数据来源 | 获取路径 | 判断标准 |
|------|---------|---------|---------|
| **融资期限** | 客户需求 | 客户经理录入 | X年 |
| **资金使用周期** | 募投项目计划 | 客户提供 | X年 |
| **期限匹配度** | 融资期限 vs 使用周期 | 系统对比 | [匹配]期限≥周期 / [不匹配]期限<周期 |
| **偿债高峰分布** | 未来各年到期债务 | 系统汇总 | 是否与已有债务到期重叠 |

#### 2.3 定价/利率竞争力（具体指标，可获取）

| 指标 | 数据来源 | 获取路径 | 判断标准 |
|------|---------|---------|---------|
| **同类型近期定价** | 近3月同类项目发行利率/定价 | Wind/同花顺 | 均值XX%，区间XX%-XX% |
| **我方建议定价** | 系统计算 | 系统参考同类+发行人资质调整 | 在同类区间[有竞争力]/高于同类[偏高] |
| **与投资者接受度匹配** | 客户经理调研/簿记历史 | 录入 | 预计认购倍数：X倍 |
| **定价对发行人成本影响** | 系统计算 | 利率×规模 | 年化成本：XX万元 |

#### 2.4 条款设计合理性（具体指标，可获取）

| 指标 | 数据来源 | 获取路径 | 判断标准 |
|------|---------|---------|---------|
| **还本付息与现金流匹配** | 还本节奏 vs 经营性现金流 | 系统对比 | [匹配]/[不匹配] |
| **增信覆盖倍数** | 增信可变现价值/融资本息 | 客户提供+系统计算 | ≥1.5倍[充分] / 1.0-1.5倍[一般] / <1.0倍[不足] |
| **投资者保护条款** | 募集说明书 | 客户提供 | 是否有交叉违约/加速到期/信息披露承诺等 |
| **条款与市场可比** | 同类项目条款 | 系统参考历史 | [优于市场]/[持平]/[弱于市场] |

#### 维度2输出格式
```
【交易结构合理性】

规模合理性：
├─ 融资规模/净资产：XX%（股权≤30%/债权≤40%，来源：录入+Wind）→ [正常]/[偏大]/[危险]
├─ 资金缺口匹配度：XX%（融资/缺口，来源：录入）→ [匹配]/[偏多]/[不足]
└─ 补充流动资金占比：XX%（来源：客户资金分配）→ [正常]/[偏高]

期限匹配度：
├─ 融资期限：X年（来源：客户需求）
├─ 资金使用周期：X年（来源：项目计划）
├─ 匹配度：[匹配]/[不匹配]（期限≥周期？）
└─ 偿债高峰：未来X年到期债务XX亿（含本次），与已有债务重叠：是/否

定价/利率竞争力：
├─ 同类型近期：XX%-XX%（来源：Wind近3月，样本XX只）
├─ 我方建议：XX%（来源：系统计算）→ [有竞争力]/[偏高]
├─ 预计认购倍数：X倍（来源：客户经理调研）
└─ 发行人年化成本：XX万元（来源：系统计算）

条款设计：
├─ 还本付息与现金流匹配：[匹配]/[不匹配]（来源：系统对比）
├─ 增信覆盖倍数：XX倍（来源：客户提供+系统计算）→ [充分]/[一般]/[不足]
├─ 投资者保护条款：有/无（交叉违约/加速到期/信息披露承诺）→ [完善]/[一般]/[缺失]
└─ 与市场可比：[优于市场]/[持平]/[弱于市场]

维度结论：[合理]/[一般]/[不合理]
└─ 依据：XX项合理，XX项一般，XX项不合理。核心指标（规模/定价/期限匹配）状态。
```

### 维度3：市场可执行性

> **本质问题**：现在能不能发出去？  
> **核心逻辑**：不看"市场活跃"这种虚话，只看四件事：利率在不在低位、投资者买不买、监管批不批、我方能不能卖得掉。

#### 3.1 利率环境（具体指标，可获取）

| 指标 | 数据来源 | 获取路径 | 判断标准 |
|------|---------|---------|---------|
| **10年期国债收益率** | 中国债券信息网/Wind | 系统API | 当前XX% |
| **近3个月走势** | 每日国债收益率 | 系统API+自动计算 | 下降XXbp/上升XXbp/稳定 |
| **央行政策** | 央行官网 | 系统抓取 | 宽松/中性/收紧 |
| **市场资金面** | Shibor/DR007 | 货币网/Wind | 宽松/中性/紧张 |

#### 3.2 投资者需求（具体指标，可获取）

| 指标 | 数据来源 | 获取路径 | 判断标准 |
|------|---------|---------|---------|
| **近1月同类型认购倍数** | Wind-债券发行/股权簿记 | 系统API | ≥2倍[旺盛] / 1-2倍[一般] / <1倍[冷淡] |
| **我方投资者覆盖** | CRM系统 | 系统查询 | 银行X家/基金X家/券商X家/保险X家 |
| **主要投资者配置需求** | 客户经理调研 | 录入 | 旺盛/一般/谨慎 |
| **近期是否有发行失败案例** | Wind/市场新闻 | 系统抓取 | 有/无（如有，说明原因） |

#### 3.3 监管环境（具体指标，可获取）

| 指标 | 数据来源 | 获取路径 | 判断标准 |
|------|---------|---------|---------|
| **近3月同类型审批通过率** | 交易所/发改委/证监会公示 | 系统抓取 | XX%（样本XX只） |
| **监管关注重点** | 审核反馈函/政策文件 | 系统抓取 | 当前重点关注：XX |
| **是否有支持政策** | 国务院/央行/证监会 | 系统抓取 | 有专项支持（如绿色/科创）/无 |
| **发行人所属行业监管态度** | 近期行业政策 | 系统抓取 | 鼓励/中性/限制 |

#### 3.4 我方承销能力（具体指标，可获取）

### 维度4：风险可控性

> **本质问题**：风险大不大？出了问题能不能兜底？  
> **核心逻辑**：不看"风险较小"这种空话，只看四件事：现金流能不能覆盖到期债务、合规有没有硬伤、增信靠不靠谱、风险分散了没有。

#### 4.1 现金流覆盖（具体指标，可获取）

| 指标 | 计算公式 | 数据来源 | 获取路径 | 阈值 |
|------|---------|---------|---------|------|
| **现金流覆盖率** | 近3年平均经营性现金流/年还本付息 | 现金流量表+融资方案 | 系统计算 | ≥1.5[正常] / 1.0-1.5[偏弱] / <1.0[危险] |
| **偿债来源集中度** | 最大单一偿债来源/总偿债来源 | 现金流+项目收益+其他 | 系统计算 | <50%[分散] / ≥50%[集中] |
| **专项偿债账户** | 募集说明书/承销协议 | 客户提供 | 录入 | 有/无 |
| **极端情景测试** | 现金流下降30%后的覆盖率 | 系统计算 | 系统计算 | ≥1.0[可承受] / <1.0[危险] |

#### 4.2 合规状态（具体指标，可获取）

| 指标 | 数据来源 | 获取路径 | 判断标准 |
|------|---------|---------|---------|
| **当前被调查状态** | 公司公告/监管公示 | 系统抓取 | 无[正常] / 有[危险]（说明机关/事由） |
| **未决诉讼** | 裁判文书网/公司公告 | 系统抓取 | 无[正常] / 有[注意]（金额/影响） |
| **重大负面舆情** | 舆情监控系统 | 系统抓取 | 无[正常] / 有[注意]（发酵程度） |
| **信息披露质量** | 近3年年报/问询函 | 系统抓取 | 无问询[正常] / 有问询[注意] / 多次问询[危险] |
| **关联交易占比** | 年报关联交易章节 | Wind/年报 | <30%[正常] / ≥30%[注意] |

#### 4.3 增信有效性（具体指标，可获取）

| 指标 | 数据来源 | 获取路径 | 判断标准 |
|------|---------|---------|---------|
| **担保方评级** | 评级机构报告 | 系统查询 | ≥AA[有效] / <AA[偏弱] |
| **担保方式** | 担保合同 | 客户提供 | 连带责任/一般保证/抵押/质押 |
| **抵押物变现能力** | 抵押物类型+市场流动性 | 客户提供+系统评估 | 高（房产/土地）/中（设备）/低（专用资产） |
| **增信覆盖倍数** | 增信可变现价值/融资本息 | 客户提供+系统计算 | ≥1.5[充分] / 1.0-1.5[一般] / <1.0[不足] |
| **信用增进机构实力** | 中债增/中证增等 | 系统查询 | 国资背景[强] / 民营[弱] |

#### 4.4 风险分散度（具体指标，可获取）

| 指标 | 数据来源 | 获取路径 | 判断标准 |
|------|---------|---------|---------|
| **投资者集中度** | 簿记建档结果 | 系统记录 | 前5大投资者占比<50%[分散] / ≥50%[集中] |
| **区域集中度** | 发行人/项目所在地 | 系统记录 | 单一区域占比<50%[分散] / ≥50%[集中] |
| **行业集中度** | 发行人所属行业 | 系统记录 | 是否已有多只同行业债券在存续期[是/否] |
| **期限集中度** | 本次期限 vs 已有债务期限 | 系统汇总 | 是否导致某一年偿债高峰[是/否] |

## 四、综合判断

### 4.1 判断逻辑

```
四个维度全部[通过] → 项目可行，可以推进
任意一个维度[不通过] → 项目有硬伤，谨慎推进或暂缓
四个维度都[通过]但某维度[注意] → 项目可行，但需关注该维度风险

判断顺序：
1. 先看维度1（发行人信用质量）→ [不通过]则其他维度不用看，直接暂缓
2. 再看维度2（交易结构合理性）→ [不通过]则方案需重新设计
3. 再看维度3（市场可执行性）→ [不通过]则等待窗口或调整策略
4. 最后看维度4（风险可控性）→ [不通过]则风险无法兜底，不能推进
```

## 五、与股权/债权融资的对应关系

| 四个维度 | 股权融资（定增/IPO）侧重 | 债权融资（公司债/中票）侧重 |

| **发行人信用质量** | 盈利增长性、股本结构、治理规范 | 偿债能力、资产负债、现金流稳定 |
| **交易结构合理性** | 定价折扣、稀释比例、锁定期 | 利率区间、期限匹配、还本付息方式 |
| **市场可执行性** | 市场承接力、投资者认购、审核通过率 | 利率环境、认购倍数、资金面 |
| **风险可控性** | 信息披露风险、股东结构风险、经营风险 | 偿债风险、合规风险、增信有效性 |
