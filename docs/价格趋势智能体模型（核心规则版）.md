# 价格趋势智能体模型

> **版本**：V1.0
> **业态**：证券投资/财富管理
> **编制依据**：陶总业务模型方法论（五大反对 + 八步方法论）
> **核心规则**：不可修改

---

## 一、目的定义

**一句话：帮助投资顾问判断"现在是不是该建议投资者关注趋势变化的时候"，不预测股价、不代客下单、不承诺收益。**

围绕这个目的，提炼两个可调用模型：
1. **业务分析模型** — 分析判断用
2. **规定动作模型** — 执行标准用

---

## 二、角色体系（三类通用角色）

| 角色 | 定位 | 核心诉求 |
|------|------|----------|
| 投资者（服务接收方） | 趋势分析服务的接收者 | 想知道现在该不该动 |
| 投资顾问（服务交付方） | 调用智能体、审核输出、对外沟通 | 需要可靠依据、可解释、可留痕 |
| 合规风控（运营组织者） | 穿透监督、把控风险 | 零合规事故、可审计 |

智能体是工具，不是角色。不独立对外，只被投资顾问调用。

---

## 三、业务分析模型（4维度）

**目的：判断"现在是什么天气、这个标怎么样、该不该动、动了亏多少认输"**

### 维度1：市场环境（现在是什么天气）

| 指标 | 含义 | 判断标准 |
|------|------|----------|
| 波动状态 | 市场活跃程度 | 历史波动率前20%→大热天；中间60%→正常天；后20%→大冷天 |
| 情绪状态 | 恐慌还是乐观过头 | 预期波动远高于实际→恐慌；远低于实际→乐观过头 |
| 趋势方向 | 大盘整体涨跌 | 沪深全A指数上升/下降/横盘 |

### 维度2：标的趋势（这个标怎么样）

| 指标 | 含义 | 判断标准 |
|------|------|----------|
| 走势形态 | 上升、下降还是横盘 | 过去20日低点逐级抬高→上升；高点逐级降低→下降；上下乱晃→横盘 |
| 高低位置 | 在半山腰、接近顶部还是接近底部 | 近期高低点对比 |
| 趋势强弱 | 涨得猛不猛、有没有过头 | 短期平均价格斜率+价格偏离度 |

### 维度3：时间一致（三个时间框架是否同向）

| 日线 | 60分钟 | 15分钟 | 信号强度 |
|------|--------|--------|----------|
| 涨 | 涨 | 涨 | **强信号** |
| 涨 | 涨 | 跌 | 中等信号 |
| 三个方向打架 | 矛盾 | 矛盾 | **无信号** |

### 维度4：风险防护（亏了怎么办）

| 指标 | 含义 | 判断标准 |
|------|------|----------|
| 仓位建议 | 根据市场温度投入多少 | 大热天→轻仓/空仓；正常天→半仓；大冷天→空仓/极轻仓 |
| 止损参考 | 如果错了，最多亏多少 | 根据近期平均波动大小动态调整，不是固定百分比 |

**模型输出格式：**

```
市场环境 = { 波动状态: [大热天/正常天/大冷天], 情绪状态: [恐慌/正常/乐观过头], 趋势方向: [上升/下降/横盘] }
标的趋势 = { 走势形态: [上升/下降/横盘], 高低位置: [接近顶部/半山腰/接近底部], 趋势强弱: [强/中/弱/过头] }
时间一致 = { 日线: [涨/跌/横], 60分钟: [涨/跌/横], 15分钟: [涨/跌/横], 信号强度: [强/中等/弱/无信号] }
风险防护 = { 建议仓位: [满仓/半仓/轻仓/空仓], 止损参考: [具体价格] }
```

## 四、规定动作模型（投资者趋势咨询响应）

**事件**：投资者询问某标的"现在趋势怎么样"
**触发**：客户主动提问 或 系统监控到持仓标的趋势变化
**执行角色**：投资顾问（调用智能体）

### 步骤

| 步骤 | 动作 | 时限 | 输出 | 异常处理 |
|------|------|------|------|----------|
| 1 | 验证投资者适当性 | 30秒 | 匹配标的风险等级与投资者适当性 | 不符→拒绝，提示风险等级 |
| 2 | 调取市场环境数据 | 30秒 | 波动状态、情绪状态、趋势方向 | 缺失→标记"数据不足" |
| 3 | 调取标的趋势数据 | 30秒 | 走势形态、高低位置、趋势强弱 | 停牌→标记"不可交易" |
| 4 | 判断时间一致性 | 30秒 | 三个时间框架同向性 | 打架→标记"无信号" |
| 5 | 生成风险防护建议 | 30秒 | 仓位建议、止损参考 | 极端行情→触发熔断建议 |
| 6 | 生成完整报告 | 30秒 | 四维度输出 + 免责声明 | 格式异常→重新生成 |
| 7 | 投资顾问审核 | 2分钟 | 确认逻辑自洽、数据有效、合规无误 | 错误→退回修改 |
| 8 | 对外输出 | 1分钟 | 通过合规渠道发送给投资者 | 追问→记录并回应 |

**总时限**：3-5分钟

**输出必须包含：**
- 四维度分析结果
- 信号强度评级（强/中等/弱/无）
- 免责声明："本建议仅供参考，不构成投资建议，投资者需独立判断"
- 数据时效标注

**验收标准：**
- 任何含"建议买入/卖出"的输出必须人工审核
- 信号强度为"弱"或"无"时，建议为"观望，不操作"
- 触发"不"事项时，直接拒绝，标记原因

## 五、三句话法（核心合规规则）

### 可以（AI自动执行，人工按需查阅）

| 事项 | 智能体动作 |
|------|------------|
| 市场环境查询 | AI自动生成，员工确认后发送 |
| 标的趋势方向查询 | AI自动生成，员工确认后发送 |
| 历史数据展示 | AI自动展示，无需审核 |

### 必须确认（AI生成→人工必须逐条审核→确认后才能对外）

| 事项 | 智能体动作 |
|------|------------|
| 含买卖建议的输出 | AI生成→员工逐条审核→确认后发送 |
| 含仓位建议的输出 | AI生成→员工逐条审核→确认后发送 |
| 含止损建议的输出 | AI生成→员工逐条审核→确认后发送 |

### 不（AI直接拒绝，标记原因，升级人工处理）

| 事项 | 智能体动作 |
|------|------------|
| 预测股价/目标价 | AI直接拒绝，输出"本系统不预测股价" |
| 代客下单 | AI直接拒绝，输出"本系统不提供交易服务" |
| 承诺保本保收益 | AI直接拒绝，输出"投资有风险，不承诺收益" |
| 适当性不符仍给建议 | AI直接拒绝，输出"该标的超出您的适当性范围" |

## 六、检查清单（陶总五大反对）

| 反对项 | 检查标准 | 是否通过 |
|--------|----------|----------|
| 反对讲意义、讲作用、讲自洽，而不讲模型本身 | 是否有明确的4维度分析模型 + 8步动作模型 | ✅ |
| 反对流程化、线性化地拆业务 | 步骤模型是否有每步标准、输出、异常，不是简单流程堆砌 | ✅ |
| 反对角色混乱、目标混乱 | 角色是否只分三类（投资者/投资顾问/合规），智能体是否只是工具 | ✅ |
| 反对主观拍脑袋和甩锅给大模型 | 每个维度是否有具体指标、判断标准、可追溯 | ✅ |
| 反对传统信息系统视角和技术自说自话 | 是否从投资者和顾问视角出发，不堆砌技术名词 | ✅ |

## 七、监督迭代

**监控指标：**
- 信号命中率：低于60%→触发调优
- 合规命中率：低于99%→触发整改
- 人工介入率：高于50%→触发AI能力升级评估
- 响应时效：超过5分钟→触发优化

**迭代触发：**
- 信号命中率连续2周低于60%→模型参数重调
- 同一异常场景超过10次/周→异常处理流程优化
- 季度标杆跟踪更新→对比行业最新做法，调整模型
- 监管政策变化→三句话法规则更新
