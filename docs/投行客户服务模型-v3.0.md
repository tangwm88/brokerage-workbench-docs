# 投行客户服务模型 v3.0（动作导向版）

> **版本**：v3.0
> **日期**：2026-05-31
> **定位**：客户服务智能体 = 客户关系维护 + 项目执行 + 持续督导
> **核心变化**：从"关系状态诊断"转向"动作执行管理"，维度按服务动作重新设计
> **原则**：客观记录动作事实 + 判断执行到位/不到位 + 输出下一步动作清单

---

## 一、整体定位

**客户服务智能体的核心动作**：

```
┌─────────────────────────────────────────┐
│          【客户服务智能体】               │
│                                         │
│  核心动作1：客户关系维护                  │
│  ├─ 日常沟通、节日问候、定期拜访           │
│  ├─ 客户活动、沙龙、路演邀请               │
│  ├─ 关键人关系深化、非业务互动             │
│  └─ 客户需求持续跟踪、信息同步              │
│                                         │
│  核心动作2：项目执行                       │
│  ├─ 项目进度管理、关键节点提醒              │
│  ├─ 材料制作协调、质量把控                 │
│  ├─ 客户问题响应、跨部门协调               │
│  └─ 执行过程沟通、状态同步                │
│                                         │
│  核心动作3：持续督导                       │
│  ├─ 督导计划执行、合规检查清单              │
│  ├─ 信息披露提醒、合规辅导                 │
│  ├─ 培训与指导、客户教育                   │
│  └─ 持续回访、关怀跟踪、服务评价           │
│                                         │
└─────────────────────────────────────────┘
```

**模型体系关系**：
```
客户分析底座（统一数据）
    ↓
② 业务分析标准模型（项目能不能做）→ 项目立项
    ↓
③ 方案/产品创设模型（方案怎么设计）→ 方案确认
    ↓
④ 交叉验证模型（有没有盲区）→ 验证通过
    ↓
项目执行过程 ←── ⑤ 客户服务智能体（项目执行动作）
    ↓
项目交割完成
    ↓
持续督导期 ←──── ⑤ 客户服务智能体（持续督导动作）
    ↓
日常客户关系维护 ← ⑤ 客户服务智能体（客户关系维护动作）
    ↓
发现新需求 → 回流 ② 业务分析标准模型（循环）
```

### 2.2 维度详解

#### 维度1：客户关系维护

**核心问题**：日常客户关系维护的动作做了哪些？做到什么程度？哪些该做没做？

**分析框架**：
```

  ├─ 上季度分层：[XX] → 本季度：[XX]（变化：升级/降级/维持）
  └─ 分层变化原因：XX（如：新增项目/需求休眠/竞品切入）

【日常沟通动作】（按时间线记录，不遗漏）

  ├─ 我方主动触达：X次 / 客户主动联络：X次

  │   ├─ 已完成动作：5月完成3次互动（2次电话+1次午餐）
  │   └─ 缺失动作：无，按计划执行
  ├─ 经办人：
  │   ├─ 当前关系状态：每周联络，纯业务
  │   ├─ 计划动作：每周1次微信/电话 + 季度1次小聚
  │   ├─ 已完成动作：5月完成4次联络 + 1次下午茶
  │   └─ 缺失动作：无，按计划执行
  └─ 关系深化薄弱点：决策层（董事长）无主动触达，[缺失]

【服务活动动作】（记录活动组织/参与情况）
  ├─ 活动1：2026-05-15 半导体行业投资策略沙龙
  │   ├─ 我方组织/参与：组织
  │   ├─ 客户参与情况：董秘+投资总监出席，董事长未出席
  │   ├─ 活动效果：客户主动询问并购趋势，我方研究员现场交流
  │   └─ 后续跟进：已发送并购案例资料，待反馈
  ├─ 活动2：2026-05-30 上市公司合规培训

【到位判断】
  ├─ [到位]：沟通频次达标、关键人关系深化按计划执行、活动组织/参与正常、需求跟踪有节奏
  ├─ [部分到位]：沟通频次达标但关键人关系深化有缺失，或活动覆盖不足，或需求跟踪无节奏
  └─ [缺失]：沟通频次不达标、关键人关系无深化动作、活动无组织/参与、需求无跟踪
```

## 三、动作标准模型（按角色 + 按动作方向）

> **定位**：各角色面对"客户关系维护、项目执行、持续督导"三个方向，该调用什么模型、做什么动作。

### 3.1 角色体系

| 角色 | 本质定位 | 核心动作方向 | 工作台 |
|------|---------|------------|--------|
| **客户经理** | 客户全周期服务执行者 | 客户关系维护 + 项目执行服务 + 持续督导执行 | 业务团队AI工作台 |
| **组织者** | 客户服务统筹者 | 客户分层 + 资源调配 + 服务监督 | 业务支持AI工作台 |
| **产品支持** | 服务方案准备者 | 活动方案 + 项目执行方案 + 督导方案 | 业务支持AI工作台 |
| **质控内核** | 服务质量把关者 | 项目执行质量检查 + 督导合规审查 | 业务支持AI工作台 |

### 3.2 客户经理动作标准模型

#### 模型A：客户关系维护执行模型

**本质目标**：系统化执行客户关系维护动作，确保日常沟通、关系深化、活动、需求跟踪按计划执行。

**调用触发**：
- 月初/季度初（制定维护计划）
- 客户状态评估显示"客户关系维护：[部分到位]"或"[缺失]"时
- 客户分层变化时（升级/降级）

**输入**：客户分层 + 当前关系网络状态 + 需求清单

**输出**：维护计划表（动作清单 + 时间表 + 责任人）

**计划模板**：
```
客户关系维护计划：XX科技（2026年6月）

【日常沟通】
  ├─ 目标频次：重要客户≥4次/月
  ├─ 计划动作：
  │   ├─ 6月05日：拜访董秘，确认债券发行规模（计划时间1小时，准备材料：债券案例）
  │   ├─ 6月10日：总经理拜访董事长（计划时间30分钟，准备材料：行业报告）
  │   ├─ 6月15日：电话经办人，了解半年报准备情况（计划时间15分钟）
  │   ├─ 6月20日：微信董秘，分享行业动态（计划时间5分钟，准备材料：新闻摘要）
  │   └─ 6月25日：电话经办人，月度例行沟通（计划时间15分钟）
  └─ 沟通覆盖率目标：决策层≥1次/月，执行层≥2次/月，经办层≥2次/月

【关键人深化】
  ├─ 董事长（当前：无主动触达）：
  │   ├─ 目标：建立主动触达，每季度≥1次
  │   ├─ 6月动作：总经理拜访（6月10日）+ 发送行业报告（6月20日）
  │   └─ 后续计划：Q3邀请参加行业峰会（9月）
  ├─ 董秘（当前：每月3次）：
  │   ├─ 目标：维持现有频次，增加非业务互动
  │   ├─ 6月动作：6月05日拜访（业务）+ 6月15日午餐（非业务）
  │   └─ 后续计划：Q3安排客户活动（7月）
  └─ 经办人（当前：每周4次）：
     ├─ 目标：维持现有频次
     ├─ 6月动作：每周微信/电话联络 + 6月25日小聚
     └─ 后续计划：维持

【服务活动】
  ├─ 6月计划：邀请参加半导体行业闭门研讨会（6月20日，我方主办）
  │   ├─ 目标：董事长或投资总监出席（决策层覆盖）
  │   ├─ 邀请对象：董事长、投资总监、董秘
  │   └─ 跟进动作：6月05日拜访时口头邀请 → 6月10日发送正式邀请函
  └─ 6月参与：配合监管组织的上市公司培训（如有）

【需求跟踪】
  ├─ 债券发行（显性）：
  │   ├─ 目标：6月确认规模和时间，推动立项
  │   ├─ 6月动作：6月05日拜访董秘确认（如已确认，则推动立项）
  │   └─ 跟踪频率：每2周1次，直至确认
  ├─ 并购顾问（隐性）：
  │   ├─ 目标：获取客户反馈，判断培育效果
  │   ├─ 6月动作：6月05日拜访时顺带了解并购意向 + 推送一对一并购案例（6月15日）
  │   └─ 跟踪频率：每月1次互动，每季度1次案例推送
  └─ 新需求发现：每次沟通记录客户提及的新需求或痛点
```

#### 模型B：项目执行服务模型

**本质目标**：项目执行过程中提供全过程服务，确保进度跟踪、沟通同步、质量把控、问题响应到位。

**调用触发**：项目立项后、进入新阶段时、关键节点前、问题出现时。

**输入**：项目计划 + 当前阶段 + 客户状态

**输出**：项目执行服务计划（进度跟踪表 + 沟通计划 + 质量检查清单 + 问题响应机制）

**计划模板**：
```
项目执行服务计划：XX科技债券发行

【进度跟踪】
  ├─ 当前阶段：材料制作阶段（申报前）
  ├─ 关键节点：
  │   ├─ 董事会决议（6月30日）：提前3天提醒客户准备材料
  │   ├─ 股东会决议（7月15日）：提前1周提醒客户安排时间
  │   ├─ 申报交易所（7月30日）：提前2周检查材料进度，确保100%完成
  │   └─ 反馈回复（预计8月）：收到反馈后24小时内协调团队准备回复
  ├─ 跟踪频率：每周1次检查（每周五下午）
  └─ 预警机制：节点前2周如进度<80%，升级组织者协调资源

【沟通同步】
  ├─ 内部沟通：
  │   ├─ 项目例会：每周1次（周一上午，团队全员）
  │   ├─ 跨部门协调：需要时随时（债券销售、研究所等）
  │   └─ 组织者汇报：每2周1次（进度+风险+需协调事项）
  ├─ 客户沟通：
  │   ├─ 定期汇报：每2周1次（向董秘汇报进度，书面+口头）
  │   ├─ 关键节点同步：节点前3天同步流程要点和注意事项
  │   └─ 问题反馈：客户提出问题后4小时内响应，24小时内给出解决方案或时间表
  └─ 沟通记录：每次沟通后24小时内记录到CRM系统

【质量把控】
  ├─ 材料检查：每版提交前必查（检查清单：数据准确性、格式合规、逻辑连贯、风险披露）
  ├─ 交付时效：尽量提前1-2天交付，预留客户反馈时间
  ├─ 客户反馈处理：收到反馈后24小时内响应，48小时内完成修改
  └─ 质量记录：每次检查记录问题清单，存档备查

【问题响应】
  ├─ 响应机制：客户提出问题 → 4小时内响应确认 → 24小时内给出解决方案或时间表
  ├─ 升级机制：复杂问题（如涉及监管判断）→ 当天协调产品支持/质控内核 → 48小时内给出专业意见
  ├─ 问题台账：记录每个问题（提出时间、响应时间、解决时间、客户反馈）
  └─ 预防机制：常见问题整理FAQ，提前发送客户，减少重复问题
```

#### 模型C：持续督导执行模型

**本质目标**：项目完成后系统执行督导动作，确保督导计划、合规跟踪、培训、回访按计划执行。

**调用触发**：项目交割完成后、进入督导期时、季度初、年度督导计划制定时。

**输入**：项目类型 + 督导期要求 + 客户状态

**输出**：督导执行计划（督导计划表 + 合规跟踪表 + 培训计划 + 回访计划）

**计划模板**：
```
持续督导执行计划：XX科技定增（督导期：2025-12至2026-12）

【督导计划】
  ├─ 月度沟通：每月15日电话/微信联络，了解经营情况（计划10-15分钟）
  ├─ 季度回访：每季度最后一个月现场回访，检查募集资金使用（计划半天）
  │   ├─ Q3回访：2026年9月（提前2周协调客户时间，确保董秘层级接待）
  │   └─ Q4回访：2026年12月（年度回访，检查全年募集资金使用+新需求挖掘）

  ├─ 年度辅导：年报披露前3天，审核关键章节，确保合规（2027年3月）
  └─ 计划执行率目标：100%（允许因客户原因延期，但需记录并补做）

【合规跟踪】
  ├─ 披露提醒：定期报告前3天提醒客户（年报/半年报/一季报/三季报）
  ├─ 问询函跟踪：收到问询函当天提醒客户，5日内回复提醒，提供回复辅导
  ├─ 募集资金检查：季度回访时同步检查，确保使用合规、进度合理
  ├─ 重大事件跟踪：关注客户重大合同、并购、人事变动等，及时提醒披露义务
  └─ 合规记录：每次合规跟踪记录到台账，存档备查

【培训与指导】
  ├─ 培训计划：
  │   ├─ Q3专项培训：2026年8月，募集资金使用规范（我方组织，客户董秘+财务+经办人参加）
  │   └─ 年度培训：配合监管组织的年度合规培训（时间待定）
  ├─ 日常指导：随需随教（问询函回复、披露要点、合规疑问等）
  └─ 培训记录：每次培训记录参与人、内容、效果，存档备查

【回访与关怀】
  ├─ 季度回访：
  │   ├─ 提前2周协调客户时间，确保决策层或执行层接待
  │   ├─ 回访内容：募集资金使用检查 + 经营情况了解 + 新需求挖掘 + 服务满意度了解
  │   └─ 回访产出：检查报告1份 + 新需求记录 + 客户反馈记录
  ├─ 关怀动作：
  │   ├─ 节日问候：元旦、春节、中秋、国庆（微信发送）
  │   ├─ 生日问候：关键人生日（微信发送，提前标记）
  │   └─ 重大事件关怀：客户重大利好/利空时发送祝贺/慰问（注意分寸）
  └─ 关怀记录：记录到CRM，避免过度打扰
```

### 3.3 组织者动作标准模型

#### 模型D：客户分层管理模型

**本质目标**：按客观指标对客户分层，制定差异化维护策略，确保资源分配合理。

**调用触发**：季度客户经营复盘、年度客户分层评审、客户状态重大变化时。

**输入**：全部客户的客户服务业务标准模型输出（汇总）

**输出**：客户分层表 + 差异化维护策略 + 资源分配方案

**分层标准**：
```
客户分层标准（客观阈值）：

【核心客户】
  ├─ 近3年项目数≥2个，或年项目价值≥500万
  ├─ 有明确后续需求（显性需求已确认或隐性需求置信度[高]）
  ├─ 关系网络至少2层有效（执行层+决策层）
  ├─ 客户服务业务标准模型：项目执行[到位] + 持续督导[到位] + 客户关系维护[到位]
  └─ 维护策略：总经理级定期拜访、专属服务团队、优先资源调配、季度复盘

【重要客户】
  ├─ 近3年项目数≥1个，或年项目价值≥100万
  ├─ 有需求信号（显性需求跟进中或隐性需求培育中）
  ├─ 关系网络至少1层有效（执行层）
  ├─ 客户服务业务标准模型：项目执行[到位] + 客户关系维护[部分到位]
  └─ 维护策略：客户经理每月维护、定期活动邀请、重点需求跟踪、半年度复盘

【一般客户】
  ├─ 近3年项目数1个，暂无明确后续需求
  ├─ 关系网络薄弱（仅经办层）
  ├─ 客户服务业务标准模型：客户关系维护[部分到位]或[缺失]
  └─ 维护策略：标准化维护（季度联络+年度回访）、被动响应需求、年度复盘

【休眠客户】
  ├─ 近3年无项目，无需求信号
  ├─ 关系无维护或已断裂
  ├─ 客户服务业务标准模型：客户关系维护[缺失]
  └─ 维护策略：激活尝试（1-2次主动联络），无响应则放入休眠池，年度评估是否移除
```

**输出示例**：
```

分层结果：
├─ 核心客户：X家（名单：XX、XX、XX）

└─ 休眠客户：X家
    └─ 激活策略：1次主动联络 + 1次活动邀请，无响应则休眠

分层变化：
├─ 升级：XX科技从[重要]→[核心]（原因：新增债券发行需求，价值300万）
├─ 降级：XX公司从[重要]→[一般]（原因：1年无需求信号，互动减少）
```

### 3.4 产品支持动作标准模型

#### 模型F：服务方案准备模型

**本质目标**：为客户关系维护、项目执行、持续督导准备配套的服务方案、材料、活动。

**调用触发**：客户活动组织前、项目执行方案设计时、督导培训准备时。

**输入**：客户服务业务标准模型输出（需求、关系状态、项目状态）

**输出**：服务方案（活动方案 / 项目执行方案 / 督导方案）

**方案类型**：

```
【持续督导方案】
  ├─ 督导计划：督导周期、频率、内容、责任人
  ├─ 培训方案：培训主题、培训对象、培训材料、培训方式

```

### 3.5 质控内核动作标准模型

#### 模型G：项目执行质量检查模型

**本质目标**：检查项目执行过程中的服务质量，确保材料质量、交付时效、问题响应达标。

**调用触发**：项目关键节点前、客户投诉时、定期质量抽查时。

**输入**：项目执行记录 + 客户反馈 + 问题台账

**输出**：质量检查报告 + 整改建议 + 预防措施

---

#### 模型H：督导合规审查模型

**本质目标**：审查持续督导过程中的合规性，确保督导动作、信息披露提醒、培训内容符合监管要求。

**调用触发**：督导动作执行后、监管检查前、客户投诉或纠纷时。

**输入**：督导记录 + 合规跟踪记录 + 培训记录

**输出**：合规审查报告 + 整改建议 + 风险预警

## 四、同步响应机制（动作导向）

```
事件：客户经理完成客户服务业务标准模型分析
    ↓
【同步触发】
    ├─ 客户经理：
    │   ├─ 获得3维度动作执行状态（客户关系维护/项目执行/持续督导）
    │   ├─ 调用客户关系维护执行模型 → 制定维护计划
    │   ├─ 调用项目执行服务模型 → 制定执行服务计划
    │   └─ 调用持续督导执行模型 → 制定督导计划
    │
    ├─ 组织者：
    │   ├─ 参考客户关系维护状态 → 调用客户分层管理模型 → 客户分层
    │   ├─ 参考3维度到位情况 → 调用服务资源调配模型 → 资源分配
    │   └─ 审批关键动作（如总经理拜访、专项培训）
    │
    ├─ 产品支持：
    │   ├─ 参考维护计划 → 调用服务方案准备模型 → 活动方案/材料
    │   ├─ 参考执行服务计划 → 准备项目执行配套方案
    │   └─ 参考督导计划 → 准备培训方案/督导材料
    │
    └─ 质控内核：
        ├─ 参考项目执行状态 → 调用项目执行质量检查模型 → 质量检查
        └─ 参考督导状态 → 调用督导合规审查模型 → 合规审查
    ↓
【同步汇聚】
    ├─ 客户经理：获得维护计划+执行计划+督导计划（已获组织者确认）
    ├─ 组织者：获得客户分层表+资源分配表+风险预警
    ├─ 产品支持：获得服务方案清单+材料准备清单
    └─ 质控内核：获得质量检查报告+合规审查报告
    ↓
【实时反馈】统一视图中各角色看到各自动作任务和状态
```
