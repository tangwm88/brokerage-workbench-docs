## 三、使用规则

### 3.1 负载计算规则

#### 核心公式

```
负载值 = Σ(任务权重 × 时间系数 × 难度系数)
负载率 = 负载值 / 满载值 × 100%
```

#### 任务权重表

| 任务类型 | 权重基准 | 说明 |
|----------|----------|------|
| 客户对接 | 1.5 | 含客户沟通、拜访、路演等 |
| 研究报告 | 2.0 | 深度研究报告撰写 |
| 协同支持 | 1.2 | 内部协同服务 |
| 内部事务 | 0.8 | 会议、培训、文档整理 |
| 紧急任务 | 基准×1.5 | 优先级P0任务 |
| 常规任务 | 基准×1.0 | 优先级P1/P2任务 |

#### 时间系数

```
时间系数 = 实际剩余工时 / 计划总工时

其中：
- 实际剩余工时 = 计划总工时 - 已投入工时
- 若实际剩余工时 < 0（超期），时间系数 = 2.0（压力倍增）
- 若剩余时间充足，时间系数 = 1.0
- 若临近截止，时间系数 = 1.0 ~ 1.5（线性增长）
```

#### 难度系数

| 难度等级 | 系数 | 判定标准 |
|----------|------|----------|
| 简单 | 1.0 | 常规工作，有成熟模板 |
| 中等 | 1.5 | 需要一定专业判断 |
| 困难 | 2.0 | 复杂问题，需多方协调 |
| 极难 | 3.0 | 创新探索，无先例可循 |

#### 负载等级划分

| 负载率范围 | 等级 | 颜色标识 | 处置建议 |
|------------|------|----------|----------|
| 0-40% | 空闲 | 🟢 绿色 | 可分配新任务 |
| 41-70% | 正常 | 🟢 绿色 | 正常状态 |
| 71-85% | 偏高 | 🟡 黄色 | 谨慎分配 |
| 86-100% | 过载 | 🔴 红色 | 停止分配，考虑减负 |
| >100% | 严重过载 | 🔴 红色 | 立即干预 |

#### 满载值设定

```
标准满载值 = 8小时/天 × 5天/周 = 40小时/周

调整系数：
- 新员工（试用期）：满载值 × 0.8
- 资深员工（3年+）：满载值 × 1.0
- 管理层：满载值 × 0.9（预留管理时间）
- 兼职/项目制：根据合同约定
```

### 3.2 负载均衡规则

#### 自动分配策略

```
分配优先级排序算法：

Score = 可用容量 × 技能匹配度 × 响应历史评分

其中：
- 可用容量 = 满载值 - 当前负载值
- 技能匹配度 = 0.5 ~ 1.5（基于技能标签匹配）
- 响应历史评分 = 0.8 ~ 1.2（基于历史协同满意度）

按Score降序排列，优先分配给高分成员
```

#### 负载均衡触发条件

| 触发条件 | 判定标准 | 处置动作 |
|----------|----------|----------|
| 单点过载 | 某成员负载>90% | 自动推荐任务转移方案 |
| 团队不均 | 标准差>20% | 生成负载均衡建议 |
| 技能瓶颈 | 特定技能过载 | 提示培训或招聘需求 |
| 临时缺口 | 多人休假 | 启动应急预案 |

#### 任务转移规则

```
任务转移评估矩阵：

可转移条件（需同时满足）：
1. 接收方负载 < 70%
2. 接收方具备相关技能标签
3. 转移后接收方负载 < 85%
4. 任务尚未进入关键阶段

不可转移任务：
- 已与客户建立直接联系
- 涉及敏感信息或特殊权限
- 剩余时间不足以完成交接
- 任务明确指定负责人
```

### 3.3 协同分配规则

#### 研究所协同分配

```
分配原则：
1. 专业匹配优先
   - 按研究领域精确匹配
   - 考虑研究深度要求

2. 负载均衡
   - 优先分配给低负载研究员
   - 避免单一研究员过度集中

3. 响应时效
   - 紧急需求（P0/P1）：30分钟内响应
   - 常规需求（P2/P3）：4小时内响应
   - 非紧急（P4）：24小时内响应

分配流程：
1. 系统自动推荐（基于上述算法）
2. 被推荐人确认承接
3. 如拒绝，转派第二位候选人
4. 如连续拒绝，升级至组长分配
```

#### 资管协同分配

```
分配原则：
1. 项目关联性
   - 优先分配给同一项目历史参与者
   - 保持项目连续性

2. 专业方向
   - 权益/固收/FOF/MOM等分类匹配
   - 策略类型匹配

3. 客户经理配对
   - 考虑历史合作默契度
   - 客户满意度历史记录

特殊规则：
- 定制化项目：指定专人负责
- 标准化服务：按轮询分配
- 紧急支持：主管指定，不受负载限制
```

#### 协同冲突处理

```
冲突场景及处理：

场景1：多人同时申请同一研究员
处理：按优先级排序，低优先级自动重新分配

场景2：研究员负载已满，新需求进入
处理：
  a) 需求方可接受延期 → 排队等待
  b) 需求方紧急 → 升级至研究所总监协调

场景3：跨部门协同优先级争议
处理：
  a) 双方主管协商
  b) 无法达成一致 → 提交共同上级裁决
```

### 3.4 绩效统计规则

#### 绩效计算公式

```
个人月度绩效分 = 
  工作量得分 × 30% +
  完成质量得分 × 25% +
  协同贡献得分 × 20% +
  客户满意得分 × 15% +
  团队协作得分 × 10%
```

#### 各维度计分规则

**工作量得分（30%）**
```
得分 = MIN(100, (实际完成工时 / 标准工时) × 100)

加分项：
- 超额完成：每超10%加5分，上限20分
- 加班支持：经批准的加班，每4小时加2分

减分项：
- 工时利用率<60%：每低10%扣10分
- 有效工时<标准工时50%：0分
```

**完成质量得分（25%）**
```
得分 = 任务完成率 × 40 + 准时交付率 × 40 + 返工率系数 × 20

其中：
- 任务完成率 = 完成任务数 / 分配任务数
- 准时交付率 = 准时完成任务数 / 完成任务数
- 返工率系数 = MAX(0, 100 - 返工任务数 × 20)
```

**协同贡献得分（20%）**
```
得分 = 协同次数系数 × 40 + 协同满意度 × 40 + 响应及时率 × 20

协同次数系数：
- 0次：0分
- 1-5次：60分
- 6-10次：80分
- 11次以上：100分

协同满意度：平均分×20

响应及时率：及时响应数/总协同请求
```

**客户满意得分（15%）**
```
得分 = 评价覆盖率系数 × 满意度平均分 × 20

评价覆盖率系数：
- <30%：0.8
- 30%-50%：0.9
- 50%-70%：1.0
- >70%：1.1

注：仅计算有效评价（服务完成后7天内）
```

**团队协作得分（10%）**
```
得分 = 同事评价平均分 × 50 + 主管评价 × 50

评价维度：
- 沟通配合度
- 知识分享
- 互助精神
- 团队贡献
```

#### 绩效等级划分

| 绩效分 | 等级 | 等级定义 | 激励措施 |
|--------|------|----------|----------|
| 95-100 | S | 卓越 | 超额奖金+晋升优先 |
| 85-94 | A | 优秀 | 标准奖金+评优资格 |
| 70-84 | B | 良好 | 标准奖金 |
| 60-69 | C | 合格 | 基础奖金 |
| <60 | D | 待改进 | 绩效改进计划 |

#### 排名展示规则

```
排名维度：
1. 全公司排名（匿名显示Top10）
2. 部门内排名（全员显示）
3. 同职级排名（匿名显示）

隐私保护：
- 排名后20%成员仅本人及上级可见具体名次
- 跨部门排名不显示姓名，仅显示部门平均
- 绩效分精确到整数，不显示小数

排名更新：
- 实时更新（每完成一项任务）
- 每日凌晨汇总统计
- 月度最终排名于次月5日锁定
```
