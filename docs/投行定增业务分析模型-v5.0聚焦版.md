## 二、业务团队动作标准模型（按角色本质设计）

> **原则**：业务团队的角色不是"执行清单"（如"打电话、发邮件、拜访客户"），而是**分析决策模型**——每个角色用模型来判断、分析、决策，模型输出指导该做什么动作。

### 2.1 客户经理动作标准模型

**本质定位**：客户经理——覆盖项目开拓、项目执行、合规把关全链条。用模型判断客户有没有需求、项目能不能做、方案怎么设计、材料是否达标、合规能不能签。

> 客户经理全周期模型：从线索到签字的完整动作标准，分为项目开拓（模型A/B/C）、项目执行（模型D/E/F）、合规把关（模型G/H）三类应用。

#### 模型A：项目线索评估模型

**本质目标**：判断这个线索有没有可能做成项目，值不值得投入时间。

**维度（4个）**：

| 维度 | 名称 | 讲人话 | 客观数据 | 输出 |
|------|------|--------|---------|------|
| 1 | **需求真实性** | 客户真的有融资需求吗？是确定要做还是随便问问？ | 客户公开表态、融资历史、资金缺口、竞争对手动态 | 需求真实性评分 |
| 2 | **时间窗口** | 客户现在做定增的时机合适吗？董事会/股东会决议了吗？ | 公司治理进程、监管申报时间、市场窗口 | 窗口判断 |
| 3 | **客户关系** | 我跟客户关键人的关系能推进到立项吗？有没有内线？ | 关系网络强度、历史互动深度、关键人信任度 | 关系评估 |
| 4 | **竞争格局** | 有多少竞争对手在抢这个项目？我们的胜算多大？ | 已知竞争机构、客户历史合作偏好、我方独特优势 | 竞争评估 |

**调用输出**：
```
输入：客户名称 + 线索来源
输出：
  ├─ 需求真实性：高/中/低 + 依据（3个数据点）
  ├─ 时间窗口：已开/即将开/未开 + 预计时间
  ├─ 客户关系：强/中/弱 + 关键人清单
  ├─ 竞争格局：激烈/一般/弱 + 竞争对手清单
  └─ 跟进建议：立即跟进/保持接触/放弃 + 具体动作
```

#### 模型B：客户触达策略模型

**本质目标**：判断怎么触达客户最有效，准备什么沟通要点和材料。

**维度（3个）**：

| 维度 | 名称 | 讲人话 | 输出 |
|------|------|--------|------|
| 1 | **客户痛点** | 客户现在最关心的问题是什么？我们怎么能帮上忙？ | 客户痛点清单 + 我方切入点 |
| 2 | **拜访人员** | 派谁去拜访客户最合适？级别够不够？专业对不对口？ | 推荐拜访人 + 备选方案 |
| 3 | **展业材料** | 带什么材料去？案例？数据？方案框架？ | 材料清单 + 核心沟通要点 |

**调用输出**：
```
输入：客户名称 + 项目线索评估结果
输出：
  ├─ 切入点：客户当前最关注的3个问题 + 我方对应能力
  ├─ 拜访策略：推荐拜访人/时间/地点/形式
  ├─ 展业材料：必带材料清单（案例/数据/方案框架）
  └─ 沟通要点：3句话讲清我方优势 + 预期客户反应 + 应对策略
```

#### 模型C：项目争取策略模型

**本质目标**：判断怎么在竞争中争取到这个项目，我们的差异化优势是什么。

**维度（3个）**：

| 维度 | 名称 | 讲人话 | 输出 |
|------|------|--------|------|
| 1 | **竞争优势** | 我们相比竞争对手的优势在哪里？客户认不认？ | 优势清单 + 客户认可度评估 |
| 2 | **费率方案** | 我们报的承销费率、服务方案有没有竞争力？ | 定价竞争力分析 + 调整建议 |
| 3 | **客户绑定** | 除了业务关系，有没有更深层的绑定？（如研究服务、投资建议） | 关系深化策略 + 增值服务方案 |

---

#### 模型D：项目可行性分析模型

**本质目标**：判断这个定增项目在技术上、合规上、市场上是否可行，关键障碍是什么。

**维度（5个）**：

| 维度 | 名称 | 讲人话 | 客观数据 | 输出 |
|------|------|--------|---------|------|
| 1 | **融资规模** | 打算融多少钱？相对于市值占比多少？市场能不能消化？ | 拟募集资金/总市值/同产业可比案例 | 规模合理性判断 |
| 2 | **定价方案** | 定价基准怎么选？折扣空间有多少？对现有股东稀释多少？ | 定价基准日/折扣率/股本稀释比例 | 定价建议区间 |
| 3 | **市场时机** | 现在发是不是好时候？监管环境松还是紧？市场窗口开没开？ | 近期监管政策/市场发行成功率/同产业案例 | 窗口判断 |
| 4 | **资金用途** | 募投项目有没有明确用途？钱怎么花？能不能讲清楚故事？ | 募投项目说明书/可行性分析/产业前景 | 募资可行性评估 |
| 5 | **合规风险** | 客户历史上有没有违规？现在有没有被调查？有没有潜在的法律风险？ | 监管处罚记录/诉讼记录/合规历史 | 合规风险评估 |

**调用输出**：
```
输入：客户名称 + 拟发行规模 + 初步定价方案
输出：
  ├─ 融资规模：合理性判断 + 可比案例（3-5个）
  ├─ 价格区间：建议区间 + 稀释影响分析
  ├─ 时机判断：当前窗口判断 + 建议申报时间
  ├─ 募资评估：项目可行性 + 故事完整性
  ├─ 合规风险：[通过]无重大风险 / [注意]需关注（具体风险点）
  └─ 综合判断：可行/谨慎推进/建议暂缓 + 关键障碍清单
```

#### 模型E：方案设计决策模型

**本质目标**：判断定增方案怎么设计最合理，发行方式、定价策略、配售安排怎么定。

**维度（4个）**：

| 维度 | 名称 | 讲人话 | 输出 |
|------|------|--------|------|
| 1 | **发行方式** | 定价发行还是竞价发行？锁定期多长？面向谁发？ | 发行方式建议 + 锁定期建议 + 发行对象类型 |
| 2 | **融资方案** | 募资金额定多少？定价区间怎么设？对股本稀释多少？ | 金额区间/价格区间/稀释比例 |
| 3 | **资金用途** | 募投项目怎么包装？钱怎么分配？预期回报怎么讲？ | 项目清单/资金分配/回报测算 |
| 4 | **风险预案** | 发行失败的风险怎么防？股价波动的风险怎么控？ | 风险预案 + 应对措施 |

#### 模型F：材料制作质量模型

**本质目标**：判断募集说明书等关键材料的质量是否达标，能不能申报。

**维度（4个）**：

| 维度 | 名称 | 讲人话 | 输出 |
|------|------|--------|------|
| 1 | **数据质量** | 财务数据、业务数据、市场数据引用正确吗？来源可靠吗？ | 数据校验清单 |
| 2 | **规律质量** | 章节之间规律连贯吗？前后一致吗？论证充分吗？ | 规律检查清单 |
| 3 | **风险披露** | 风险因素说全面了吗？应对措施讲清楚了吗？ | 风险完整性检查 |
| 4 | **格式规范** | 格式符合监管要求吗？披露充分吗？有没有遗漏？ | 格式合规检查 |

---

#### 模型G：签字风险评估模型

**本质目标**：判断这个项目我能不能签字，签完字后有多大风险，需要哪些前置条件。

**维度（4个）**：

| 维度 | 名称 | 讲人话 | 客观数据 | 输出 |
|------|------|--------|---------|------|
| 1 | **尽调质量** | 尽职调查做充分了吗？有没有遗漏？底稿齐不齐？ | 尽调覆盖率/底稿完整性/访谈记录 | 尽调质量评估 |
| 2 | **信息披露** | 信息披露充分吗？有没有隐瞒？风险说清了吗？ | 披露完整性/风险因素充分性 | 披露质量评估 |
| 3 | **利益冲突** | 我跟这个项目有没有利益冲突？发行人跟我有没有关联？ | 利益冲突排查/关联交易审查 | 利益冲突判断 |
| 4 | **风险可控性** | 项目最大的风险是什么？我签字后可能面临什么风险？ | 风险清单/风险等级/应对措施 | 签字风险判断 |

**调用输出**：
```
输入：项目全套材料 + 尽调底稿 + 利益冲突排查结果
输出：
  ├─ 尽调到位：[通过]/[不通过] + 遗漏清单
  ├─ 披露全面：[通过]/[不通过] + 缺失清单
  ├─ 利益冲突：[通过]无冲突 / [注意]需说明（具体说明）
  ├─ 风险可控：高/中/低 + 最大风险点
  └─ 签字建议：可以签/谨慎签/不能签 + 前置条件清单
```

#### 模型H：合规把关决策模型

**本质目标**：判断项目在合规层面有没有硬伤，监管会不会否，签字后会不会被追责。

**维度（4个）**：

| 维度 | 名称 | 讲人话 | 输出 |
|------|------|--------|------|
| 1 | **申报条件** | 客户符合定增的法定条件吗？盈利能力？合规记录？ | 法定条件检查清单 |
| 2 | **决策程序** | 内部决策程序走完了吗？董事会/股东会决议了吗？ | 程序合规检查清单 |
| 3 | **监管态度** | 监管近期对这种类型的项目态度怎么样？有没有被否的案例？ | 监管态度判断 |
| 4 | **追责风险** | 如果项目出问题，我被追责的可能性有多大？ | 追责风险评估 |

## 三、业务支持中心动作标准模型（同步响应）

### 3.1 组织者角色

#### 模型I：投行经营分析模型

**本质目标**：判断定增业务整体健康度。

**维度（4个）**：业务负荷/执行效率/项目质量/盈利能力 → 输出经营健康度判断

#### 模型J：重大风险处置模型

**本质目标**：判断风险等级与处置策略。

**维度（3个）**：风险等级/责任分配/处置策略 → 输出风险定级+处置方案

#### 模型K：资源调配决策模型

**本质目标**：判断资源怎么分配最合理。

**维度（3个）**：资源匹配/优先级排序/调配方案 → 输出调配方案

### 3.2 质控内核角色

#### 模型L：项目质量分析模型

**本质目标**：生成质控报告（A/B/C/D评级）。

**维度（4个）**：材料完整性/规律质量/风险可控性/标准达标 → 输出质控报告

#### 模型M：合规趋势研判模型

**本质目标**：判断监管风向。

**维度（3个）**：政策环境/市场案例/趋势变化 → 输出合规趋势判断

#### 模型N：交叉验证执行模型

**本质目标**：独立验证关键产出。

**维度（3个）**：数据来源/分析方法/结论可靠性 → 输出验证报告

### 3.3 产品支持角色

#### 模型O：政策支持分析模型

**本质目标**：分析政策环境，指导方案设计。

**维度（3个）**：政策环境/市场案例/我们该怎么应对 → 输出政策判断+方案建议

#### 模型P：监管对接策略模型

**本质目标**：设计监管沟通策略。

**维度（3个）**：跟谁对接/什么时候/怎么沟通 → 输出对接策略+沟通方案

#### 模型Q：方案交付评估模型

**本质目标**：判断方案可执行性。

**维度（3个）**：资源匹配/时间可行性/风险可控性 → 输出可执行性评估

## 四、聚焦：投行定增业务分析标准模型（重点完善）

> **原则**：这是业务标准模型中最核心的一个，先把它做深做透。业务团队（客户经理）和业务支持中心（组织者/质控内核/产品支持）都会调用或参考这个模型。

### 4.1 模型定位

**模型名称**：投行定增业务分析标准模型  
**模型类型**：业务标准模型（不是动作标准模型）  
**目标角色**：
- **主要调用者**：客户经理（判断项目可行性）
- **参考使用者**：客户经理（判断值不值得跟进、能不能签）、组织者（判断资源分配）、质控内核（判断合规风险）、产品支持（判断方案设计）
**核心诉求**：这个定增项目从业务角度看，能不能做？关键参数是什么？风险在哪？

### 4.2 本质维度（5个）

> **硬性标准**：5个维度，不能再多。每个维度讲人话，有客观数据支撑。

| # | 维度名称 | 讲人话 | 客观数据来源 | 谁最关心 |
|---|---------|--------|------------|---------|
| 1 | **融资规模** | 打算融多少钱？相对于市值占比多少？市场能不能消化？会不会因为规模太大发不出去？ | 拟募集资金金额 / 总市值 / 同产业可比案例发行规模 / 市场承接能力 | 客户经理、组织者、产品支持 |
| 2 | **定价方案** | 定价基准怎么选？折扣空间有多少？对现有股东稀释多少？价格定高了没人买，定低了股东骂 | 定价基准日 / 折扣率（通常8折）/ 股本稀释比例 / 同产业可比定价 | 客户经理、产品支持 |
| 3 | **市场时机** | 现在发是不是好时候？监管环境松还是紧？市场窗口开没开？错过了可能要等半年 | 近期监管政策 / 市场发行成功率 / 同产业案例发行时间 / 公司自身经营周期 | 客户经理、组织者 |
| 4 | **资金用途** | 募投项目有没有明确用途？钱怎么花？能不能讲清楚故事？监管和投资者信不信？ | 募投项目说明书 / 可行性分析报告 / 产业前景数据 / 公司历史募投执行率 | 客户经理、质控内核 |
| 5 | **合规风险** | 客户历史上有没有违规？现在有没有被调查？有没有潜在的法律风险？签完字会不会被追责？ | 监管处罚记录 / 诉讼记录 / 合规历史 / 财务异常指标 | 客户经理、质控内核 |

### 4.3 维度详解

#### 维度1：融资规模

**分析框架**：
```
【绝对规模】
  ├─ 拟募集资金金额：XX亿元
  ├─ 公司当前总市值：XX亿元
  └─ 募集资金/总市值比例：XX%（警戒阈值：>30%需谨慎）

【相对规模】
  ├─ 同产业可比案例平均融资规模：XX亿元
  ├─ 同产业可比案例平均募资/市值比例：XX%
  └─ 我方项目与产业平均对比：偏高/持平/偏低

【市场承接】
  ├─ 当前市场日均成交额：XX亿元
  ├─ 同产业近期定增平均认购倍数：XX倍
  └─ 预计我方项目认购倍数：XX倍（乐观/中性/悲观）

【判断标准】
  ├─ [通过] 合理：规模适中，市场承接力充足，同产业有成功案例
  ├─ [注意] 需谨慎：规模偏大，需关注认购风险，建议引入战略投资者
  └─ [不通过] 不合理：规模过大，市场难以承接，建议缩减规模或分批发行
```

#### 维度2：定价方案

**分析框架**：
```
【定价基准】
  ├─ 可选基准：董事会决议公告日 / 股东大会决议公告日 / 发行期首日
  ├─ 监管要求：定价基准日前20/60/120个交易日均价
  └─ 建议基准：XX（考虑市场窗口+监管偏好）

【折扣空间】
  ├─ 监管允许折扣：不低于定价基准的80%
  ├─ 市场常见折扣：85%-90%
  ├─ 建议折扣：XX%（考虑市场环境+投资者接受度）
  └─ 与同产业的对比：我方折扣 vs 产业平均

【稀释影响】
  ├─ 发行前总股本：XX亿股
  ├─ 预计发行股数：XX亿股
  ├─ 股本稀释比例：XX%（警戒阈值：>25%需谨慎）
  └─ 对每股收益的影响：摊薄XX%

【判断标准】
  ├─ [通过] 合理：定价有吸引力，稀释可控，同产业可接受
  ├─ [注意] 需谨慎：定价偏高或稀释偏大，需调整发行结构
  └─ [不通过] 不合理：定价无吸引力或稀释过大，建议重新设计
```

#### 维度3：市场时机

**分析框架**：
```
【监管环境】
  ├─ 近期定增政策：宽松/中性/收紧
  ├─ 近期定增审核通过率：XX%（产业平均）
  ├─ 同类项目近期审核结果：通过/有条件通过/被否
  └─ 监管关注重点：XX（如资金用途真实性、定价合理性）

【市场环境】
  ├─ 当前大盘指数位置：高位/中位/低位
  ├─ 同产业板块近期走势：上涨/震荡/下跌
  ├─ 近期定增市场活跃度：活跃/一般/冷淡
  └─ 预计发行时市场环境：乐观/中性/悲观

【公司周期】
  ├─ 公司当前经营周期：旺季/淡季/转型期
  ├─ 公司近期重大事项：业绩发布/重大合同/股权变动
  ├─ 公司内部决策进度：董事会已通过/待股东会/待监管申报
  └─ 最佳发行时间窗口：XX月（综合监管+市场+公司周期）

【判断标准】
  ├─ [通过] 合适：监管宽松，市场活跃，公司内部就绪
  ├─ [注意] 需谨慎：某一项不利，需等待或调整策略
  └─ [不通过] 不合适：多项不利，建议推迟或改期
```

#### 维度4：资金用途

**分析框架**：
```
【项目必要性】
  ├─ 项目是否符合公司战略：是/否/部分
  ├─ 项目是否解决公司痛点：是/否
  ├─ 不做这个项目公司会怎样：影响重大/影响一般/无影响
  └─ 项目紧迫性：紧急/计划内/可推迟

【项目可行性】
  ├─ 技术可行性：成熟技术/需要研发/技术风险高
  ├─ 市场可行性：市场需求明确/需培育/不确定
  ├─ 财务可行性：投资回报率>XX%/盈亏平衡期<XX年
  └─ 执行可行性：公司有执行能力/需外部合作/执行风险高

【资金使用】
  ├─ 资金分配清晰度：清晰/较清晰/模糊
  ├─ 各子项目资金占比：XX%/XX%/XX%
  ├─ 补充流动资金占比：<30%[通过] / >30%[注意]（监管关注）
  └─ 与历史募投对比：方向一致/有调整/完全新方向

【故事完整性】
  ├─ 投资者听得懂吗：通俗易懂/较专业/晦涩难懂
  ├─ 规律链条完整吗：完整/有缺失/规律跳跃
  ├─ 数据支撑充分吗：充分/部分/不足
  └─ 与同产业案例对比：优于平均/持平/弱于平均

【判断标准】
  ├─ [通过] 靠谱：必要性明确，可行性高，资金分配清晰，故事完整
  ├─ [注意] 需完善：某一项有瑕疵，需补充论证或调整方案
  └─ [不通过] 不靠谱：多项存疑，需重新设计资金用途
```

#### 维度5：合规风险

**分析框架**：
```
【历史合规】
  ├─ 近3年监管处罚记录：有/无（如有，次数、类型、严重程度）
  ├─ 近3年诉讼记录：有/无（如有，金额、类型、结果）
  ├─ 近3年财务重述记录：有/无
  └─ 历史合规综合评价：良好/一般/较差

【当前状态】
  ├─ 是否正在被调查：是/否（如有，调查类型、进展）
  ├─ 是否存在未决诉讼：是/否（如有，金额、影响）
  ├─ 是否存在重大合同违约：是/否
  └─ 是否存在关联交易未披露：是/否

【财务健康】
  ├─ 近3年审计意见：标准无保留/带强调事项/保留意见
  ├─ 近3年净利润：持续增长/波动/下滑
  ├─ 近3年经营现金流：持续为正/波动/持续为负
  └─ 资产负债率：XX%（产业对比：高于/持平/低于平均）

【定增特有风险】
  ├─ 是否存在同业竞争：是/否（如有，解决方案）
  ├─ 是否存在重大资产权属瑕疵：是/否
  ├─ 是否存在环保/安全等重大违规：是/否
  └─ 实控人是否存在股权质押风险：是/否（质押比例XX%）

【判断标准】
  ├─ [通过] 风险小：历史合规良好，当前无重大风险，财务健康
  ├─ [注意] 需关注：某一项有风险，需专项说明或采取措施
  └─ [不通过] 风险大：多项风险，建议暂缓或终止
```

### 4.5 模型调用方式

**业务团队调用**：
```
客户经理：
  输入：客户名称 + 初步线索
  调用：项目线索评估模型 → 判断值不值得跟进
  参考：定增业务分析模型（维度1/2/3初步判断）

客户经理：
  输入：客户名称 + 拟融资规模 + 初步方案
  调用：项目可行性分析模型 + 方案设计决策模型
  核心调用：定增业务分析标准模型（5维度完整分析）

客户经理：
  输入：项目全套材料
  调用：签字风险评估模型 + 合规把关决策模型
  参考：定增业务分析标准模型（维度5合规风险）
```

**业务支持中心调用**：
```
组织者：
  参考：定增业务分析标准模型（维度1/2/3，判断资源分配）

质控内核：
  核心调用：项目质量分析模型 + 交叉验证执行模型
  参考：定增业务分析标准模型（维度4/5，判断募投和合规）

产品支持：
  核心调用：政策支持分析模型 + 方案交付评估模型
  参考：定增业务分析标准模型（维度1/2/3，设计方案）
```

## 五、同步响应机制（所有角色共享模型）

### 5.1 统一底座

```
客户分析底座（身份证/账户/关系网络/社会分层）
    ↓
【同步触发】定增业务分析标准模型（5维度）
    ↓
【同步输出】所有角色同步看到分析结果
    ├─ 客户经理：看到"市场时机"→ 判断跟进策略
    ├─ 客户经理：看到"规模/定价/募资"→ 设计方案
    ├─ 客户经理：看到"合规风险"→ 判断能不能签
    ├─ 组织者：看到"时机/规模"→ 判断资源分配
    ├─ 质控内核：看到"募资/合规"→ 预判风险
    └─ 产品支持：看到"规模/定价"→ 设计对接策略
```

### 5.2 事件驱动

```
事件：客户经理输入"XX科技，拟定增5亿"
    ↓
【同步触发】
    ├─ 客户经理：触发项目线索评估模型（已跟进中）
    ├─ 客户经理：触发项目可行性分析模型 + 定增业务分析标准模型
    ├─ 客户经理：触发合规风险预判（初步）
    ├─ 组织者：触发经营分析（项目资源影响）
    ├─ 质控内核：触发合规趋势研判（政策环境）
    └─ 产品支持：触发政策分析（方案方向）
    ↓
【同步汇聚】
    ├─ 客户经理：5维度分析报告
    ├─ 客户经理：合规风险初步判断
    ├─ 组织者：资源调配建议
    ├─ 质控内核：政策环境判断
    └─ 产品支持：政策利好/利空判断
    ↓
【实时反馈】所有角色看到统一视图，各自决策
```
