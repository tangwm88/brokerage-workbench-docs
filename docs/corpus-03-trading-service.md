> 本文档为交易服务模块提供完整的AI辅助语料、数据需求定义和使用规则规范。

## 一、参考语料

### 1.1 交易咨询AI提示词

#### 系统角色设定
```
你是一位专业的量化交易顾问，精通TWAP/VWAP算法交易、订单执行策略、
风险管理和市场微观结构分析。你的职责是帮助用户理解交易执行过程、
优化订单策略、解读成交分析报告。

核心能力：
- 算法交易策略咨询（TWAP/VWAP/POV/Iceberg）
- 订单执行质量分析（滑点、成交率、市场冲击）
- 异常交易识别与处理建议
- 实时交易状态查询与解释

回答原则：
1. 专业准确：使用标准金融术语，数据精确
2. 通俗易懂：复杂概念用通俗语言解释
3. 风险提示：涉及交易风险时主动提示
4. 数据驱动：基于实际数据给出分析和建议
```

#### 策略咨询提示词模板

**TWAP策略咨询**
```
用户询问TWAP策略参数设置时，按以下框架回答：

【参数解释】
- 时间窗口：建议根据订单量和市场流动性设定，通常为15-60分钟
- 切片数量：总订单量 ÷ 切片数量 = 单笔交易量
- 执行间隔：时间窗口 ÷ 切片数量

【设置建议】
- 大盘股/高流动性：可适当增大单笔交易量，缩短执行时间
- 小盘股/低流动性：减小单笔交易量，延长执行时间，降低市场冲击
- 波动行情：增加切片数量，降低单次交易风险

【示例】
"假设您要买入100万股某大盘股，建议：
- 时间窗口：30分钟
- 切片数量：20笔
- 单笔交易量：5万股
- 执行间隔：90秒"
```

**订单执行时机咨询**
```
用户询问最佳执行时机时：

【分析维度】
1. 市场流动性：查看买卖盘口深度、换手率
2. 波动率状态：近期价格波动幅度
3. 市场情绪：大盘走势、板块热点
4. 时间因素：开盘/收盘/午盘特点

【建议模板】
"根据当前市场数据，建议：
- 开盘后30分钟：流动性好但波动大，适合小单试探
- 盘中平稳期：波动较小，适合大单分批执行
- 收盘前15分钟：流动性下降，建议减少订单量"
```

## 三、使用规则

### 3.1 风控规则（10种异常类型）

#### 规则1：大单交易监控
```
规则ID：RISK-001
规则名称：大单交易预警
触发条件：
  - 单笔委托数量 ≥ 50万股
  - 或 单笔委托金额 ≥ 500万元
  - 或 占该标的流通盘比例 ≥ 0.5%

告警级别：中级
处理方式：
  1. 系统自动记录并生成告警
  2. 通知风控人员人工复核
  3. 超过100万股需审批后方可继续

例外情况：
  - 指数成分股可放宽至100万股
  - ETF套利交易除外
```

#### 规则2：频繁撤单监控
```
规则ID：RISK-002
规则名称：频繁撤单预警
触发条件：
  - 统计窗口：1小时
  - 撤单率 ≥ 30%（撤单数/委托数）
  - 或 连续撤单次数 ≥ 5次
  - 或 1小时内撤单次数 ≥ 20次

告警级别：高级
处理方式：
  1. 生成告警并通知交易员
  2. 撤单率超过50%自动限制新委托（冷却期15分钟）
  3. 记录到行为审计日志

风险说明：
  - 交易所可能对高频撤单采取监管措施
  - 可能影响市场流动性判断
```

#### 规则3：价格偏离监控
```
规则ID：RISK-003
规则名称：价格偏离预警
触发条件：
  - 买入委托价格 ≥ 最新成交价 × 1.05
  - 或 卖出委托价格 ≤ 最新成交价 × 0.95
  - 或 委托价格偏离前收盘价 ± 10%

告警级别：中级
处理方式：
  1. 生成价格偏离告警
  2. 提示交易员确认价格
  3. 偏离超过15%需二次确认
  4. 超过涨跌停限制自动拒绝
```

#### 规则4：持仓集中度监控
```
规则ID：RISK-004
规则名称：持仓集中度预警
触发条件：
  - 单一标持仓市值 / 总持仓市值 ≥ 50%
  - 或 单一行业持仓占比 ≥ 60%
  - 或 前5大重仓股占比 ≥ 80%

告警级别：高级
处理方式：
  1. 生成集中度告警
  2. 限制该标的继续买入
  3. 建议分散投资提醒
```

#### 规则5：日内回转监控
```
规则ID：RISK-005
规则名称：异常回转交易预警
触发条件：
  - T日买入后T日卖出（T+1市场）
  - 或 同一标的同日买卖次数 ≥ 5次
  - 或 回转交易金额 ≥ 当日成交额50%

告警级别：中级
处理方式：
  1. 标记为日内回转交易
  2. 生成合规性检查报告
  3. 异常情况上报合规部门
```

#### 规则6：波动率异常监控
```
规则ID：RISK-006
规则名称：波动率异常预警
触发条件：
  - 标的价格5分钟内波动 ≥ 5%
  - 或 标的换手率1小时内 ≥ 20%
  - 或 盘口价差突然扩大3倍以上

告警级别：高级
处理方式：
  1. 暂停该标的的新算法订单
  2. 正在执行的TWAP/VWAP降低执行速度
  3. 通知交易员市场异常
  4. 建议启用保守策略参数
```

#### 规则7：流动性风险监控
```
规则ID：RISK-007
规则名称：流动性不足预警
触发条件：
  - 订单金额 > 盘口深度 × 3
  - 或 买卖价差 > 1%
  - 或 近5分钟成交量 < 订单量 × 10%

告警级别：中级
处理方式：
  1. 提示流动性风险
  2. 建议延长执行时间窗口
  3. 或建议减小单笔成交量
  4. 流动性极差时暂停执行
```

#### 规则8：交易限额监控
```
规则ID：RISK-008
规则名称：交易限额预警
触发条件：
  - 当日累计买入 ≥ 日买入限额 × 80%
  - 或 当日累计卖出 ≥ 日卖出限额 × 80%
  - 或 单笔委托 ≥ 单笔限额

告警级别：中级/高级（超限100%为高级）
处理方式：
  1. 80%阈值：生成预警，提示剩余额度
  2. 100%阈值：拒绝超限委托
  3. 记录到限额使用日志
```

#### 规则9：黑名单监控
```
规则ID：RISK-009
规则名称：黑名单交易拦截
触发条件：
  - 委托标的在系统黑名单中
  - 或 标的处于ST/*ST状态且策略未授权
  - 或 标的处于停牌/涨跌停无法交易状态

告警级别：严重
处理方式：
  1. 直接拒绝委托
  2. 记录违规尝试日志
  3. 通知合规管理人员
  4. 疑似违规操作上报
```

#### 规则10：异常行为模式监控
```
规则ID：RISK-010
规则名称：异常交易行为检测
触发条件（满足任意2项）：
  - 同一秒发送 ≥ 10笔委托
  - 或 委托价格呈现规律性变化（疑似操纵）
  - 或 多账户协同交易同一标的
  - 或 交易时间与重大消息发布高度吻合

告警级别：严重
处理方式：
  1. 立即标记并暂停相关账户交易
  2. 冻结未成交委托
  3. 生成详细行为分析报告
  4. 上报合规与法务部门
  5. 配合监管调查
```

### 3.3 告警升级规则

```
【时间维度升级】

示例：
中级告警 ──1h──► 高级告警 ──4h──► 严重告警 ──24h──► 系统强制处理并通知管理层
```

#### 升级规则表

|---------|-------------|-------------|-------------|-----------|

| 中级(MEDIUM) | 1小时 | 4小时 | 24小时 | 中级→高级→严重 |

```
  └─ 告警面板显示

第二级：交易员通知（初始级别中级以上立即触发）
  └─ 交易终端弹窗
  └─ 短信通知

第三级：风控经理通知（升级后触发）
  └─ 电话通知

第四级：管理层通知（严重级别或强制处理）
```

### 3.4 交易限额规则

#### 限额层级结构

```
【四层限额体系】

第一层：系统级限额（全平台）
  ├─ 系统日成交额上限
  ├─ 系统单标的上限
  └─ 系统并发订单上限

第二层：机构级限额（客户/产品）
  ├─ 机构日交易限额
   ├─ 买入限额：[金额]元/日
  ├─ 卖出限额：[金额]元/日
  └─ 净交易限额：[金额]元/日

第三层：账户级限额（交易账户）
  ├─ 账户日买入限额
  ├─ 账户日卖出限额
  ├─ 单笔交易限额
  └─ 持仓市值限额

第四层：策略级限额（算法策略）
  ├─ 策略单订单限额
  ├─ 策略日累计限额
  └─ 策略同时运行数限制
```

#### 限额检查时机

| 检查点 | 检查内容 | 超限处理 |
|-------|---------|---------|
| 订单创建 | 单笔限额、策略限额 | 直接拒绝 |
| 风控审核 | 账户限额、机构限额 | 待审批/拒绝 |
| 切片执行前 | 实时可用额度 | 暂停/拆分 |
| 成交回报后 | 累计限额更新 | 限制新订单 |
| 日终结算 | 日限额使用统计 | 生成报告 |

#### 限额超限处理

```
【80%预警处理】
触发条件：已用额度 ≥ 限额 × 80%
处理动作：
  1. 发送预警通知
  2. 在交易界面显示剩余额度
  3. 建议用户合理安排剩余交易

【95%临界处理】
触发条件：已用额度 ≥ 限额 × 95%
处理动作：
  1. 强制弹窗提醒
  2. 限制大额委托
  3. 要求确认后继续
  4. 通知风控人员

【100%超限处理】
触发条件：已用额度 ≥ 限额
处理动作：
  1. 拒绝新委托
  2. 正在执行的TWAP/VWAP暂停新增切片
  3. 已有成交正常结算
  4. 生成超限报告

【超限申请】
如需超限交易：
  1. 提交超限申请（含理由、金额、时限）
  2. 风控经理审批
  3. 临时提升限额（仅限当日有效）
  4. 超限交易完成后恢复原限额
```

#### 限额恢复机制

| 限额类型 | 恢复周期 | 恢复时间点 |
|---------|---------|-----------|
| 日买入限额 | 每日 | 开盘前（09:15）|
| 日卖出限额 | 每日 | 开盘前（09:15）|
| 单笔限额 | 实时 | 成交/撤单后即时恢复 |
| 持仓限额 | 实时 | 卖出成交后即时恢复 |
| 临时提升限额 | 当日 | 日终自动失效 |
