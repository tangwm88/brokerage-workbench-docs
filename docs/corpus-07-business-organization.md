# 业务组织模块 - 参考语料与使用规则

## 模块概述

业务组织模块是金融/证券业务系统的核心运营模块，负责前端客户的资质审核、开户流程管理、分支机构业务跟踪以及系统开通后的反馈管理。

---

## 一、参考语料

### 1.1 客户审核标准话术

#### 1.1.1 客户资质初审话术

**开场白：**
```
您好，[客户姓名]先生/女士，我是[公司名称]客户服务专员[工号]，
现就您提交的开户申请进行资质初审确认。
本次通话将核对您的身份信息及资质材料，预计需要3-5分钟，感谢您的配合。
```

**身份核验话术：**
```
首先进行身份核验，请您提供：
1. 身份证号码后四位
2. 开户申请时预留的手机号码
3. 工作单位及职务

请问以上信息与您提交的资料是否一致？
```

**资质询问话术：**
```
接下来核实您的投资资质：
1. 请问您是否具备[X年]以上的证券投资经验？
2. 您的风险承受能力评级为[保守型/稳健型/积极型]，是否正确？
3. 您的金融资产是否达到[X万元]以上？

如有任何疑问，我可以为您详细解释相关标准。
```

**审核结果通知话术：**

- **通过：**
```
恭喜您，您的资质审核已通过。后续开户流程将由专属客户经理[姓名]为您服务，
预计将在[时间]内与您联系。如有任何问题，欢迎拨打客服热线[电话]。
```

- **需补充材料：**
```
您的资质审核需要补充以下材料：
[列出缺失材料清单]
请您在[时限]内通过[渠道]提交，我们将优先为您处理。
```

- **不通过：**
```
感谢您的申请。经审核，您目前暂不符合开户资质要求，主要原因：
[具体原因说明]
如您有疑问，可申请人工复核或咨询客户经理了解详细情况。
```

#### 1.1.2 客户信息确认话术

**基本信息确认：**
```
为确保账户安全，请您确认以下信息：
□ 姓名：[XXX]
□ 证件类型及号码：[身份证 XXXXXXXXXXXXXXXX]

□ 紧急联系人：[姓名 电话]

以上信息是否有变更？如有变更请说明，我将为您更新。
```

**投资偏好确认：**
```
请确认您的投资偏好设置：
- 投资品种偏好：[股票/基金/债券/衍生品/其他]
- 投资期限偏好：[短期/中期/长期]
- 单笔投资金额区间：[X万元 - X万元]

```

## 三、使用规则

### 3.1 客户审核规则

#### 3.1.1 资质审核标准

**基本准入条件：**

| 条件项 | 标准要求 | 核验方式 | 备注 |
|-------|---------|---------|------|
| 年龄要求 | 18-70周岁 | 证件信息 | 65岁以上需额外评估 |
| 身份真实性 | 证件真实有效 | 联网核查 | 人脸识别比对 |
| 风险测评 | 完成有效测评 | 系统记录 | 测评有效期2年 |
| 资产要求 | 符合产品门槛 | 资产证明 | 视业务类型而定 |
| 投资经验 | 符合产品要求 | 客户声明 | 需辅助证明材料 |

**分级审核标准：**

```
┌─────────────────────────────────────────────────────────┐
│                    客户分级审核流程                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐ │
│  │  普通客户   │───→│  自动审核   │───→│  快速通过   │ │
│  │  (标准业务) │    │  (系统校验) │    │  (即时开通) │ │
│  └─────────────┘    └─────────────┘    └─────────────┘ │
│         │                                               │
│         ↘ 不符合自动审核条件                            │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐ │
│  │  标准客户   │───→│  人工初审   │───→│  人工复审   │ │
│  │  (常规业务) │    │  (资质核实) │    │  (风险把关) │ │
│  └─────────────┘    └─────────────┘    └─────────────┘ │
│         │                                               │
│         ↘ 高风险特征识别                                │

│  │ (复杂业务)  │    │  (实地核查) │    │  (集体决策) │ │
│  └─────────────┘    └─────────────┘    └─────────────┘ │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**高风险客户识别规则：**

触发以下任一条件需升级为人工重点审核：
- 年龄 ≥ 65岁 或 < 22岁
- 风险测评结果为"保守型"但申请高风险业务
- 资产规模与收入明显不匹配
- 短期内频繁变更关键信息
- 涉及制裁名单或敏感地区
- 职业为敏感行业（如政府官员、军人等）
- 联系方式异常（如虚拟运营商号段）

#### 3.1.2 审核操作规范

**审核步骤：**

1. **资料完整性检查**
   - 核对申请材料是否齐全
   - 检查影像资料清晰度
   - 确认签字/盖章完整性

2. **信息一致性核验**
   - 证件信息与申请表一致性
   - 联网核查身份信息
   - 验证联系方式真实性

3. **资质符合性审查**
   - 对照业务准入标准
   - 评估风险匹配度
   - 识别异常或可疑情况

4. **审核结论记录**
   - 明确审核意见
   - 记录审核依据
   - 如驳回需说明原因

**禁止行为：**
- ❌ 未经核实即通过审核
- ❌ 擅自降低审核标准
- ❌ 代替客户填写或修改信息
- ❌ 泄露客户审核信息
- ❌ 与审核对象存在利益冲突时不回避

### 3.2 任务分配规则

#### 3.2.1 负载均衡算法

**分配原则：**

```
任务分配得分 = 基础权重 × 能力系数 × 负载系数 × 时效系数

其中：
- 基础权重：根据岗位配置设定（默认1.0）
- 能力系数：根据历史效率调整（0.8-1.2）
- 负载系数：根据当前待办量调整（反比）
- 时效系数：根据预计完成时间调整
```

**负载均衡策略：**

| 策略类型 | 适用场景 | 说明 |
|---------|---------|------|
| **轮询分配** | 标准任务 | 按顺序循环分配 |
| **能力优先** | 紧急/复杂任务 | 分配给处理效率最高的员工 |
| **负载优先** | 日常任务均衡 | 优先分配给待办量最少的员工 |
| **专业匹配** | 特殊业务 | 根据业务类型匹配专业审核员 |
| **就近分配** | 需线下配合 | 优先分配给客户所在区域人员 |

**负载均衡触发条件：**
- 单个审核员待办量超过平均值 150% 时，暂停新任务分配
- 单个审核员待办量低于平均值 50% 时，优先分配新任务
- 连续 3 次超时未完成，降低该审核员任务优先级

#### 3.2.2 任务优先级规则

**优先级定义：**

| 优先级 | 判定条件 | 处理时限 | 分配策略 |
|-------|---------|---------|---------|
| 🔴 紧急 | VIP客户/监管时限/投诉升级 | 2小时 | 立即分配给最优资源 |
| 🟡 高 | 普通VIP/接近时限/重点客户 | 4小时 | 优先分配 |
| 🟢 中 | 常规业务/正常时限 | 24小时 | 正常分配 |
| ⚪ 低 | 批量处理/非紧急 | 72小时 | 空闲时分配 |

**优先级动态调整：**
- 任务等待时间超过50%时限，自动提升一级优先级
- 客户催促一次，优先级提升一级（最高到高优先级）
- 关联投诉升级，立即设为紧急优先级

#### 3.2.3 任务转派规则

**允许转派场景：**
- 审核员请假/休假
- 专业能力不匹配
- 存在利益冲突
- 工作量严重失衡（经督导批准）

**转派限制：**
- 单个任务最多转派 2 次
- 转派必须记录原因
- 已处理超过50%的任务不得转派
- 紧急任务转派需经主管批准

### 3.3 审核时限规则

#### 3.3.1 标准时限要求

| 业务类型 | 受理时限 | 审核时限 | 总时限 | 说明 |
|---------|---------|---------|-------|------|
| 普通客户开户 | 实时 | 4小时 | 4小时 | 工作日 |
| VIP客户开户 | 实时 | 2小时 | 2小时 | 工作日 |
| 普通资质审核 | 1小时 | 4小时 | 5小时 | 工作日 |
| 复杂资质审核 | 2小时 | 8小时 | 10小时 | 工作日 |
| 系统权限开通 | 实时 | 2小时 | 2小时 | 7×24小时 |
| 投诉问题处理 | 30分钟 | 4小时 | 4.5小时 | 7×24小时 |

#### 3.3.2 时限计算规则

**工作时间定义：**
- 工作日：9:00 - 17:00
- 非工作时间：17:00 - 次日9:00 + 周末 + 法定节假日
- 时效计算：仅在工作时间内累计

**时限计算示例：**
```
任务提交时间：周五 16:00
业务类型：普通客户开户（时限4小时）

计算过程：
周五 16:00-17:00 → 计入1小时
周五 17:00-周一 9:00 → 不计入（非工作时间）
周一 9:00-12:00 → 计入3小时

截止时间：周一 12:00
```

**超时分级：**
- 黄色预警：剩余时限 ≤ 25%
- 橙色预警：超出时限 ≤ 50%
- 红色预警：超出时限 > 50%

#### 3.3.3 时限豁免情形

以下情形可申请时限豁免：
- 客户原因导致延误（需客户确认）
- 系统故障导致无法处理
- 需要补充材料且客户未及时提供
- 不可抗力因素

**豁免申请：**
- 需在原时限到期前提交申请
- 需说明原因并提供证明材料
- 需经主管审批同意

### 3.4 统计口径规则

#### 3.4.1 客户统计口径

**新开户统计：**
- 统计口径：账户状态为"正常"且开户日期在统计周期内
- 去重规则：同一证件号码仅统计一次
- 排除项：测试账户、内部账户、注销账户

**有效户判定：**
- 标准定义：开户后30日内完成首次交易或资产≥[X]元
- 观察期：开户后90日内
- 统计时点：以判定生效日期为准

**客户归属：**
- 归属原则：以客户首次开户所在机构为准
- 变更规则：客户主动申请变更且经审批
- 冲突处理：多机构争议由区域协调决定

#### 3.4.2 业绩统计口径

**资产统计：**
- 统计时点：每个交易日日终
- 统计范围：客户账户内所有金融资产
- 计算方式：市值法（含浮动盈亏）

**收入统计：**
- 佣金收入：实际成交产生的佣金
- 其他收入：利息、服务费等
- 归属规则：以客户交易归属机构为准

**业绩分配：**
- 开发业绩：首年收入的[X]%计入开发人
- 服务业绩：后续收入的[Y]%计入服务人
- 协作业绩：多人协作按约定比例分配

#### 3.4.3 服务质量统计口径

**满意度统计：**
- 采样方式：随机抽样 + 事后回访
- 有效样本：完成全部问题的评价
- 计算方式：加权平均分

**投诉统计：**
- 统计范围：正式登记的客户投诉
- 分类标准：按问题类型和严重程度
- 解决判定：客户确认满意或问题已修复

**时效统计：**
- 计算起点：任务创建/客户提交时间
- 计算终点：任务完成/客户确认时间
- 排除项：豁免时限的非工作时间
