# 投资银行数字员工智能体方案

**方案版本：** V1.0  
**编制日期：** 2026年3月5日  
**业务线：** 投资银行（股权、债券、并购）  
**方案定位：** 八大应用智能体之一  

---

## 一、方案概述

### 1.1 战略目标契合

本方案基于公司级**自动驾驶大模型底座**设计，实现投行业务能力向前端团队开放，支撑组织从"割裂的多部门"走向"统一的能力平台"。

### 1.2 业务定位

投资银行智能体覆盖**股权业务**（IPO、再融资）、**债券业务**（公司债、企业债、ABS）、**并购业务**（财务顾问）三大核心板块，通过AI化专业能力赋能前端展业。

### 1.3 核心价值

- **能力开放：** 中后台投行专业能力通过AI向前端打开，无需层层审批
- **流程提速：** 招股书/募集说明书生成效率提升60%+
- **合规保障：** 关键节点人工确认，满足监管要求
- **知识沉淀：** 项目经验结构化存储，新人快速上手

## 三、角色设计（1+5架构）

### 3.1 组织者：投资银行协调官

**核心职责：** 统筹投行业务全流程，协调各超级数字员工协作，对接公司战略目标。

| 职责维度 | 具体内容 |
|----------|----------|
| 目标统筹 | 统筹五大业务目标执行；协调各超级数字员工协作；确保项目按时推进 |
| 重点事项请示 | 识别需公司层面决策事项，向领导层圆桌会议提请：重大项目立项（超权限的IPO/并购项目）、重大风险事项（合规危机、监管问询升级）、跨部门协调受阻（需其他业务线支持）、资源瓶颈（人力/时间/成本超预算）、创新业务模式（新类型项目/新结构设计） |
| 战略目标接收 | 接收公司管理层派发的年度/季度业务目标；拆解为可执行任务；分配至各数字员工 |
| 考核管理 | 执行AI驱动的双重考核机制；生成绩效报表；预警偏离指标 |
| 质量把控 | 审核关键交付物质量；确保合规标准；把控项目风险 |

## 六、考核机制（AI驱动双重考核）

### 6.1 考核框架

| 考核对象 | 考核类型 | 考核内容 | 考核方式 |
|----------|----------|----------|----------|
| 业务支持中心（中后台） | 公司总量考核 | 项目承做总数、申报材料质量评分、过会率/发行成功率、底座能力贡献度、知识沉淀数量 | AI自动统计：项目完成数、内核通过率、监管反馈轮次、数据回流质量 |
| 前台业务团队（项目承揽） | 单项业务达成考核 | 项目立项数、成功发行项目数、承销金额、客户满意度评分、新客户开发数 | AI自动统计：个人业绩看板、项目跟进及时率、客户拜访记录、收入贡献 |

### 6.2 AI驱动考核机制说明

- **实时数据采集：** 底座自动记录每个项目阶段推进情况、文档质量评分、时间节点达成率
- **智能评分：** 基于业务规则自动计算考核得分，申报材料质量由AI辅助评估
- **预警提醒：** 项目进度偏离计划、质量指标下降时自动提醒相关人员
- **双向反馈：** 前台业绩数据回流到底座，优化客户画像和风控模型
- **知识沉淀量化：** 新创建/复用的模板、案例、最佳实践计入考核加分项

### 6.3 关键考核指标（KPI）

| 指标类型 | 指标名称 | 目标值 | 数据来源 |
|----------|----------|--------|----------|
| **效率指标** | 招股书初稿生成时间 | ≤5个工作日 | AI记录 |
| | 尽调报告生成时间 | ≤3个工作日 | AI记录 |
| | 反馈回复初稿生成时间 | ≤2个工作日 | AI记录 |
| | 项目立项审批时效 | ≤1个工作日 | 工作流引擎 |
| **质量指标** | 内核一次通过率 | ≥80% | 内核系统 |
| | 监管反馈轮次 | ≤2轮 | 申报系统 |
| | 材料格式差错率 | ≤1% | AI质检 |
| **业务指标** | 项目完成数（年） | 按业务目标 | 项目管理系统 |
| | 承销金额 | 按业务目标 | 财务系统 |

## 七、提示词工程规范

### 7.1 词汇规范

- 以**动词+名词**为主，占比>80%
- 形容词使用<10%，避免冗余修饰
- 使用指令性语句，动作边界清晰

### 7.2 表达简洁性

- 单句<15字，便于理解和执行
- 一句话一个动作，避免复合指令
- 去除"应该"、"尽量"等模糊表述

### 7.3 示例对比

| 不规范表述 | 规范表述 |
|------------|----------|
| 请尽可能详细地收集客户的财务信息 | 收集客户近三年财报；提取关键财务指标；识别异常波动 |
| 适当关注客户的关联交易情况 | 识别关联方清单；比对交易定价；标记公允性异常 |
| 根据实际情况生成尽调报告 | 按尽调清单输出报告；引用底稿索引；标注重大风险 |

### 7.4 双层提示词结构

| 层级 | 用途 | 内容 |
|------|------|------|
| **工程级（系统提示词）** | 定义角色、约束、工具 | Role/Goal/Backstory、可调用的底座能力、禁止事项、输出格式 |
| **交互级（用户提示词）** | 动态上下文、个性化 | 具体项目信息、当前任务、特殊要求、参考案例 |

## 八、合规与风控机制

### 8.1 Human-in-the-loop节点

| 节点 | 人工确认内容 | 确认方式 |
|------|-------------|----------|
| 项目立项 | 立项报告审核、项目风险评级、团队配置 | 立项委员会审批 |
| 内核会议 | 申报材料质量、重大风险事项、推荐意见 | 内核委员表决 |
| 申报提交 | 招股书终稿、重大事项承诺、签字页 | 质控负责人+保荐代表人双签 |
| 发行定价 | 定价区间、配售方案、战略投资者 | 发行委员会审批 |
| 重大事项 | 客户重大风险事件、舆情危机、监管问询升级 | 部门负责人+合规总监 |

### 8.2 数据安全与合规

- **敏感数据脱敏：** 客户敏感信息在AI交互中自动脱敏
- **权限分级：** 项目信息按密级设置访问权限
- **操作审计：** 所有AI操作记录完整日志，支持追溯
- **合规检查：** 自动检测信息披露合规性，标记风险点

### 8.3 风险熔断机制

- **异常检测：** AI识别异常操作或风险信号时自动暂停
- **人工接管：** 关键节点必须由人工确认后方可继续
- **版本回滚：** 文档修改历史保存，支持快速回滚
- **紧急停止：** 紧急情况可一键停止AI操作，转人工处理
