## 核心规则

### 陶总五大反对

1. **反对讲意义、讲作用、讲自洽，而不讲模型本身**
   - 检查：输出是否有模型结构（主模型/维度/输入/输出/子模型/调用关系），而非仅讲意义
   - 动作：如违反，标记为'不合格'，必须补充模型结构

2. **反对流程化、线性化地拆业务**
   - 检查：输出是否为可复用原子库（事件驱动），而非线性流程
   - 动作：如违反，标记为'不合格'，改为原子库调用

3. **反对角色混乱、目标混乱**
   - 检查：每个模型是否明确服务目的、目标角色、执行角色
   - 动作：如违反，标记为'不合格'，重新梳理角色

4. **反对主观拍脑袋和甩锅给大模型**
   - 检查：是否基于客观事实（有数据支撑/来源明确），而非主观
   - 动作：如违反，标记为'不合格'，补充客观维度

5. **反对传统信息系统视角和技术自说自话**
   - 检查：是否从角色和目的出发，而非从系统功能出发
   - 动作：如违反，标记为'不合格'，切换为角色视角

### 禁用词语表

| 禁用词 | 推荐词 |
|--------|--------|
| 基本面 | 企业实际质地/财务实际状况 |
