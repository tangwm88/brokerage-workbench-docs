# 机构经纪员工端工作台 - 一个月MVP实现方案

**版本**：v1.0  
**日期**：2026-03-06  
**目标**：基于已设计原型，一个月完成机构经纪业务员工端工作台落地

---

## 一、方案概述

### 1.1 核心思路
- **自动化功能**：采用OpenClaw方式实现（客户查重、表单预审、AI画像生成等）
- **智能功能**：采用智能体调用方式（复杂咨询、智能推荐、请示内容生成等）
- **前端入口**：保持Web H5方式，与原型设计保持一致

### 1.2 技术选型
| 层级 | 技术方案 |
|------|----------|
| 智能体编排 | OpenClaw Gateway |
| 前端框架 | Vue3 + Vite + TypeScript |
| 后端服务 | Node.js / Python FastAPI |
| 数据库 | PostgreSQL + Redis |
| 部署 | Docker Compose |

---

## 二、MVP功能范围（精简聚焦）

| 模块 | 功能 | 实现方式 | 优先级 |
|------|------|----------|--------|
| **工作台首页** | 业绩看板、待办提醒、快捷入口 | H5 + API | P0 |
| **机器人对话** | 客户查询、事项推进、方案定制 | OpenClaw Agent | P0 |
| **客户开发** | 线索登记、客户查重、AI画像 | OpenClaw自动化 + 智能体 | P0 |
| **快速开户** | 表单填写、AI预审、进度跟踪 | OpenClaw自动化 | P0 |
| **我的客户** | 客户列表、详情查看、持仓查询 | H5 + API | P1 |
| **请示协同** | 向上请示、横向协同申请 | 智能体调用 | P1 |
| **交易服务** | 算法下单、成交查询 | API对接 | P2 |

## 四、自动化 vs 智能体调用划分

| 功能场景 | 自动化 (OpenClaw) | 智能体调用 |
|---------|------------------|-----------|
| **客户查重** | ✓ 自动比对数据库，返回结果 | - |
| **AI画像生成** | ✓ 调用底座关系网络自动构建 | - |
| **表单预审** | ✓ 规则引擎自动检查 | - |
| **进度状态同步** | ✓ 定时任务自动更新 | - |
| **智能推荐** | - | ✓ 大模型推理后返回 |
| **复杂咨询** | - | ✓ 多轮对话理解意图 |
| **请示审批** | ✓ 流程自动流转 | ✓ 内容智能生成 |

## 六、关键接口设计

### 6.1 自动化Tool接口（OpenClaw Skills）

```yaml
# skill: customer-duplicate-check
name: 客户查重
description: 自动查询客户是否已存在，返回相似度分析
inputs:
  - customer_name: 客户名称
  - unified_code: 统一社会信用代码
outputs:
  - is_duplicate: 是否重复
  - similar_customers: 相似客户列表
  - confidence: 置信度

---

# skill: account-pre-check  
name: 开户预审
description: 自动检查开户资料完整性和合规性
inputs:
  - form_data: 表单数据
outputs:
  - check_result: 通过/不通过
  - missing_items: 缺失项列表
  - risk_alerts: 风险提醒

---

# skill: customer-profile-generate
name: AI画像生成
description: 基于底座数据自动生成客户画像
inputs:
  - customer_id: 客户ID
outputs:
  - profile: 画像数据（行业/规模/需求/风险）
```

### 6.2 H5前端API

```yaml
# ========== 工作台首页 ==========
GET /api/dashboard/summary        # 业绩摘要
GET /api/dashboard/todos          # 待办列表
GET /api/dashboard/performance    # 业绩趋势

# ========== 客户管理 ==========
GET /api/customers                # 客户列表
GET /api/customers/:id            # 客户详情
POST /api/customers               # 新增客户
POST /api/customers/:id/check-duplicate  # 查重

# ========== 开户流程 ==========
POST /api/account-opening         # 提交开户申请
GET /api/account-opening/:id/status      # 查询进度
POST /api/account-opening/:id/pre-check  # AI预审

# ========== 请示协同 ==========
POST /api/collaboration/request   # 提交请示
GET /api/collaboration/requests   # 请示列表
```

### 6.3 智能体调用接口

```yaml
# 机器人对话（WebSocket）
WS /ws/agent/chat
  → { message: "我想查看客户XXX的持仓", session_id: "xxx" }
  ← { type: "text", content: "..." }
  ← { type: "card", content: {持仓卡片数据} }

# 智能体服务
POST /api/agent/ask               # 单轮问答
POST /api/agent/task              # 任务委托（生成经营方案等）
```

## 八、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 底座接口不稳定 | 高 | 第1周优先对接，预留Mock方案 |

| 业务需求变更 | 中 | 每周对齐，冻结第3周需求 |
| 性能不达标 | 中 | 第3周开始压测，预留优化时间 |
