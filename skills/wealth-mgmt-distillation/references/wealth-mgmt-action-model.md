# 资产管理业务规定动作模型（7事件）

> **版本**：v1.0
> **日期**：2026-06-03
> **定位**：为资产管理业务分析模型配套设计可执行的规定动作
> **核心原则**：分析模型判断"是什么"，规定动作判断"做什么"
> **角色体系**：资管产品官（产品创设+销售支持）+ 合规守卫（合规风控）+ 组织者（统筹）

---

## 整体动作链路

```
产品立项 → 策略回测 → 结构设计 → 合规审查 → 备案发行 → 产品销售 → 投后服务
   ↑___________________________________________________________↓
                          （发现新需求，循环）
```

---

## 事件1：产品立项

**事件定义**：产品需求洞察完成，启动正式产品立项
**触发条件**：
- 产品需求洞察报告完成
- 客户/市场调研显示有足够需求
- 初步产品概念已通过内部讨论
**事件等级**：重点

### 响应角色与动作

| 角色 | 动作 | 时限 | 依赖 |
|------|------|------|------|
| 资管产品官 | 需求评估与立项申请 | 3个工作日内 | 需求洞察完成 |
| 组织者 | 资源预判与优先级排序 | 1个工作日内 | 需求洞察完成 |

#### 资管产品官：需求评估与立项申请

**触发条件**：产品需求洞察完成
**执行步骤**：
1. 调用分析模型维度1（产品需求），评估需求真实性：
   - 目标客户画像确认（年龄/资产规模/投资经验）
   - 需求来源验证（客户意向书/调研数据/竞品启发）
   - 需求紧迫度评估（急需/近期/长期）
2. 评估竞争格局：
   - 竞品数量与市场容量
   - 竞品表现（收益/回撤/规模变化）
   - 差异化空间分析
3. 预判目标募集规模：
   - 基于渠道能力/品牌认知度/历史表现
   - 规模合理性评估（目标/市场容量比例）
4. 形成《产品立项评估表》
5. 提交立项申请
**输出物**：
- 《产品立项评估表》（含需求分析、竞争格局、规模预判）
- 立项建议：[建议立项]/[需谨慎]/[建议暂缓]
**异常处理**：
- 需求验证不足（意向客户<30人）→ 标记"需求不足" → 扩大客户调研 → 组织者判断是否继续
- 竞品过多（市场已有>50只同类产品）→ 标记"竞争激烈" → 评估差异化空间 → 如差异化不明显 → 建议暂缓
- 目标规模过大（目标/市场>20%）→ 标记"规模偏高" → 调整目标 → 或分阶段发行
**时限要求**：3个工作日内完成
**风险等级**：必须确认

#### 组织者：资源预判与优先级排序

**触发条件**：需求洞察完成
**执行步骤**：
1. 登记立项到资管经营看板（产品名称、预计规模、预计时间、需求来源）
2. 预判资源需求：
   - 是否需要特定策略团队支持（量化/固收/权益）
   - 预计占用产品额度（如：大集合限额）
3. 检查是否有历史产品冲突（同策略/同目标客群的产品）
4. 判断立项优先级（高/中/低）
**输出物**：
- 《立项登记台账》更新
- 资源预判结论：[需专项支持]/[常规跟进]/[暂不投入]
**异常处理**：
- 发现历史产品冲突（如：已有同策略产品在售）→ 标记冲突 → 评估是否替代或互补
- 策略团队资源不足 → 标记"资源缺口" → 协调资源或推迟立项
**时限要求**：1个工作日内
**风险等级**：可以

#### 同步机制

```
需求洞察完成
    │
    ├─→ 资管产品官（3日）需求评估+立项申请 ──┐
    │                                         ├─→ 立项评估表
    └─→ 组织者（1日）资源预判+优先级排序 ─────┘
                                              │
                                              ▼
                                   组织者判断：立项/补充/暂缓
```

## 事件2：策略回测

**事件定义**：产品立项通过，进入投资策略回测阶段
**触发条件**：
- 产品立项批准
- 初步投资策略已形成
**事件等级**：重点

### 响应角色与动作

| 角色 | 动作 | 时限 | 依赖 |
|------|------|------|------|
| 资管产品官 | 策略回测与风险评估 | 5个工作日内 | 立项通过 |

#### 资管产品官：策略回测与风险评估

**触发条件**：产品立项通过，初步策略已形成
**执行步骤**：
1. 策略描述确认：
   - 策略类型（股票多头/量化对冲/债券增强/多策略FOF）
   - 核心逻辑（基于什么规律/因子/信号）
   - 投资范围（股票/债券/衍生品/商品）
   - 策略容量（最多能管多少钱）
2. 历史回测执行：
   - 回测区间至少5年（含牛熊周期）
   - 回测年化收益、最大回撤、夏普比率、Calmar比率
   - 月度胜率、收益分布
3. 风险测算：
   - VaR（95%置信度下日/周/月最大损失）
   - 情景分析（极端市场表现）
   - 压力测试（利率/股市/信用违约冲击）
   - 策略失效风险识别
   - 流动性风险评估
4. 形成《策略回测报告》
**输出物**：
- 《策略回测报告》（含策略描述、回测结果、风险测算）
- 策略建议：[建议采用]/[需优化]/[不建议采用]
**异常处理**：
- 回测数据不足（<5年）→ 标记"数据不足" → 补充数据或缩短回测期 → 注明局限性
- 策略回测不通过（收益<基准或回撤>30%）→ 标记"策略不通过" → 优化策略或终止立项
- 策略容量有限（<目标规模）→ 标记"容量不足" → 调整目标规模或分多期发行
**时限要求**：5个工作日内完成
**风险等级**：必须确认

## 事件3：结构设计

**事件定义**：策略回测通过，进入产品结构设计和合同准备
**触发条件**：
- 策略回测通过
- 产品概念和策略已确定
**事件等级**：重点

### 响应角色与动作

| 角色 | 动作 | 时限 | 依赖 |
|------|------|------|------|
| 资管产品官 | 产品结构设计 | 5个工作日内 | 策略回测通过 |
| 合规守卫 | 合规预审 | 2个工作日内 | 结构设计完成 |

#### 资管产品官：产品结构设计

**触发条件**：策略回测通过
**执行步骤**：
1. 产品类型确定：
   - 公募基金/私募基金/券商资管计划/信托计划
   - 单层/双层/FOF/MOM结构
2. 份额设计：
   - A类（零售）/B类（机构）/C类（定制）
   - 各类份额的费率、起投金额、开放机制
3. 费率结构设计：
   - 管理费（XX%/年）
   - 托管费（XX%/年）
   - 业绩报酬（XX%，高水位法）
4. 开放机制设计：
   - 封闭式（XX年）/定期开放（每季度）/每日申赎
5. 形成《产品结构设计书》
**输出物**：
- 《产品结构设计书》（含产品类型、份额、费率、开放机制）
**异常处理**：
- 产品结构复杂（如FOF嵌套）→ 标记"结构复杂" → 评估合规可行性 → 简化结构
- 费率竞争力不足（高于同类平均）→ 标记"费率偏高" → 优化费率或增加差异化服务
**时限要求**：5个工作日内完成
**风险等级**：必须确认

#### 合规守卫：合规预审

**触发条件**：结构设计完成
**执行步骤**：
1. 审查产品类型合规：投资范围是否符合该类型产品限制
2. 审查杠杆比例：是否超过监管上限（如：券商资管≤200%）
3. 审查集中度限制：单一资产/发行人占比是否超限（如：≤10%）
4. 审查关联交易：是否涉及关联交易（需披露+审批）
5. 审查投资者适当性：目标客户是否符合合格投资者要求
6. 审查备案要求：是否需要备案/注册/审批
**输出物**：
- 《合规预审意见》（合规状态/需调整项/不通过项）
**异常处理**：
- 杠杆比例超限 → 标记"杠杆超标" → 调整结构（降低杠杆）
- 集中度超限 → 标记"集中度超标" → 分散投资或调整持仓上限
- 涉及关联交易 → 标记"关联交易" → 补充披露文件 + 合规审批流程
**时限要求**：2个工作日内完成
**风险等级**：必须确认

#### 同步机制

```
策略回测通过
    │
    ├─→ 资管产品官（5日）结构设计 ──────┐
    │                                   ├─→ 结构设计书
    └─→ 合规守卫（2日，结构提交后）预审 ─┘
                                        │
                                        ▼
                             [合规] → 备案准备
                             [需调整] → 退回修改
                             [不通过] → 终止
```

## 事件4：合规审查

**事件定义**：结构设计合规预审通过，进入正式合规审查
**触发条件**：
- 结构设计合规预审通过
- 合同/招募说明书初稿完成
**事件等级**：重大

### 响应角色与动作

| 角色 | 动作 | 时限 | 依赖 |
|------|------|------|------|
| 合规守卫 | 合规审查与风险复核 | 5个工作日内 | 合同初稿完成 |
| 资管产品官 | 合同与招募书制作 | 10个工作日内 | 结构设计完成 |

#### 资管产品官：合同与招募书制作

**触发条件**：结构设计完成
**执行步骤**：
1. 编制合同/协议：
   - 资产管理合同（与管理人的权利义务）
   - 托管协议（与托管人的权利义务）
2. 编制招募说明书：
   - 产品基本情况（名称/类型/投资范围/策略）
   - 风险收益特征（适合什么风险承受能力的客户）
   - 费率结构（管理费/托管费/业绩报酬）
   - 投资限制（集中度/杠杆/关联交易）
   - 风险揭示（市场风险/流动性风险/策略失效风险）
3. 引用数据质量检查（交叉核对历史回测/风险测算）
4. 格式合规检查（符合证监会/基金业协会格式要求）
**输出物**：
- 《资产管理合同》初稿
- 《托管协议》初稿
- 《招募说明书》初稿
- 数据引用核对表
**异常处理**：
- 合同条款与监管要求冲突 → 标记"条款冲突" → 修改条款 → 合规确认
- 风险揭示不充分 → 标记"风险不足" → 补充风险条目 → 合规审查确认
**时限要求**：10个工作日内完成
**风险等级**：必须确认

#### 合规守卫：合规审查与风险复核

**触发条件**：合同/招募说明书初稿完成
**执行步骤**：
1. 审查合同合规性：
   - 合同条款是否符合监管要求（如：不得承诺保本保收益）
   - 权利义务是否对等
2. 审查招募说明书：
   - 信息披露是否充分（产品基本信息/风险收益/费率）
   - 风险揭示是否完整（市场风险/流动性风险/策略风险）
   - 历史业绩引用是否合规（如：需注明"过往业绩不代表未来表现"）
3. 审查适当性安排：
   - 投资者适当性匹配机制是否明确
   - 客户风险测评是否覆盖
4. 复核利益冲突（产品与管理人/托管人是否存在关联关系）
5. 出具合规审查意见
**输出物**：
- 《合规审查意见》（合同合规/招募书合规/适当性/利益冲突）
**异常处理**：
- 合同条款违规（如：承诺保本保收益）→ 标记"违规条款" → 删除违规条款 → 重新审查
- 历史业绩引用不合规 → 标记"引用不合规" → 补充免责声明 → 重新审查
**时限要求**：5个工作日内完成
**风险等级**：必须确认

## 事件5：备案发行

**事件定义**：合规审查通过，进入监管备案/注册阶段
**触发条件**：
- 合规审查通过
- 全套材料齐备（合同/招募书/回测报告/合规意见）
**事件等级**：重大

### 响应角色与动作

| 角色 | 动作 | 时限 | 依赖 |
|------|------|------|------|
| 资管产品官 | 备案材料准备与提交 | 5个工作日内 | 合规审查通过 |
| 合规守卫 | 备案合规复核 | 3个工作日内 | 备案材料齐备 |
| 组织者 | 备案进度跟踪 | 持续 | 备案提交后 |

#### 资管产品官：备案材料准备与提交

**触发条件**：合规审查通过
**执行步骤**：
1. 准备备案材料清单：
   - 资产管理合同
   - 招募说明书
   - 策略回测报告
   - 合规审查意见
   - 其他监管要求材料（如：法律意见书）
2. 材料格式检查（符合证监会/基金业协会格式要求）
3. 提交备案/注册申请
4. 跟进备案进度，记录备案反馈
**输出物**：
- 《备案材料清单》+ 全套备案材料
- 备案申请回执
**异常处理**：
- 备案材料被退回 → 标记"退回" → 根据反馈修改 → 重新提交
- 备案反馈需补充材料 → 标记"需补充" → 补充材料 → 重新提交
**时限要求**：5个工作日内完成材料准备并提交
**风险等级**：必须确认

#### 合规守卫：备案合规复核

**触发条件**：备案材料齐备
**执行步骤**：
1. 复核备案材料完整性（对照监管要求的材料清单）
2. 复核备案材料一致性（合同/招募书/回测报告是否一致）
3. 复核备案材料合规性（是否符合最新监管要求）
4. 出具备案合规意见
**输出物**：
- 《备案合规意见》（完整性/一致性/合规性）
**异常处理**：
- 材料不完整 → 标记"材料缺失" → 补充缺失材料
- 材料不一致（如：合同与招募书费率不一致）→ 标记"不一致" → 修改统一
**时限要求**：3个工作日内完成
**风险等级**：必须确认

#### 组织者：备案进度跟踪

**触发条件**：备案提交后持续跟踪
**执行步骤**：
1. 每日/每周跟踪备案进度
2. 记录备案反馈（如有）
3. 协调补充材料（如被要求）
4. 判断备案通过时间窗口
**输出物**：
- 《备案进度跟踪表》（提交日期/反馈日期/补充材料/预计批复时间）
**异常处理**：
- 备案超时（超过平均备案周期）→ 标记"备案超时" → 了解原因 → 协调补充材料
- 备案被拒 → 标记"备案被拒" → 分析原因 → 重新设计或终止
**时限要求**：持续跟踪，关键决策当日完成
**风险等级**：可以

## 事件6：产品销售

**事件定义**：产品备案完成，进入销售阶段
**触发条件**：
- 监管备案/注册通过
- 产品正式成立
**事件等级**：重点

### 响应角色与动作

| 角色 | 动作 | 时限 | 依赖 |
|------|------|------|------|
| 资管产品官 | 客户画像与匹配推荐 | 持续 | 产品成立 |
| 合规守卫 | 适当性审查与销售合规 | 持续 | 客户认购申请 |

#### 资管产品官：客户画像与匹配推荐

**触发条件**：产品成立，开始销售
**执行步骤**：
1. 调用分析模型维度4（客户匹配），评估目标客户群：
   - 客户分层（高净值/富裕/普通）
   - 风险承受能力（C1-C5）
   - 投资经验（新手/有经验/资深）
   - 投资目标（保值/增值/高收益）
   - 流动性需求
2. 评估产品风险等级（R1-R5）
3. 执行适当性匹配：
   - 客户风险等级 ≥ 产品风险等级（如：C4客户可买R4及以下产品）
   - 计算匹配率（目标客户群中符合适当性要求的比例）
4. 制定销售策略：
   - 高匹配客户 → 重点推荐
   - 中匹配客户 → 培育后推荐
   - 低匹配客户 → 不主动推荐
5. 生成客户推荐清单
**输出物**：
- 《客户匹配分析报告》（客户画像/风险等级/匹配率/销售策略）
- 《客户推荐清单》（高匹配客户名单）
**异常处理**：
- 目标客户匹配率过低（<50%）→ 标记"匹配率低" → 重新评估目标客户群 → 或调整产品结构
- 客户风险测评过期（>1年）→ 标记"测评过期" → 要求客户重新测评 → 匹配后才能推荐
**时限要求**：产品成立后立即启动，持续进行
**风险等级**：必须确认

#### 合规守卫：适当性审查与销售合规

**触发条件**：客户提交认购申请
**执行步骤**：
1. 审查客户风险测评结果：
   - 测评是否在有效期内（≤1年）
   - 测评结果是否真实（无异常答题模式）
2. 执行适当性匹配：
   - 客户风险等级 ≥ 产品风险等级
   - 如不匹配 → 拒绝认购 → 建议更适合的产品
3. 审查销售话术合规：
   - 是否承诺保本保收益（红线）
   - 是否夸大历史业绩（红线）
   - 是否充分揭示风险（必须）
4. 审查客户资金来源合规
5. 出具适当性审查意见
**输出物**：
- 《适当性审查意见》（测评有效性/匹配结果/销售合规/资金来源）
**异常处理**：
- 客户风险测评过期 → 标记"测评过期" → 要求重新测评 → 测评后才能继续
- 客户风险等级不匹配（如C2客户要买R4产品）→ 标记"不匹配" → 拒绝认购 → 推荐R2及以下产品
- 销售话术违规 → 标记"话术违规" → 叫停销售 → 重新培训销售人员
**时限要求**：每个客户认购申请T+1日内完成审查
**风险等级**：必须确认

---

## 事件7：投后服务

**事件定义**：产品签约完成，进入投后服务阶段
**触发条件**：
- 产品正式成立并运行
- 客户完成认购
**事件等级**：常规

### 响应角色与动作

| 角色 | 动作 | 时限 | 依赖 |
|------|------|------|------|
| 资管产品官 | 投后报告生成与净值跟踪 | 持续 | 产品运行 |
| 合规守卫 | 关联交易监测与风险预警 | 持续 | 产品运行 |

#### 资管产品官：投后报告生成与净值跟踪

**触发条件**：产品运行，需要定期跟踪
**执行步骤**：
1. 净值跟踪：
   - 每日更新净值（开放式产品）/ 每周更新净值（封闭式产品）
   - 对比业绩基准（沪深300/中证500/中债总指数）
   - 计算超额收益
2. 风险指标监控：
   - 最大回撤（实时计算）
   - 波动率（滚动计算）
   - 夏普比率
3. 收益归因分析（月度/季度）：
   - 收益来自选股/择时/行业配置
   - 各因子的贡献度
4. 生成投后报告：
   - 周报：净值表现 + 市场回顾
   - 月报：净值表现 + 风险指标 + 持仓变化
   - 季报：净值表现 + 收益归因 + 策略回顾 + 展望
**输出物**：
- 《投后周报》（净值/基准/超额收益）
- 《投后月报》（净值/风险指标/持仓变化）
- 《投后季报》（净值/归因分析/策略回顾/展望）
**异常处理**：
- 净值异常波动（单日涨跌幅>5%）→ 标记"净值异常" → 排查原因 → 通知客户
- 最大回撤接近警戒线（如：15%）→ 标记"回撤预警" → 加强风控 → 通知客户
- 业绩持续跑输基准（连续3个月）→ 标记"业绩落后" → 策略复盘 → 考虑调整
**时限要求**：周报每周一、月报每月5日前、季报每季度结束后15日内
**风险等级**：可以

#### 合规守卫：关联交易监测与风险预警

**触发条件**：产品运行，需持续监测
**执行步骤**：
1. 关联交易监测：
   - 每日扫描持仓，识别关联方交易
   - 记录关联交易金额、占比、定价
   - 检查关联交易是否已披露+审批
2. 风险指标监控：
   - 集中度监控（前10大持仓占比）
   - 杠杆监控（总杠杆比例）
   - 流动性监控（持仓日均成交量）
3. 风险预警：
   - 绿色（正常）：所有指标正常
   - 黄色（注意）：某指标接近警戒线
   - 红色（预警）：某指标突破警戒线
4. 合规报告（月度）：
   - 关联交易汇总
   - 风险指标统计
   - 异常事件记录
**输出物**：
- 《关联交易监测日报》（交易笔数/金额/占比）
- 《风险指标监测周报》（集中度/杠杆/流动性）
- 《合规报告月报》（关联交易汇总/风险统计/异常事件）
**异常处理**：
- 发现未披露关联交易 → 标记"未披露关联交易" → 立即披露 → 补充审批流程
- 关联交易定价不公允 → 标记"定价不公允" → 重新定价 → 补偿投资者
- 风险指标突破警戒线（如：集中度>40%）→ 标记"集中度超标" → 减仓调整
**时限要求**：日报每日、周报每周、月报每月
**风险等级**：可以

## 与资管分析模型的衔接

| 分析模型维度 | 在哪个事件中被调用 | 谁调用 | 输出如何使用 |
