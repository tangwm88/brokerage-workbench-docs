# 智能体业务建模方法论

## 一、目标分析框架

### 1.1 组织转型目标符合性检查

| 检查项 | 检查要点 | 符合标准 |
|--------|----------|----------|
| **能力开放** | 是否将中后台专业能力通过AI方式向前端团队"打开" | 前端可直接调用中后台能力，无需层层审批 |
| **组织扁平化** | 是否打破"割裂的多部门"走向"统一的能力平台" | 减少跨部门协调成本，提升响应效率 |
| **组织者角色** | 是否体现业务支持中心的核心协调角色 | 中后台从被动响应变为主动赋能的组织者 |
| **前端赋能** | 是否让前端从"层层请示"变为"直接调用能力" | 3-4步内完成业务达成 |

### 1.2 业务打开目标符合性检查

| 检查项 | 检查要点 | 符合标准 |
|--------|----------|----------|
| **业务覆盖** | 是否覆盖目标业务线的核心场景 | 主要业务场景无遗漏，边界清晰 |
| **流程打通** | 是否打通业务全流程断点 | 端到端流程闭环，异常处理完善 |
| **标准输出** | 是否输出标准化的动作、模版、规范 | 可复用、可推广、可度量 |
| **展业支持** | 是否真正提升前端展业效率 | 展业步骤简化，新人上手时间缩短 |

## 二、角色设计方法论

### 2.1 角色类型定义

| 角色类型 | 核心职责 | 设计要点 |
|---------|---------|---------|
| **组织者 (Orchestrator)** | 统筹业务目标执行、协调角色协作、异常升级决策 | 需定义请示边界、决策权限、协调规则 |
| **超级数字员工** | 执行规定动作、完成具体任务 | 需定义role/goal/backstory、动作清单、输入输出 |
| **底座能力** | 提供统一数据、规则、身份验证 | 需明确调用接口、数据格式、权限控制 |

### 2.2 组织者设计模板

```yaml
组织者:
  名称: "XXX协调官"
  职责:
    - "统筹四大业务目标执行"
    - "协调各超级数字员工协作"
    - "识别需公司层面决策事项，向领导层圆桌会议提请"
  请示事项:
    - "重点客户支持（战略级客户特殊政策）"
    - "重大困难（跨部门协调受阻、资源瓶颈）"
    - "重大风险（系统性风险、合规危机）"
    - "重要事项（组织架构调整、流程变革）"
    - "创新支持（新业务模式、技术突破）"
    - "重大投入（超预算资源申请、基础设施升级）"
  底座调用:
    - "统一语料"
    - "身份权限验证"
    - "工作流引擎"
```

### 2.3 超级数字员工设计模板

```yaml
超级数字员工:
  名称: "客户开发员"
  角色定义:
    role: "负责线索收集、资质初筛、客户画像、分配跟进的业务执行者"
    goal: "将线索高效转化为意向客户，确保线索分配及时、跟进有效"
    backstory: "你是一位经验丰富的机构客户开发专家，熟悉各类机构客户的业务特点和需求，能够快速识别高价值线索并制定跟进策略"
  
  规定动作:
    - 名称: "线索收集"
      触发条件: "新线索录入或AI挖掘发现"
      执行步骤:
        - "提取线索基本信息"
        - "查询企查查/天眼查补充工商信息"
        - "初步评估客户资质"
      输出: "线索基础档案"
      
    - 名称: "资质初筛"
      触发条件: "线索基础档案完成"
      执行步骤:
        - "核对准入标准"
        - "查询负面信息"
        - "评估风险等级"
      输出: "初筛报告"
      
    - 名称: "客户画像"
      触发条件: "初筛通过"
      执行步骤:
        - "整合多维度数据"
        - "生成客户标签"
        - "评估业务潜力"
      输出: "客户画像报告"
      
    - 名称: "分配跟进"
      触发条件: "画像完成"
      执行步骤:
        - "匹配合适客户经理"
        - "生成分配建议"
        - "触发跟进任务"
      输出: "分配记录"
  
  底座调用:
    - "机构账户关联（查重、画像查询）"
    - "风控规则标准（准入审核）"
```

## 三、业务建模流程

### 3.1 业务目标拆解

```
业务目标（一级）
    └── 规定动作（二级）
            └── 执行步骤（三级）
                    └── 底座调用（支撑）
```

### 3.2 语料收集框架

| 语料类型 | 来源 | 用途 |
|---------|------|------|
| **制度规范** | 公司内部制度、监管文件 | 定义合规边界、审核标准 |
| **业务指引** | 操作手册、流程文档 | 定义标准动作、流程节点 |
| **参考话术** | 销售话术、客服话术 | 提示词工程参考 |
| **数据样例** | 历史数据、样表 | 数据建模参考 |

### 3.3 数据建模要点

#### ER图设计原则
- 识别核心实体（客户、线索、订单、账户等）
- 定义实体间关系（1:1, 1:N, N:M）
- 标注外键约束和业务规则

#### 状态机设计
- 定义完整的状态流转图
- 标注每个流转的触发条件
- 定义停留时限和超时处理

#### 字段设计规范
- 使用英文小写+下划线命名
- 标注数据类型、约束、默认值
- 添加业务含义说明

## 四、组网与底座融合

### 4.1 智能体组网原则

```
单智能体试点（3-6个月）
    ↓ 验证业务价值和技术可行性
同业务线多智能体组网（6-12个月）
    ↓ 实现业务线内协作
跨业务线底座统一（12-24个月）
    ↓ 形成公司级自动驾驶能力
```

### 4.2 底座能力调用规范

```yaml
底座调用规范:
  
  机构账户关联:
    功能: "客户查重、画像查询、持仓查询"
    输入:
      - "客户名称/统一社会信用代码"
      - "查询类型"
    输出:
      - "客户档案"
      - "关联账户列表"
      - "持仓信息"
    
  风控规则标准:
    功能: "准入审核、交易限额、异常监控"
    输入:
      - "审核对象"
      - "审核类型"
    输出:
      - "审核结果"
      - "风险提示"
      - "限额配置"
```

## 五、实施路径标准

```
第一阶段：单智能体试点（3-6个月）
  - 选择1-2个高价值场景
  - 验证业务价值和技术可行性
  - MVP范围明确，快速迭代

第二阶段：同业务线多智能体组网（6-12个月）
  - 实现业务线内智能体协作
  - 采用MCP/A2A标准协议
  - 形成业务线级能力中台

第三阶段：跨业务线底座统一（12-24个月）
  - 统一语料、身份、关系网络
  - 形成公司级自动驾驶能力
  - 组织形态重构完成
```

## 六、输出物清单

完整的智能体方案应包含：

1. **目标分析报告** - 组织转型符合性检查
2. **角色设计文档** - 组织者+超级数字员工定义
3. **业务逻辑文档** - 规定动作、流程闭环
4. **数据建模文档** - ER图、表结构、状态机
5. **语料库** - 制度规范、参考话术
6. **提示词工程文档** - 系统提示词+用户提示词
7. **评审报告** - 九维度评审结论
