# 证券业务知识库

## 一、智能体设计原则（证券公司AI转型）

### 1.1 战略目标

**基本目标**：通过智能体实现业务向前端团队打开
**理想目标**：形成公司整体运营管理的自动驾驶

### 1.2 架构分层

#### 底座层：公司级自动驾驶大模型
- **统一语料**：打通公司全量数据
- **身份打通**：员工账户与系统账户统一
- **关系网络**：关联交易、合同、行为、规则、制度
- **公共能力**：基于AI形成公司级共享能力池

#### 应用层：八大应用智能体
1. 资产管理智能体（产品销售、产品创设服务）
2. 投资银行智能体（股权、债券、并购业务）
3. 债券销售交易智能体
4. 机构经纪智能体
5. 财富管理智能体
6. 信用交易智能体
7. 研究所智能体
8. 固收投资智能体

### 1.3 业务达成模式
- **组织者**：业务支持中心的核心协调角色
- **超级数字员工**：各关键角色的AI分身
- **执行标准**：规定动作 + 动作标准 + 动作模版
- **前端体验**：3-4个关键步骤达成业务或交易

## 二、机构经纪业务知识

### 2.1 业务组织架构

| 部门/角色 | 职责 | AI化机会 |
|-----------|------|---------|
| 机构经纪业务委员会 | 业务决策部署 | 战略级决策支持 |
| 机构经纪业务办公室 | 业务组织与管理、准入审核 | 组织者角色、自动审批 |
| PB专员 | 业务承揽、尽调、客户维护 | 超级数字员工 |
| 信息技术管理部 | 系统运维服务 | 底座能力 |
| 运维中心 | 业务运维、日终清算 | 自动化运维 |
| 合规与法律事务部 | 制度合规审查 | 合规审查智能体 |
| 风险管理部 | 风险审查、交易监控 | 风控规则引擎 |

### 2.2 业务流程

```
分支机构及业务团队对拟合作对象进行尽职调查
    ↓
拟合作项目的预沟通
    ↓
合同的拟定、审核及签订
    ↓
账户的开立
    ↓
日常交易监控
```

### 2.3 准入要求

#### 专业机构准入条件
- 营业执照、税务登记证等证照齐全
- 私募基金须具备资产管理业务批复/私募基金管理人登记证明
- 产品结构、募资、投资者等各产品要素符合监管规定
- 提供各资质证明资料，且资料内容真实、准确、完整

#### 高净值客户准入条件
- 同意并自愿承担使用投资交易管理系统可能导致的所有风险事项
- 证券投资资产 ≥ 50万元
- 申请专业投资交易管理系统的高净值客户原则上应为专业投资者

### 2.4 尽调要求

#### 尽调内容（根据客户类型）

**高净值个人**：
- 基本情况：身份证、年龄、职业、收入、金融资产
- 专业投资者认定：《专业投资者告知及确认书》
- 资金来源：金融资产证明或近六个月交易流水
- 证券资产证明：≥50万元
- 负面信息核查：搜索引擎、企查查、失信记录查询平台
- 异常交易风险评估：投资风格、撤单比、交易频率
- 内幕交易风险评估：是否在上市公司任职
- 利益输送风险评估：背景情况
- 账户操作真实性：是否本人操作

**高净值机构**：
- 基本情况：营业执照、法人身份证、机构信用代码证
- 专业投资者认定
- 资金来源：近一年审计报告和财务报表
- 证券资产证明
- 负面核查（同个人）
- 异常交易风险评估（同个人）
- 内幕交易风险评估：机构人员是否在上市公司任职
- 利益输送风险评估：内控制度、关联交易
- 账户操作真实性：是否员工操作、是否内部地址

**产品类客户**：
- 管理人基本情况：营业执照、资质证明
- 管理人负面信息核查
- 产品备案/登记情况
- 产品募集情况：是否向合格投资者募集
- 产品合同审查：分级比例、劣后级、保本保收益、通道、关联关系
- 资金池业务核查
- 投资顾问资质
- 异常交易、内幕交易、利益输送风险评估
- 交易员身份核查：身份证、劳动合同/从业公示
- 登录地址核查：是否为管理人内部地址

#### 尽调要求
- 必须双人进行
- 必须留存工作底稿
- 底稿必须有索引编号
- 底稿相互引用时需交叉注明索引编号
- 尽调报告需双人签字确认

### 2.5 预沟通机制

- PB专员就拟合作项目与机构经纪业务办公室进行预沟通
- 预沟通成功后可进入下一流程
- 预沟通失败的则停止流程
- 可向机构经纪业务委员会申请复议

### 2.6 合同与司务流程

1. 机构经纪业务办公室协助PB专员拟定产品合同及协议
2. PB专员发起主经纪商业务申请流程
3. 司务附件提交
4. 分支机构根据相关指引进行司务申报
5. 司务审批流程通过后，签订相关合同、协议及用印

### 2.7 账户开立

#### 资金账户开立
- 专业机构：各分支机构按照产品银行结算账户名开立资金账户
- 高净值客户：按公司现有规定执行

#### 投资交易系统账户开立
- 由分支机构及业务团队PB专员向机构经纪业务办公室申请
- 报备IP及MAC地址
- 运维中心办理交易系统账户开立

### 2.8 程序化交易管理

#### 程序化交易报告信息
1. 账户基本信息（投资者名称、联系人、联系方式、证券账户代码等）
2. 账户资金信息（资金规模及来源、杠杆资金规模及来源、杠杆率等）
3. 交易信息（交易策略类型、交易指令执行方式、最高申报速率等）
4. 交易软件信息（软件名称及版本号、开发主体等）
5. 交易所规定的其他信息

#### 程序化交易报告流程
1. 客户首次开展程序化交易前报告
2. 分支机构协同信息技术管理部、机构经纪业务办公室核查
3. 风险管理部审核
4. 运维中心向交易所报告

### 2.9 日常交易监控

#### 监控要求
- 风险管理部统一纳入公司客户交易行为管理进行监控
- 客户交易行为存在异常的，分支机构须及时联系客户，警示其合规交易
- 原则上，分支机构须在预警当天完成客户交易行为分析
- 2个工作日内通过录音电话回访客户
- 5个工作日内提交流程

#### 回访要求
- TradeX高净值版开通后1个月内完成回访

### 2.10 FixStation模式

#### 特殊要求
- 必须现场尽调
- 必须采集现场影像：办公场地、硬件设备、经办人合影、MAC地址、资金查询、持仓查询、下单界面、风控界面、监控界面
- 必须提交策略模块说明书
- 申报速率限制：单市场≤300笔/秒且单日≤20000笔

#### 审批流程
分支机构发起 → 机构经纪业务办公室审核 → 信息技术管理部技术审查 → 合规与法律事务部合规审查 → 风险管理部风险审查 → 机构经纪业务办公室负责人审核 → 分管领导审核 → 运维中心开通

## 三、投资者适当性管理

### 3.1 适当性管理分工
- 机构经纪业务办公室：划分风险等级，制定准入标准
- 分支机构及业务团队：对客户进行投资者分类、风险承受能力评估

### 3.2 风险等级
- TradeX高净值版：中风险（R3）
- 客户风险承受能力要求：C3及以上（稳健型及以上）

### 3.3 签署文件
- 《TradeX极速交易系统高净值版使用风险提示书》
- 《适当性匹配意见及投资者确认书》
- 风险揭示、告知信息和匹配意见以及客户签署文件应当全过程录音并录像

## 四、反洗钱要求

### 4.1 反洗钱背景调查内容
1. 核查专业机构的成立时间、注册资本，评估历史产品运行情况
2. 评估专业机构注册资金与参投产品的自有资金比例是否合理
3. 评估专业机构出具的资金来源说明是否合理
4. 评估专业机构的主营业务是否涉及较高的洗钱风险
5. 专业机构是否具备开展相关业务的资质证明，所发行的产品是否通过备案
6. 其他反洗钱相关事项

### 4.2 受益所有人识别
- 识别直接或间接拥有≥25%权益的自然人
- 股权穿透至自然人层面
- 复杂股权结构需提供控制关系说明

## 五、智能体设计参考

### 5.1 机构经纪智能体角色设计

**组织者：机构经纪协调官**
- 统筹四大业务目标执行
- 协调各超级数字员工协作
- 识别需公司层面决策事项，向领导层圆桌会议提请
- 接收战略目标，拆解为可执行任务
- 执行AI驱动的双重考核机制

**请示事项**：
- 重点客户支持（战略级客户特殊政策）
- 重大困难（跨部门协调受阻、资源瓶颈）
- 重大风险（系统性风险、合规危机）
- 重要事项（组织架构调整、流程变革）
- 创新支持（新业务模式、技术突破）
- 重大投入（超预算资源申请、基础设施升级）

**超级数字员工**：
- 客户开发员：线索收集、资质初筛、客户画像、分配跟进
- 开户专员：准入审核、协议生成、权限配置、账户激活
- 交易服务员：算法配置、订单执行、异常预警、成交反馈
- 客户关怀员：咨询响应、对账服务、投诉处理、满意度管理、舆情跟踪、监管关联、重大事件监控、风险预警、公司治理跟踪

**转发边界**：
- 信用交易需求 → 信用交易智能体
- 研究服务需求 → 研究所智能体

### 5.2 底座调用

- 机构账户关联：查重、画像、持仓查询
- 风控规则标准：准入审核、交易限额、异常监控
- 身份权限验证：操作权限范围

## 七、实施路径

```
第一阶段：单智能体试点（3-6个月）
  - 选择1-2个高价值场景
  - 验证业务价值和技术可行性
  - MVP范围明确，快速迭代

第二阶段：同业务线多智能体组网（6-12个月）
  - 实现业务线内智能体协作
  - 采用MCP/A2A标准协议
  - 形成业务线级能力中台

第三阶段：跨业务线底座统一（12-24个月）
  - 统一语料、身份、关系网络
  - 形成公司级自动驾驶能力
```
